# 联网自学约束扩充流程

> AI 在对话中遇到预设选项不够时，主动联网搜索扩充。  
> 不向用户提问"应该有哪些选项"——自己去学。

---

## 核心原则：单一纯净特征

联网搜索每个维度时，**只搜该维度的单一纯净特征**，不搜复合设计结果。

- 体型 → 只搜体型分类名称（倒三角、沙漏型、萝莉体型…），不搜"精灵翼少女"（那是种族+体型复合）
- 武器 → 只搜武器类型名称（长枪、自动步枪、匕首…），不搜"虹吸水晶矛"（那是武器+元素+力量复合）
- 服装 → 只搜服装板式名称（水手服、JK、西装…），不搜"星空碎片护甲裙"（那是服装+元素+流派复合）
- 主/副元素 → 只搜单一元素（太阳、月亮、龙、羽毛、云…），不搜复合描述（"月光石纹理"=月亮+石头）
- 发型 → 只搜发型名称（齐刘海、波波头、脏辫…）
- 力量来源 → 只搜力量/能量概念名称（星陨、梦境、幽泉…），不搜"+力场""+脉动"等后缀

**判断标准**：提炼出的特征值去掉修饰词后，还能独立存在 → 是纯净特征。去掉修饰词就失去意义 → 是复合结果，丢弃。

---

## 执行时机

1. **对话启动时**：检查 `constraints/rendering/` 和 `constraints/design_movement/` 的约束文件数量。如果少于 5 个，主动联网搜索补充。
2. **用户说"还有哪些"时**：当前预设不够，联网搜索更多选项。
3. **用户自定义输入新值时**：联网搜索该值的设计特征，提炼约束入库。
4. **全随机模式下**：随机到约束库中不存在的特征时，后台联网补充。

### ⑪ 主元素 / ⑫ 副元素

**搜索关键词**:
```
"visual design elements motifs list"
"design elements sun moon feather dragon mechanical"
"设计元素 太阳 月亮 羽毛 龙 机械 云 星辰"
"character motif element list"
```

**提炼**: 单一元素名（名词）。可以是：
- 自然元素：太阳、月亮、星辰、火焰、冰雪、雷电、风、水、大地、植物、花朵、藤蔓
- 动物元素：龙、凤、虎、狼、鹰、蛇、蝶、鱼
- 人工元素：机械、齿轮、时钟、锁链、十字架、玫瑰、皇冠、钥匙
- 抽象元素：音符、太极、眼睛、骷髅、翅膀、光环

**关键**: **元素是一个名词，不是形容词+名词，不是名词+名词。**
- "太阳" ✓ → "太阳纹理" ✗（那是元素+附着方式）
- "龙" ✓ → "龙鳞纹样" ✗（那是元素+呈现方式）

**过滤**:
- 单一名词（如"月亮""机械""羽毛"）→ 保留
- 复合描述（如"月光石纹理"=月亮+石头+纹理，"梦境水波纹章"=梦境+水+波纹+章）→ 丢弃，拆解后保留最早层的名词

## XP / 萌属性 特别说明

在二次元语境中，XP（"喜好点"/"戳中萌点的特征"）不是单一设计维度，而是**跨维度的萌属性集合**。常见分类：

| 外貌类 | 性格类 | 身份类 | 服装类 |
|--------|--------|--------|--------|
| 白毛/银发 | 傲娇/ tsundere | 女仆/ maid | 水手服 |
| 兽耳/猫耳/犬耳 | 天然呆 | 学生/ JK | 旗袍 |
| 眼镜 | 病娇/ yandere | 巫女/ miko | 军装 |
| 黑丝/绝对领域 | 无口/ kuudere | 大小姐/ ojou-sama | 洛丽塔 |
| 红瞳/异色瞳 | 毒舌 | 吸血鬼 | 西装 |
| 虎牙 | 中二病 | 天使/恶魔 | 绷带 |

**处理方式**: XP 不是独立的约束维度，而是拆解到已有维度中。
- 白毛 → 发色（归入发型约束）
- 兽耳 → 种族特征（归入种族约束）
- 傲娇 → 性格 + 表情
- 黑丝 → 服装细节
- 女仆 → 服装板式（水手服/女仆装）

搜索"萌属性 分类"或"character moe traits list"来获取完整列表，然后按维度归类。

## AI 提炼通用规则

对搜索结果中的每个候选：

1. **纯净性检测**: 去掉所有修饰词后，剩余核心词是否仍是一个独立的{维度}名称？
   - 是 → 保留核心词
   - 否 → 丢弃或拆解

2. **提炼 MUST**: 该特征的不可省略元素。基于搜索结果中的客观描述，不编造。

3. **提炼 FORBID**: 该特征与其他特征的已知冲突。至少标注最明显的 1 个。

4. **提炼 SHOULD**: 该特征的常见变体或建议方向。

5. **配色约束**: 该特征是否有天然的颜色倾向。

6. **低像素适配**: 该特征在极端像素下的简化方式。

7. **按 _TEMPLATE.md 格式写入**对应分类文件夹。

## 预设列表维护规则

- SKILL.md 中的预设列表显示前 6-8 个最常用的选项
- 末尾保留"【更多选项请联网搜索推荐】"作为动态入口
- 约束库中的文件数量 = 实际可用的完整选项数
- 预设列表不必与约束库完全同步——约束库是完整知识库，预设列表是快捷入口
