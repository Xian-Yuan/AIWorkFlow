# 第三十二章·案例 11：XGSampleEMail 插件架构与四类核心设计

## 概述

XGSampleEMail 是一个自我完备的 TCP 邮件发送插件，脱离 Engine 模块依赖，可兼容独立程序运行。插件以**异步行动模式**为核心，通过四类核心类划分职责：

| 核心类 | 基类 | 职责 | 导出宏 |
|--------|------|------|--------|
| `XGSampleEMailAsyncAction` | `TSharedFromThis<>` | 异步行动封装，持有 Runnable 智能指针，管理 SMTP 状态机 | 无（内部类） |
| `XGSampleEMailRunnable` | `FRunnable` | TCP 线程主体，收发原始字节流，不关心业务协议 | 无（内部类） |
| `XGSampleEMailSubsystem` | C++ 单例 | 管理所有 AsyncAction 实例的容器 | 无（内部类） |
| `XGSampleEMailBPLibrary` | `UObject` | 蓝图层统一调用入口，查询/移除/释放行动 | 有 |

## 依赖配置

**Build.cs** 仅引用必要模块，不依赖 Engine，为独立程序预留兼容性：

```cpp
PublicDependencyModuleNames.AddRange(new string[] {
    "Core",
    "CoreUObject",
    "Networking",
    "Sockets"
});
```

- `Networking`：提供 `FTcpSocketBuilder` 等高层 Socket API
- `Sockets`：提供 `FSocket`、`ISocketSubsystem` 等底层网络基础设施
- 不含 `Engine` 模块——这是与前面章节插件（如 XGWebSocket）的关键区别

**.uplugin** 采用 `PreLoadingScreen` 加载阶段，`Runtime` 模块类型。

## 四类核心设计详解

### 1. AsyncAction（TSharedFromThis 体系）

区别于前几章继承 `UBlueprintAsyncActionBase` 的做法，本章 AsyncAction 选择继承 `TSharedFromThis`，原因是：

- 需要脱离 UObject 体系以支持独立程序使用
- `TSharedFromThis` 提供智能指针引用计数，Runnable 线程中通过 `AsShared()` 获取 shared_ptr 防止提前析构
- 线程安全模式：`ESPMode::ThreadSafe`

```cpp
class XGSampleEMailAsyncAction : public TSharedFromThis<XGSampleEMailAsyncAction, ESPMode::ThreadSafe>
```

**关键成员：**
- `FGuid ActionGUID`：构造函数中生成，唯一标识，生命周期内不可变
- `EXGSampleEMailStatus Status`：SMTP 协议状态机枚举
- `TSharedPtr<XGSampleEMailRunnable> XGEmailRunnable`：持有的 TCP 线程对象
- `FRunnableThread* EmailThread`：TCP 线程句柄
- `bool bIsActive`：活跃标志

**公开方法：**
- `CreateSendEmail()`：静态工厂方法，创建 AsyncAction → 注册到 Subsystem → 绑定回调 → 跨帧 Activate
- `Stop()`：停止行动，触发关闭流程
- `GetID()` / `GetStatus()` / `IsActive()`：查询状态
- `ExecuteDelegate()`：将委托分发到游戏线程执行

### 2. Runnable（FRunnable）

纯 TCP 通信层，只关心字节收发，不关心 SMTP 协议细节：

- 通过 `TArray<TDelegate<void()>>` 数组存储回调代理
- 回调在 AsyncAction 的 `CreateRunnable` 中绑定到 shared_ptr 生命周期上
- 回调类型：Init、InitError、Receive、Send、Close

### 3. Subsystem（C++ 单例）

非 `UGameInstanceSubsystem`，是纯 C++ 单例：

```cpp
class XGSampleEMailSubsystem {
    static XGSampleEMailSubsystem* Get();
    static void Destroy();
    // ...
};
```

- **生命周期**：在 Module `StartupModule()` 中调用 `Get()` 初始化，在 `ShutdownModule()` 中调用 `Destroy()`
- **容器**：`TMap<FGuid, TSharedPtr<XGSampleEMailAsyncAction>>` 管理所有 AsyncAction
- **CRUD 操作**：`CreateAction` / `QueryStatus` / `GetAllAction` / `RemoveAction` / `ReleaseAll`
- **设计意图**：蓝图只能通过 GUID 间接操作 AsyncAction，不能直接持有智能指针——隔离上层使用和内部实现

### 4. BPLibrary（UObject，非 UBlueprintFunctionLibrary）

与常见做法不同，本章 BP Library 继承 `UObject` 而非 `UBlueprintFunctionLibrary`：

```cpp
UCLASS()
class XGSAMPLEEMAIL_API UXGSampleEMailBPLibrary : public UObject
```

- `UObject` 的子类仍可在蓝图中作为函数库调用
- 加 `XGSAMPLEEMAIL_API` 导出宏供外部 C++ 调用
- 公开方法：`CreateSendEmail` / `GetActionNum` / `QueryAction`（GUID→状态/布尔） / `RemoveAction` / `ReleaseAll`
