# 第三方库插件搭建与模块组织

## 概述

XGSampleWebP 使用 UE 的第三方库（Third Party Library）插件模板创建。搭建过程中需要手动清理模板生成的多余文件，并按照规范的模块组织方式构建子模块目录。

## 环境准备

1. 在 `001_XGSampleDemo` 项目中新增 `023_LibWebP` 目录，创建测试用 `AXGSampleWebpActor`
2. 使用 UE 编辑器的"Add Plugin" → 选择"Third Party Library"模板，创建 `XGSampleWebP` 插件
3. 删除模板自动生成的多余代码

代码位置：
- Actor: [XGSampleWebpActor.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Source/XGSampleDemo/023_LibWebP/XGSampleWebpActor.h)
- 插件: [XGSampleWebP/](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleWebP/)

## 第三方库插件模板处理

创建 Third Party Library 模板后，需要删除的内容：

1. **删除 Module 内的默认代码**：模板自动生成的 `.cpp`/`.h` 文件全部清理
2. **只保留插件框架**：`.uplugin` 文件、`Build.cs` 文件
3. **注意 VS 可能自动创建额外文件**：如刷新后在项目源码目录中产生多余的 `.cpp`，需手动删除

## Build.cs 配置要点

### 第三库模块引入

在 `XGSampleWebPLibrary.Build.cs` 中配置静态库路径：

```csharp
// Windows 平台下添加 lib 文件
PublicAdditionalLibraries.Add(Path.Combine(ModuleDirectory, "x64", "Release", "libwebp.lib"));
PublicAdditionalLibraries.Add(Path.Combine(ModuleDirectory, "x64", "Release", "libwebpdemux.lib"));
PublicAdditionalLibraries.Add(Path.Combine(ModuleDirectory, "x64", "Release", "libwebpmux.lib"));
```

### 模块依赖

插件主模块需要依赖 `Engine` 和 `Renderer` 等模块：

```csharp
PrivateDependencyModuleNames.AddRange(new string[] { "Engine", "Renderer", ... });
```

### Public/Private 路径配置

使用 `PublicIncludePaths` 将插件内部子目录暴露给外部引用：

```csharp
PublicIncludePaths.Add(Path.Combine(ModuleDirectory, "Public/Core"));
PublicIncludePaths.Add(Path.Combine(ModuleDirectory, "Public/Subsystem"));
PublicIncludePaths.Add(Path.Combine(ModuleDirectory, "Public/Type"));
```

## 常见编译错误

| 错误 | 原因 | 修复 |
|------|------|------|
| Link 2019 | 模块未导入 / Build.cs 缺少依赖 | 检查 DependencyModuleNames |
| 找不到头文件 | Public/Private 路径未正确配置 | 在 Build.cs 中配置 PublicIncludePaths |
| 第三方库链接失败 | .lib 路径拼写错误或平台不匹配 | 确认 `ModuleDirectory` 路径和文件名 |
| VS 自动创建多余文件导致冲突 | 重载/刷新项目时 VS 在源码目录生成了同名文件 | 删除多余文件 |
