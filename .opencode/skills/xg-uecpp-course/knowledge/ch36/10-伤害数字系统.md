# 伤害数字系统

## 架构设计

伤害数字系统采用四层组件化架构，将请求、组件逻辑、Actor 生成和 UI 显示分层隔离：

```
请求层：  FXGRPGNumberPopRequest（数据请求结构体）
组件层：  UXGRPGNumberPopComponent（抽象基类）
         UXGRPGNumberPopComponent_UMG（UMG 实现子类）
Actor层： AXGRPGDamageNumActor（3D 空间中的伤害数字实体）
UI层：    UUI_DamgeNumber（Widget 控件，显示数字 + 动画）
```

这种架构借鉴了成熟项目的设计模式，将伤害数字的显示从 PlayerController 中解耦。

## 请求结构体（FXGRPGNumberPopRequest）

```cpp
struct FXGRPGNumberPopRequest
{
    FVector WorldLocation;             // 伤害发生的世界位置
    FGameplayTagContainer SourceTags;  // 源标签（预留元素反应等用途）
    FGameplayTagContainer TargetTags;  // 目标标签
    float NumberToDisplay;             // 显示的数值
    bool bIsCriticalDamage;            // 是否暴击
    FLinearColor ColorToDisplay;       // 显示颜色
};
```

`SourceTags` 和 `TargetTags` 是预留字段，可用于冰火属性等元素反应的视觉特效扩展。

## 抽象基类（UXGRPGNumberPopComponent）

```cpp
class UXGRPGNumberPopComponent : public UActorComponent
{
    virtual void AddNumberPop(const FXGRPGNumberPopRequest& Request) PURE_VIRTUAL(...);
};
```

这是一个纯接口，只声明功能不实现，由子类决定具体的显示方式。

## UMG 实现（UXGRPGNumberPopComponent_UMG）

```cpp
class UXGRPGNumberPopComponent_UMG : public UXGRPGNumberPopComponent
{
    TSubclassOf<AXGRPGDamageNumActor> NumPopActorClass;  // 要生成的 Actor 类
    
    void AddNumberPop(const FXGRPGNumberPopRequest& Request) override
    {
        if (!GetWorld() || !NumPopActorClass) return;
        
        // 仅在本地客户端执行
        if (GetOwnerRole() == ROLE_AutonomousProxy || 
            (GetOwnerRole() == ROLE_Authority && IsNetMode(NM_Standalone)))
        {
            // 根据摄像机位置添加位置随机偏移
            FVector SpawnLocation = Request.WorldLocation;
            // Spawn Actor
            AXGRPGDamageNumActor* Actor = GetWorld()->SpawnActor<AXGRPGDamageNumActor>(
                NumPopActorClass, SpawnLocation, FRotator::ZeroRotator);
            if (Actor)
            {
                Actor->UpdateNum(Request.NumberToDisplay);
                Actor->UpdateNumColor(Request.ColorToDisplay);
            }
        }
    }
};
```

组件挂载在 PlayerController 上，仅在本地客户端执行。

## 伤害数字 Actor（AXGRPGDamageNumActor）

```cpp
class AXGRPGDamageNumActor : public AActor
{
    USceneComponent* SceneComponent;     // 根组件
    UWidgetComponent* WidgetComponent;   // Widget 组件
    
    void UpdateNum(float Num);           // 更新显示的数值
    void UpdateNumColor(FLinearColor Color); // 更新颜色
};
```

- `InitialLifeSpan = 4.0f`：4 秒后自动销毁
- WidgetComponent 挂载 `UUI_DamgeNumber` Widget
- 关闭碰撞（不需要物理碰撞检测）
- 不允许网络同步（服务端不需要创建，节约内存）

## UI 控件（UUI_DamgeNumber）

```cpp
class UUI_DamgeNumber : public UUI_Base
{
    UTextBlock* NumTex;                // 数字文本框（BindWidget 绑定）
    
    void UpdateNum(float Num);         // 设置文本内容
    void UpdateNumColor(FLinearColor Color); // 设置颜色和透明度
};
```

在 `NativeConstruct` 中播放 `FadeAnimation` 渐变动画：

```cpp
void UUI_DamgeNumber::NativeConstruct()
{
    Super::NativeConstruct();
    PlayAnimation(FadeAnimation);  // 自动播放渐变动画
}
```

## 调用链路

```
GameplayCue 触发（或 GEEC 中直接调用）
    → 获取本地 PlayerController
    → 获取 NumberPopComponent
    → 构建 FXGRPGNumberPopRequest（位置、数值、颜色、是否暴击）
    → 调用 AddNumberPop()
        → UXGRPGNumberPopComponent_UMG::AddNumberPop()
            → SpawnActor<AXGRPGDamageNumActor>()
                → 通过 WidgetComponent 获取 UUI_DamgeNumber
                → 调用 UpdateNum() / UpdateNumColor()
```

## 颜色规范

项目定义了伤害数字类型枚举 `EXRPGDamageNumberType`：

| 类型 | 颜色 | 场景 |
|------|------|------|
| Red（红色） | 红色 | 普通伤害 |
| Green（绿色） | 绿色 | 生命恢复 |
| Blue（蓝色） | 蓝色 | 魔法恢复 |

暴击时可以设置为更醒目的颜色或放大字体。

## 注意事项

- 组件挂载在 PlayerController 上，不设置网络同步
- `SpawnActor` 前的 `IsNetMode` 判断确保仅在本地生成，节约服务端内存
- WidgetComponent 需要正确设置 Widget Class（蓝图中的对应类）
- 位置随机偏移避免多个伤害数字完全重叠
- `FadeAnimation` 播放结束后 Widget 不应销毁，生命周期由 Actor 控制
- 蓝图层需要 k 一个 FadeAnimation 动画给 C++ 的 `PlayAnimation` 调用
