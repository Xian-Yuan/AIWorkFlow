# 技能系统

## 技能分类

项目中一共有以下几类技能，通过独立的 GameplayAbility 实现：

| 技能 | 对应类 | InputTag | 功能 |
|------|--------|---------|------|
| 轻攻击 | XGRPGGameplayAbility（基类） | InputTag.LightAttack | 多段连击 |
| 重攻击 | XGRPGGameplayAbility | InputTag.HeavyAttack | 碰撞盒子伤害 |
| 远程魔法 | XGRPGGameplayAbility | InputTag.Skill_1 / Skill_2 | 弹道投射 |
| 冲刺 | XGRPGGameplayAbility | InputTag.Dash | 前向加速位移 |
| 大招 | XGRPGGameplayAbility | InputTag.Ultimate | 大范围 AOE |
| 生命恢复 | XGRPGGameplayAbility | InputTag.Skill_3 | 回复血量 |
| 魔法恢复 | XGRPGGameplayAbility | InputTag.Skill_4 | 回复魔法值 |

## 技能通用配置

所有技能共享以下配置：

```cpp
// 能力基类中配置
InstancingPolicy = EGameplayAbilityInstancingPolicy::InstancedPerActor;
NetExecutionPolicy = EGameplayAbilityNetExecutionPolicy::LocalPredicted;
ReplicationPolicy = EGameplayAbilityReplicationPolicy::ReplicateYes;
```

### 激活条件

能力通过 `CanActivateAbility` 检查以下条件：
- 冷却是否就绪（Cooldown GE 是否已过期）
- 资源是否充足（Mana/Stamina 是否满足 Cost GE）
- 角色是否处于可施法状态（没有被眩晕、死亡等）

## 魔法值消耗

通过 GameplayEffect 的 `Cost GameplayEffect` 槽位实现：

1. 创建 Cost GE，Modifier 设为 Mana 属性
2. Modifier Op = Additive，Magnitude 为负数
3. 将 GE 设置到 GA 的 `Cost GameplayEffect` 属性
4. 系统自动在激活前调用 `CheckCost`、激活时调用 `ApplyCost`

## 冷却系统

### 实现方式

冷却通过设置 Duration 的 GameplayEffect 实现，没有独立的冷却管理类：

1. 创建 Cooldown GE，Duration 设为冷却时长
2. 将 GE 设置到 GA 的 `Cooldown GameplayEffect` 属性
3. 系统自动在激活前检查冷却、激活后应用冷却

### 冷却监控

冷却状态通过 ASC 的 `GetActiveEffects` 或 `HasActiveEffect` 查询。UI 层通过 ASC 的委托或每帧读取来监控冷却状态并更新显示。

## 冲刺技能

冲刺通过 `AnimNotifyState_AddForce` 实现：

```
冲刺 GA 激活
    → PlayMontage（冲刺蒙太奇）
        → AnimNotifyState_AddForce 开始
            → 每帧向角色前向施加冲力
            → 持续到 Notify State 结束
    → 蒙太奇结束 → EndAbility
```

`AnimNotifyState_AddForce` 内部获取角色的 `SkeletalMeshComponent`，每帧施加冲击力，实现冲刺效果。

### 辅助 AnimNotifyState

冲刺期间同时使用以下 Notify State 控制角色行为：

| NotifyState | 作用 |
|------------|------|
| `AnimNotifyState_StopSpeed` | 冲刺期间停止速度变化 |
| `AnimNotifyState_StopRotation` | 冲刺期间锁定角色朝向 |
| `AnimNotifyState_IgnoreInput` | 冲刺期间忽略输入（防止打断） |

## 大招技能

大招（Ultimate）是对范围内所有敌人造成 AOE 伤害的能力：

```
大招 GA 激活
    → PlayMontage（大招蒙太奇）
        → AnimNotify 触发
            → 生成范围型 HitBox（RANGE 类型）
                → 检测周围所有敌人
                → 应用 GE → 伤害计算
    → 蒙太奇结束 → EndAbility
```

大招使用 `HITCOLLISIONTYPE_RANGE` 碰撞类型，碰撞盒子以角色为中心展开，检测范围内的所有敌人。

## 生命/魔法恢复技能

通过周期性（Duration + Period）GameplayEffect 实现：

1. 创建 Regen GE
2. Duration 设为恢复持续时间
3. Period 设为每次恢复的时间间隔
4. Modifier 设为 Health/Mana 属性，Modifier Op = Additive，值为正数
5. 每 Period 触发一次恢复

与 Cost GE 不同，恢复 GE 是正向的 Modifier（加血/加蓝）。

## 技能蓝图配置

所有技能的蓝图层配置包括：

- `Ability Tags`：能力的标签
- `Cancel Abilities with Tag`：激活时自动取消拥有指定标签的能力
- `Block Abilities with Tag`：激活时阻止指定标签的能力激活
- `Activation Owned Tags`：激活后赋予自身的标签
- `Cooldown GameplayEffect`：冷却 GE
- `Cost GameplayEffect`：消耗 GE

## 注意事项

- 冷却 GE 的 Duration 应设置为 `HasDuration = true`
- 如果 `CheckCost` 返回 false，能力不会激活
- 冲刺的 AnimNotifyState_AddForce 应获取角色网格体而非胶囊体
- 恢复技能的 Period 属性需要在 Service 中设置合适的值，过短会导致频繁计算
- 大招的 AOE 检测应包含 `IsExist` 判重，防止同一目标多次受伤
