# 背包系统

## 整体架构

背包系统由三部分组成：数据组件 + UI 面板 + UI 插槽。

```
XGRPGInventoryComponent（背包数据组件，Replicated）
    └── TArray<FXGRPGInventoryItem> InventoryItems（背包物品数组）
        ↓ RPC 通知
UI_InventoryPanel（背包面板）
    └── UniformGridPanel（网格布局）
        └── UI_InventorySlot × N（插槽，支持拖拽交互）
```

## 物品数据结构（FXGRPGInventoryItem）

```cpp
struct FXGRPGInventoryItem
{
    UXGRPGItem* RPGItem;    // 物品引用
    int32 ItemCount;         // 物品数量
    bool IsValid();          // 有效性检查
    void ResetSelf();        // 重置
};
```

## 背包组件（XGRPGInventoryComponent）

### 核心接口

```cpp
class UXGRPGInventoryComponent : public UXGRPGComponentBase
{
    // 物品操作
    void AddInventoryItem(FXGRPGInventoryItem NewItem);
    void RemoveInventoryItem(int32 InventoryId);
    void RemoveInventoryItemNyItem(FXGRPGInventoryItem Item);
    void ReplaceInventoryItem(FXGRPGInventoryItem Item, int32 InventoryId);
    void SwapInventoryItem(int32 InventoryId_A, int32 InventoryId_B);
    
    // 查询
    FXGRPGInventoryItem GetInventoryItemByID(int32 InventoryId);
    FXGRPGInventoryItem GetInventoryItemInfoByID(int32 InventoryId);
    bool HasThisInventoryItem(FXGRPGInventoryItem Item);
    
    // 技能激活
    void ActiveSkillByInventoryId(int32 InventoryId);
    
    // 同步
    void CallServerDownLoadInfo();
    
    // 数据
    UPROPERTY(Replicated)
    TArray<FXGRPGInventoryItem> InventoryItems;
};
```

### 网络同步

- `InventoryItems` 标记为 `Replicated`，自动从服务端同步到客户端
- `InventoryItemChanged` 是以 `Client, Reliable` 方式发送的 RPC，强制 UI 刷新
- 服务端修改物品列表后通知客户端更新显示

### 物品添加流程

```
拾取物品
    → 服务端调 AddInventoryItem
        → 检查是否有相同物品（可堆叠）
            → 是：增加 ItemCount
            → 否：添加新条目
        → 调用 InventoryItemChanged RPC
            → 客户端收到通知
                → UI_InventoryPanel::OnInventoryItemChanged
                    → 重新生成所有插槽
```

## UI 背包面板（UUI_InventoryPanel）

```cpp
class UUI_InventoryPanel : public UUI_Base
{
    UUniformGridPanel* InventoryArray;      // 网格布局容器
    TSubclassOf<UUI_InventorySlot> InventoryClass;  // 插槽类配置
    
    void OnInventoryItemChanged(const TArray<FXGRPGInventoryItem>& Items)
    {
        // 清空网格
        InventoryArray->ClearChildren();
        // 重新生成插槽
        for (int32 i = 0; i < Items.Num(); i++)
        {
            UUI_InventorySlot* Slot = CreateWidget<UUI_InventorySlot>(this, InventoryClass);
            Slot->InventoryId = i;
            Slot->SetItemData(Items[i]);
            InventoryArray->AddChildToUniformGrid(Slot, i / Columns, i % Columns);
        }
    }
};
```

## UI 背包插槽（UUI_InventorySlot）

```cpp
class UUI_InventorySlot : public UUI_SlotBase
{
    int32 InventoryId;                        // 槽位 ID
    UTextBlock* Num;                          // 数量显示
    TSubclassOf<UUI_ICONDragDrog> ICODragDrogClass;  // 拖拽图标类
    
    void HideInventoryNum();                  // 隐藏数量
    void ShowInventoryNum();                  // 显示数量
    void OnClickedWidget();                   // 点击激活（使用物品）
    
    // 拖拽支持
    FReply NativeOnMouseButtonDown(...) override;
    void NativeOnDragDetected(...) override;
    bool NativeOnDrop(...) override;
};
```

### 拖拽交互

插槽支持三项拖拽操作：

1. **NativeOnMouseButtonDown**：检测鼠标按下，触发拖拽检测
2. **NativeOnDragDetected**：检测到拖拽时创建拖拽图标（`UUI_ICONDragDrog`），携带物品数据
3. **NativeOnDrop**：放下时检测目标槽位，执行交换逻辑

拖拽图标 `UUI_ICONDragDrog` 是一个跟随鼠标的 Widget，显示被拖拽物品的图标。

## 物品基类（UXGRPGItem）

```cpp
class UXGRPGItem : public UObject
{
    // 物品通用属性（在蓝图中配置）
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    FText ItemName;
    
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    UTexture2D* ItemIcon;
    
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly)
    FText ItemDescription;
};
```

物品是 `UObject` 而非 `AActor`，不直接存在于世界中。世界中的拾取物是一个 Actor，其上挂载物品数据引用。

## 注意事项

- 背包数据通过 `Replicated` 和 RPC 双保险同步
- 同类物品只需增加 `ItemCount` 即可，不需要新增条目
- 插槽的拖拽交互走 Widget 原生事件，不需要额外的输入组件
- 物品激活（`ActiveSkillByInventoryId`）用于消耗品（如药水）
- 背包 UI 的刷新策略是重新生成所有 Slots（数据量小，性能可接受）
