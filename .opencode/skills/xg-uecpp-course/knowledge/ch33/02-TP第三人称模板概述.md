# TP（第三人称）模板概述

## 架构特点

TP 模板与 FP 共享相同的 GameMode → PlayerController → Character 基础框架，但以下方面不同：

| 方面 | FP | TP |
|------|----|----|
| 摄像机 | Camera 挂在 SpringArm 下，第一人称视角 | FollowCamera 挂在 SpringArm 下，第三人称跟拍 |
| 角色 Mesh | 只有手部 Mesh（OwnerNoSee） | 完整全身骨骼网格体 |
| 动画 | 动画蓝图（手部） | 动画蓝图 + 状态机 + ControlRig 脚部 IK（本章未深讲） |
| IMC 注册 | PlayerController::BeginPlay | Character::BeginPlay |

## IMC 注册差异

TP 蓝图中，IMC 注册在 Character 的 BeginPlay 中完成，而非 PlayerController：

```
Character::BeginPlay()
  → 获取 Controller → Cast 到 APlayerController
  → 获取 LocalPlayer → ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>()
  → AddMappingContext(IMC_Default, 0)
```

## 输入与移动

TP 的输入绑定与 FP 一致（Move/Look/Jump），区别在于 CameraBoom 的角色：
- CameraBoom 挂在 CapsuleComponent 下
- FollowCamera 挂在 CameraBoom 下
- `bUsePawnControlRotation = true` 使 CameraBoom 跟随控制器旋转
- 角色旋转由动画控制，与摄像机旋转解耦

## 多人网络用途

本章的 TP 项目将作为后续章节（多人网络游戏基础）的起点项目。其 PIE 调试模式展示了虚幻的多客户端架构：

| PIE 模式 | 说明 |
|---|---|
| Standalone | 独立游戏（无联网） |
| Client + Server | 多个客户端连接一个专属服务器 |
| Listen Server | 一个客户端同时作为服务器 |

## C++ 代码结构

TP 的 C++ 实现相对简洁：
- **GameMode**：仅设置默认蓝图类（BP_ThirdPersonGameMode），无自定义逻辑
- **Character**：声明 `UCameraBoom*` + `UCameraComponent*`，BeginPlay 注册 IMC，SetupPlayerInputComponent 绑定输入
- **PlayerController**：无额外逻辑
