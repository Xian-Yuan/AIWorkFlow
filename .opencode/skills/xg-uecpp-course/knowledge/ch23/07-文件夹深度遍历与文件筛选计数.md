# 文件夹深度遍历与文件筛选计数

## 概述

本章的核心算法：在分帧状态下逐帧执行目录遍历→文件筛选→行数统计，每帧只处理一个任务单元（一个目录或一个文件），避免单帧卡死。

## 阶段1：逐帧目录遍历 (SearchDirectroy)

### 算法思想

使用**队列式广度优先遍历**替代一次性递归。每帧从队列头部取一个目录，将其子目录追加到队列尾部，将文件追加到筛选文件队列。

### 核心代码

```cpp
void FXGSSPCountCode::SearchDirectroy()
{
    if (SearchedDirectories.Num() == 0)
    {
        // 当前阶段完成 → 切换到阶段2
        CountCodeStatus = ECountCodeStatus::SearchFile;
        SearchDirectoryTime = FDateTime::Now();
        return;  // 本帧不再处理，下帧进入 SearchFile
    }

    // 从队列头部取一个目录
    FString CurrentDir = SearchedDirectories[0];
    SearchedDirectories.RemoveAt(0);

    // 找子目录 → 追加到队列尾部
    TArray<FString> SubDirs;
    IFileManager::Get().FindFiles(SubDirs, *(CurrentDir / TEXT("*")), false, true);
    for (const FString& SubDir : SubDirs)
    {
        SearchedDirectories.Insert(CurrentDir / SubDir, 0);  // 头部插入=深度优先
        // 注：字幕中使用 Insert(0) 是为了让日志输出更直观（按层级顺序）
    }

    // 找文件 → 添加到待筛选队列
    TArray<FString> Files;
    IFileManager::Get().FindFiles(Files, *(CurrentDir / TEXT("*")), true, false);
    for (const FString& File : Files)
    {
        SearchedFiles.Add(CurrentDir / File);
    }

    PrintToDisplay(FString::Printf(TEXT("Dir: %s | +%d dirs | +%d files"),
        *CurrentDir, SubDirs.Num(), Files.Num()));
}
```

**关键设计**：
- 每 Tick 只处理**一个目录**（拿出队首、处理、追加）
- 不能一口气遍历完 → 否则大工程（千万行）会卡死

## 阶段2：逐帧文件筛选 (SearchFile)

确认 `SearchedFiles` 队列为空后进入阶段3。不为空时每帧取一个文件做筛选检查：

```cpp
void FXGSSPCountCode::SearchFile()
{
    if (SearchedFiles.Num() == 0)
    {
        CountCodeStatus = ECountCodeStatus::CountCode;
        SearchFileTime = FDateTime::Now();
        return;
    }

    FString FilePath = SearchedFiles[0];
    SearchedFiles.RemoveAt(0);

    // 文件存在性检查（可能在遍历过程中被删了）
    if (!CheckFilePath(FilePath))
        return;

    // 后缀筛选 + 路径排除
    FString Extension = FPaths::GetExtension(FilePath);
    if (Extension == TEXT("h") || Extension == TEXT("cpp"))
    {
        if (!FilePath.Contains(TEXT("/Intermediate/")) &&
            !FilePath.Contains(TEXT("/Binaries/")) &&
            !FilePath.Contains(TEXT("/Build/")))
        {
            CountedFiles.Add(FilePath);  // 通过筛选，加入计数队列
            PrintToDisplay(FString::Printf(TEXT("Add to count: %s"), *FilePath));
            return;
        }
    }
    PrintToDisplay(FString::Printf(TEXT("Discard: %s"), *FilePath));
}
```

**筛选规则**（硬编码）：
1. 后缀必须为 `.h` 或 `.cpp`
2. 路径中不能包含 `Intermediate`、`Binaries`、`Build`

**文件存在性检查** (`CheckFilePath`)：在遍历过程中文件可能已被删除，每次都需检查 `FPaths::FileExists()`。

## 阶段3：逐帧代码行数统计 (CountCode)

```cpp
void FXGSSPCountCode::CountCode()
{
    if (CountedFiles.Num() == 0)
    {
        CountCodeStatus = ECountCodeStatus::Finish;
        CountedCodeTime = FDateTime::Now();
        return;
    }

    FString FilePath = CountedFiles[0];
    CountedFiles.RemoveAt(0);

    if (!CheckFilePath(FilePath))
        return;  // 文件已丢失

    // 读取文件全部行 → 计数
    TArray<FString> Lines;
    FFileHelper::LoadFileToStringArray(Lines, *FilePath);

    CodeNum += Lines.Num();

    PrintToDisplay(FString::Printf(TEXT("%s: %d lines | Total: %d"),
        *FilePath, Lines.Num(), CodeNum));
}
```

使用 `FFileHelper::LoadFileToStringArray` 读取文件为 `TArray<FString>`，每项是一行。

## 阶段4：汇总完成 (Finish)

```cpp
void FXGSSPCountCode::FinishCountCode()
{
    FinishTime = FDateTime::Now();

    // 计算各阶段耗时
    double TotalTime = (FinishTime - StartTime).GetTotalSeconds();
    double DirTime = (SearchDirectoryTime - StartTime).GetTotalSeconds();
    double FileTime = (SearchFileTime - SearchDirectoryTime).GetTotalSeconds();
    double CodeTime = (CountedCodeTime - SearchFileTime).GetTotalSeconds();

    PrintToDisplay(FString::Printf(TEXT("=== SUMMARY ===")));
    PrintToDisplay(FString::Printf(TEXT("Total Code Lines: %d"), CodeNum));
    PrintToDisplay(FString::Printf(TEXT("Total: %.3fs | Dir: %.3fs | File: %.3fs | Code: %.3fs"),
        TotalTime, DirTime, FileTime, CodeTime));

    // 重置状态，等待下一次激活
    CodeNum = 0;
    CountCodeStatus = ECountCodeStatus::Idle;
}
```

## 分帧思想总结

| 维度 | 说明 |
|------|------|
| 核心原则 | 大任务 → 拆成小任务 → 每帧处理一个小任务 |
| 实现方式 | 状态机 + 队列 + 每 Tick 移除队首并处理 |
| 效果 | UI 保持响应（帧率 100fps），不会因大任务卡死 |
| 可扩展性 | 可进一步加入多线程（参考第17章），将任务分发到多个线程并行 |
| 类比 | 暴力破解压缩包：把所有密钥组合拆成子任务，多线程并发测试 |
