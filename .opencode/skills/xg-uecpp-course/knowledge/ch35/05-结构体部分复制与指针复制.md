# 结构体部分复制与指针复制

## 结构体部分复制

结构体整体参与网络复制时，可通过 `UPROPERTY(NotReplicated)` 标记字段，使其不参与同步。

代码位置：[026_Net/XGNetNotRep.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Source/XGSampleDemo/026_Net/XGNetNotRep.h)

### 结构体定义

```cpp
USTRUCT(BlueprintType)
struct FXGNetNotRepStruct
{
    GENERATED_BODY()

    UPROPERTY()
    int32 RepInt;           // 参与同步

    UPROPERTY(NotReplicated)
    int32 NotRepInt;        // 不参与同步
};
```

### Actor 使用

```cpp
UPROPERTY(ReplicatedUsing = OnRep_NetNotRepStruct)
FXGNetNotRepStruct NetNotRepStruct;
```

在服务器 Tick 中同时自增两个字段，客户端可通过打印对比观察差异：
- `RepInt`：在客户端与服务器保持一致
- `NotRepInt`：只在服务器自增，客户端保持初始值

### 使用场景

- 结构体中包含客户端计算的临时数据
- 结构体中包含只在服务器使用的统计信息
- 避免不必要的网络传输

## 指针/对象引用复制

### 自动 GUID 映射

UE 对复制的 UObject 属性使用 `FNetworkGUID` 进行跨机器映射：服务器分配 GUID，通知所有已连接的客户端，各终端通过 GUID 找到关联的对象实例。

```cpp
UPROPERTY(Replicated)
AActor* TargetActor;
```

### 复制引用的前提

| 对象类型 | 条件 |
|---------|------|
| 已复制的 Actor | 自动通过 GUID 映射 |
| 未复制的 Actor | 必须有可靠命名（Stable Name）——从数据包直接加载，如地图中直接摆放的对象 |
| 已复制的 Component | 自动通过 GUID 映射 |
| 未复制的 Component | 必须有可靠命名 |
| 其他 UObject（非 Actor/组件） | 必须由加载的数据包直接提供 |

### 可靠命名

在服务器和客户端上存在同名对象，即直接从数据包加载（非游戏期间动态生成）。例如直接摆放在地图中的 Actor，在世界大纲中有唯一名称，可通过名字在不同终端间关联。

动态生成的对象不能用于可靠命名——不同终端随着运行时间增加，命名序列差异会越来越大。
