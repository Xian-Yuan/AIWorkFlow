# 条件复制与所有权控制

## 概述

条件复制（Conditional Replication）控制特定属性只复制给满足条件的客户端，用于带宽优化。最常见的场景是"只复制给所属客户端"（COND_OwnerOnly）。

## COND_OwnerOnly

只有拥有该 Actor 所有权的客户端才能收到属性更新。

代码位置：[026_Net/XGNetDerivedConditionRepUsing.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Source/XGSampleDemo/026_Net/XGNetDerivedConditionRepUsing.h)

### 声明与注册

```cpp
UPROPERTY(ReplicatedUsing = OnRep_OwnerOnlyValue)
int32 OwnerOnlyValue;

void GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    DOREPLIFETIME_CONDITION(AXGNetDerivedConditionRepUsing, OwnerOnlyValue, COND_OwnerOnly);
}
```

### 所有权设置

所有权必须在服务器上设置：

```cpp
// 在服务器上，将 Actor 所有权交给指定玩家的 PlayerController
SetOwner(PlayerControllerRef);
```

客户端通过 `GetOwner()` 检查自己是否为 Owner，只有 Owner 能收到 `COND_OwnerOnly` 属性的更新。

客户端只能获取自己的 PlayerController：

```cpp
// 客户端只能获取自己的控制器
APlayerController* MyPC = GetWorld()->GetFirstPlayerController();
// GetPlayerController(1) 在客户端无效
```

## DOREPLIFETIME_CONDITION_NOTIFY

同时控制复制条件和 OnRep 通知行为：

```cpp
DOREPLIFETIME_CONDITION_NOTIFY(Class, Property, COND_OwnerOnly, REPNOTIFY_Always);
```

参数：
- 条件：`COND_OwnerOnly` / `COND_None` 等
- 通知模式：`REPNOTIFY_Always` / `REPNOTIFY_OnChange`

## DOREPLIFETIME_ACTIVE_OVERRIDE 与 PreReplication

在 `PreReplication()` 中动态控制属性是否参与同步，比条件复制更灵活。

代码位置：[026_Net/XGNetDerivedConditionRepUsingAN.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Source/XGSampleDemo/026_Net/XGNetDerivedConditionRepUsingAN.h)

```cpp
void AXGNetDerivedConditionRepUsingAN::PreReplication(IRepChangedPropertyTracker& ChangedPropertyTracker)
{
    Super::PreReplication(ChangedPropertyTracker);
    DOREPLIFETIME_ACTIVE_OVERRIDE(
        AXGNetDerivedConditionRepUsingAN,
        DynamicValue,
        bEnableDynamicReplication
    );
}
```

`DOREPLIFETIME_ACTIVE_OVERRIDE` 需要包含 `Net/UnrealNetwork.h`，并在模块 Build.cs 中添加 `"NetCore"` 依赖。

## 完整的条件复制工具

| 宏 | 功能 | 使用场景 |
|----|------|---------|
| `DOREPLIFETIME` | 默认复制 | 基础属性同步 |
| `DOREPLIFETIME_CONDITION` | 带条件复制 | OwnerOnly、模拟代理等 |
| `DOREPLIFETIME_CONDITION_NOTIFY` | 条件复制 + 通知模式 | 需要指定 REPNOTIFY_Always 时 |
| `DOREPLIFETIME_ACTIVE_OVERRIDE` | 动态启用/禁用复制 | PreReplication 中运行时控制 |

## 自定义复制条件

通过重写 `IsNetRelevantFor()` 可以实现更复杂的自定义条件，但相比内置条件（如 COND_OwnerOnly）开销更大。建议优先使用内置条件宏。
