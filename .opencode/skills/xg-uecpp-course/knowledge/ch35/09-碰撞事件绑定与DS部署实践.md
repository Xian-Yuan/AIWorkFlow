# 碰撞事件绑定与 DS 部署实践

## 碰撞事件绑定时机

### 问题：不要在 Constructor 中绑定 OnHit

在 Constructor 中绑定碰撞事件会导致不可预知的行为：

```cpp
// 错误：在 Constructor 中绑定
AXGMyActor::AXGMyActor()
{
    // 此时网络角色（LocalRole）尚未设置
    // CDO（Class Default Object）构造时处于无网络状态
    OnActorHit.AddDynamic(this, &AXGMyActor::OnHit);
}
```

### 正确做法：在 BeginPlay 中按角色绑定

```cpp
void AXGMyActor::BeginPlay()
{
    Super::BeginPlay();
    
    if (HasAuthority())
    {
        // 仅在服务器上绑定
        OnActorHit.AddDynamic(this, &AXGMyActor::OnHit);
    }
}
```

### 原因

- Constructor 阶段：Actor 正在构造 CDO，网络角色未分配
- CDO 的网络角色始终为 `ROLE_None`
- `HasAuthority()` 在 Constructor 中返回不可靠结果
- 服务端和客户端共用同一个 Constructor，不区分网络角色

### 客户端与服务端碰撞的区别

- **服务器**：真实碰撞检测，产生伤害/击退等游戏效果
- **客户端**：仅播放视觉效果（粒子、音效），不应产生逻辑效果
- 碰撞回调中通过 `HasAuthority()` 区分执行逻辑

### Target.cs 配置

DS 打包需要配置多个 Target 文件：

| Target 文件 | 用途 |
|------------|------|
| `XGNetDemoForDSEditor.Target.cs` | 编辑器 Target |
| `XGNetDemoForDSGame.Target.cs` | 游戏客户端 Target |
| `XGNetDemoForDSClient.Target.cs` | 纯客户端 Target（不含服务器逻辑） |
| `XGNetDemoForDSServer.Target.cs` | 专用服务器 Target |

### Server Target 配置示例

```cpp
public class XGNetDemoForDSServerTarget : TargetRules
{
    public XGNetDemoForDSServerTarget(TargetInfo Target) : base(Target)
    {
        Type = TargetType.Server;
        DefaultBuildSettings = BuildSettingsVersion.V2;
        ExtraModuleNames.AddRange(new string[] { "XGNetDemoForDS" });
    }
}
```

### 部署流程

1. 打包 Server Target → 生成 DedicatedServer 可执行文件
2. 打包 Client Target → 生成客户端可执行文件
3. 服务器启动 DS 进程（无需渲染窗口）
4. 客户端通过 IP 直连或服务器列表连接

### DS 与 Listen Server 的区别

| 维度 | Dedicated Server | Listen Server |
|------|-----------------|---------------|
| 渲染画面 | 不渲染（无窗口） | 渲染（主机玩家） |
| 资源消耗 | 较低 | 较高（需同时渲染和运算） |
| 主机玩家 | 无主机玩家 | 有主机玩家（也是玩家） |
| 使用场景 | 正式部署 | 小型合作/Custom Match |
