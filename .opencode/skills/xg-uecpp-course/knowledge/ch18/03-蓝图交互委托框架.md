# 蓝图交互委托框架

## 概述

Subsystem 通过两组动态多播委托暴露给蓝图，用于异步初始化流程的进度通知和结果回调。同时，C++ 内部使用原生多播委托管理并行子任务的结果汇总。

## 动态多播委托声明

### 进度通知委托

```cpp
DECLARE_DYNAMIC_MULTICAST_DELEGATE_TwoParams(FXGControlFlowsInitProgress, FGuid, InitAsyncID, float, ProgressValue);
```

- **InitAsyncID（FGuid）**：本次初始化会话的唯一标识，用于区分多次初始化调用
- **ProgressValue（float）**：当前进度值（0~1），由 QueueStep 的第四个参数传入

### 结果通知委托

```cpp
DECLARE_DYNAMIC_MULTICAST_DELEGATE_ThreeParams(FXGControlFlowsInitResult, FGuid, InitAsyncID, bool, bResult, FString, Message);
```

- **InitAsyncID（FGuid）**：本次初始化会话的唯一标识
- **bResult（bool）**：成功/失败
- **Message（FString）**：结果描述（如 "Init Success" 或错误信息）

### 蓝图绑定属性

```cpp
UPROPERTY(BlueprintAssignable, Category = "XGControlFlows")
FXGControlFlowsInitProgress InitProgress;

UPROPERTY(BlueprintAssignable, Category = "XGControlFlows")
FXGControlFlowsInitResult InitResult;
```

`BlueprintAssignable` 标记使蓝图可以在 EventGraph 中直接绑定事件。

## FGuid 实例标识

每次调用 `InitLevel()` 生成唯一 FGuid：

```cpp
void UXGControlFlowsSubsystem::InitLevel()
{
    InitAsyncID = FGuid::NewGuid();
    bIniting = true;
    // ...
}
```

FGuid 的作用：
- 区分多次初始化调用，蓝图可通过对比 InitAsyncID 判断回调是否属于当前会话
- 作为 TMap 中 ManageTask 的键
- 在日志中追踪异步调用链路

## 委托广播实现

### 进度广播

```cpp
void UXGControlFlowsSubsystem::CallInitProgress(FGuid InInitAsyncID, float ProgressValue)
{
    FXGControlFlowsInitProgress TempDelegate = InitProgress;

    AsyncTask(ENamedThreads::GameThread, [TempDelegate, InInitAsyncID, ProgressValue]()
    {
        TempDelegate.Broadcast(InInitAsyncID, ProgressValue);
    });
}
```

关键设计：
- **复制委托对象**：先取副本 `TempDelegate = InitProgress`，避免在异步 Lambda 中直接引用 UObject 属性
- **AsyncTask 推回 GameThread**：确保 Broadcast 在 GameThread 执行（动态多播委托必须在 GameThread 广播）
- **Lambda 按值捕获**：`TempDelegate`、`InInitAsyncID`、`ProgressValue` 全部按值捕获，避免悬挂引用

### 结果广播

```cpp
void UXGControlFlowsSubsystem::CallInitResult(FGuid InInitAsyncID, bool bResult, FString Message)
{
    FXGControlFlowsInitResult TempDelegate = InitResult;

    AsyncTask(ENamedThreads::GameThread, [TempDelegate, InInitAsyncID, bResult, Message]()
    {
        TempDelegate.Broadcast(InInitAsyncID, bResult, Message);
    });
}
```

## 原生多播委托

C++ 内部使用的原生多播委托（非动态，不暴露给蓝图）：

```cpp
DECLARE_MULTICAST_DELEGATE_TwoParams(FXGControlFlowsManageTaskResponseResult, bool, FString)
```

用在 `FXGControlFlowsManageTask` 结构体中，当管理任务的所有子任务完成时触发。通过 `AddLambda` 绑定回调：

```cpp
XGManageTaskPtr->ManageTaskResponse.AddLambda([SubFlow, InProgreesValue, this](bool bResult, FString Message)
{
    if (bResult)
    {
        SubFlow->ContinueFlow();
    }
    else
    {
        SubFlow->CancelFlow();
    }
});
```

## 广播时机

在 5 步骤 Flow 中，委托广播发生在以下时机：

| 步骤 | 广播 | 说明 |
|------|------|------|
| InitLocalAsset | InitProgress | 后台耗时完成后回主线程广播进度 |
| InitNetInfo | InitProgress（成功）/ InitResult（失败） | ManageTask 全部完成后广播 |
| InitUserInfo | InitProgress | 文件持久化完成后广播 |
| NotifyMainUI | InitResult（失败模式） | 宏控制广播失败结果 |
| FinishThisInit | InitProgress + InitResult | 最终广播成功结果 |
