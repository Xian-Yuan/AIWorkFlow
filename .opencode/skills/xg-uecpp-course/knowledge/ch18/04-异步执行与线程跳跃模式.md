# 异步执行与线程跳跃模式

## 概述

本章的异步执行采用 **线程跳跃模式**：后台线程做耗时操作（睡眠、文件写入），然后跳回主线程推进 Flow 并广播进度。这种模式确保耗时操作不阻塞 GameThread，同时 UObject 操作和委托广播始终在主线程进行。

## AsyncTask 线程跳跃

### 基本模式

```cpp
AsyncTask(ENamedThreads::AnyThread, [this, SubFlow, InProgreesValue]()
{
    // AnyThread：后台线程执行耗时操作
    FPlatformProcess::Sleep(0.2);

    AsyncTask(ENamedThreads::GameThread, [this, SubFlow, InProgreesValue]()
    {
        // GameThread：主线程广播进度并推进流程
        InitProgress.Broadcast(InitAsyncID, InProgreesValue);
        SubFlow->ContinueFlow();
    });
});
```

**执行流**：

```
GameThread:  InitLocalAsset 调用 → (立即返回)
                          ↓ AsyncTask(AnyThread)
AnyThread:   Sleep(0.2) → 后台等待
                          ↓ AsyncTask(GameThread)
GameThread:  Broadcast 进度 → ContinueFlow() → 下一步
```

### InitLocalAsset 实现

```cpp
void UXGControlFlowsSubsystem::InitLocalAsset(FControlFlowNodeRef SubFlow, double InProgreesValue)
{
    AsyncTask(ENamedThreads::AnyThread, [this, SubFlow, InProgreesValue]()
    {
        FPlatformProcess::Sleep(0.2);

        AsyncTask(ENamedThreads::GameThread, [this, SubFlow, InProgreesValue]()
        {
            InitProgress.Broadcast(InitAsyncID, InProgreesValue);
            SubFlow->ContinueFlow();
        });
    });
}
```

## 文件持久化（InitUserInfo）

InitUserInfo 在后台线程执行文件写入，演示真实异步 I/O 场景：

```cpp
void UXGControlFlowsSubsystem::InitUserInfo(FControlFlowNodeRef SubFlow, double InProgreesValue)
{
    AsyncTask(ENamedThreads::AnyThread, [this, SubFlow, InProgreesValue]()
    {
        // 后台：构建路径并写入文件
        FString UserInfoPath = FPaths::ConvertRelativePathToFull(
            FPaths::ProjectSavedDir() / TEXT("XGControlFlowsUserInfo.txt"));

        FString UserInitAsyncID = InitAsyncID.ToString();
        FString UserInitDataTime = FDateTime::Now().ToHttpDate();

        TArray<FString> MyStringInfo;
        MyStringInfo.Add(UserInitAsyncID);
        MyStringInfo.Add(UserInitDataTime);

        FFileHelper::SaveStringArrayToFile(MyStringInfo, *UserInfoPath,
            FFileHelper::EEncodingOptions::ForceUTF8WithoutBOM);

        // 回主线程：广播进度并推进
        AsyncTask(ENamedThreads::GameThread, [this, SubFlow, InProgreesValue]()
        {
            InitProgress.Broadcast(InitAsyncID, InProgreesValue);
            SubFlow->ContinueFlow();
        });
    });
}
```

关键 API：

| API | 说明 |
|-----|------|
| `FPaths::ProjectSavedDir()` | 获取项目 Saved 目录路径 |
| `FPaths::ConvertRelativePathToFull()` | 相对路径转绝对路径 |
| `FFileHelper::SaveStringArrayToFile()` | 将字符串数组写入文件，`ForceUTF8WithoutBOM` 强制 UTF-8 编码 |
| `FDateTime::Now().ToHttpDate()` | 获取当前时间的 HTTP 日期格式字符串 |

## 同步执行模式

某些步骤不需要异步，直接在 GameThread 执行：

```cpp
void UXGControlFlowsSubsystem::NotifyMainUI(FControlFlowNodeRef SubFlow, double InProgreesValue)
{
    if (XGCONCTROLRESULT)
    {
        InitProgress.Broadcast(InitAsyncID, InProgreesValue);
        SubFlow->ContinueFlow();
    }
    else
    {
        // 失败模式：广播结果后取消流程
        AsyncTask(ENamedThreads::GameThread, [this]() {
            InitResult.Broadcast(InitAsyncID, false, TEXT("测试失败"));
        });
        SubFlow->CancelFlow();
        bIniting = false;
        InitAsyncID = {};
    }
}
```

## XGCONCTROLRESULT 宏

```cpp
#define XGCONCTROLRESULT 0
```

- **0**：模拟失败模式 — InitNetInfo 的第 7 个子任务（i==6）会失败，NotifyMainUI 直接广播失败
- **1**：正常模式 — 所有步骤成功执行，最终广播初始化成功

这是 Epic 风格的开发期调试手段，通过宏开关在编译期切换运行路径，无需修改代码逻辑。

## 三种异步执行模型对比

| 模型 | 步骤时序 | 执行线程 | 适用场景 |
|------|----------|----------|----------|
| **同步执行** | 所有步骤在同一帧依次执行 | GameThread | 轻量计算、状态更新 |
| **逐步骤异步** | 每步骤独立后台执行，完成回 GameThread | AnyThread → GameThread | 网络请求、文件 I/O、独立计算 |
| **ManageTask 并行** | N 个子任务并行在后台执行，Tick 轮询完成状态 | AnyThread → GameThread（汇总） | 大量独立子任务、批量检测 |
