# 多线程详解

## 概述

Unreal Engine 的多线程体系以 TaskGraph 线程池为核心，提供 FRunnable（独立线程）、AsyncTask/Async（轻量异步）、FGraphEvent（任务依赖 DAG）和 ParallelFor（并行循环）四种方案。核心原则：长时间任务用 FRunnable 独立线程，短任务用 AsyncTask，依赖编排用 FGraphEvent，数据并行用 ParallelFor。

## FRunnable 独立线程

适合长期后台任务（如 TCP 通信、音频消费），创建完全独立于 TaskGraph 的线程。

### 接口

```cpp
class FXGXXXRunnable : public FRunnable
{
    virtual bool Init() override;
    virtual uint32 Run() override;
    virtual void Stop() override;
    virtual void Exit() override;
};
```

### 启动与停止

```cpp
// 启动
FXGXXXRunnable* Runnable = new FXGXXXRunnable();
FRunnableThread* Thread = FRunnableThread::Create(Runnable, TEXT("ThreadName"));

// 停止
if (Thread)
{
    Thread->Kill(true);
    delete Thread;
    Thread = nullptr;
}
```

### 线程安全标志

| 方案 | 适用场景 |
|------|---------|
| `FThreadSafeBool` | 简单布尔标志（推荐，开销最小） |
| `FCriticalSection + FScopeLock` | 保护共享数据结构（TArray、TMap） |
| `std::atomic<bool>` | 同为轻量标志，但 UE 推荐 FThreadSafeBool |
| `IsInGameThread()` | 运行时断言线程身份 |

```cpp
FThreadSafeBool bStopThread = false;

// 线程内检查
uint32 Run() override
{
    while (!bStopThread)
    {
        // 工作
    }
    return 0;
}
```

```cpp
FCriticalSection VoiceDataSection;
TArray<float> VoiceData;

// 写
{
    FScopeLock Lock(&VoiceDataSection);
    VoiceData.Append(InData);
}

// 读
{
    FScopeLock Lock(&VoiceDataSection);
    OutData = VoiceData;
}
```

## AsyncTask 轻量异步

在 TaskGraph 线程池中执行短任务，通过 ENamedThreads 实现线程跳跃。

```cpp
// 后台执行
AsyncTask(ENamedThreads::AnyThread, [this]()
{
    // 耗时操作，不可访问 UObject
});

// 结果回传主线程
AsyncTask(ENamedThreads::AnyThread, [this]()
{
    // 后台工作
    TArray<uint8> Result = DoWork();

    AsyncTask(ENamedThreads::GameThread, [this, Result]()
    {
        // 回到游戏线程，可访问 UObject
        OnComplete.Broadcast(Result);
    });
});
```

注意：AsyncTask 使用 TaskGraph 线程池，长时间任务会阻塞引擎。

## Async + TFuture

需要返回值的异步操作，支持批量等待。

```cpp
// 发起异步任务
TFuture<bool> Future = Async(EAsyncExecution::Thread, [this]()
{
    return DoHeavyWork();
});

// 等待结果（阻塞当前线程）
bool bResult = Future.Get();

// 批量等待
TArray<TFuture<bool>> Futures;
for (int32 i = 0; i < 10; i++)
    Futures.Add(Async(EAsyncExecution::Thread, [i]() { return Work(i); }));

for (auto& Future : Futures)
    Future.Wait();
```

## FGraphEvent 任务依赖

构建任务 DAG，支持流水线编排。

```cpp
// 单个先决条件
FGraphEventRef PreWork = FFunctionGraphTask::CreateAndDispatchWhenReady([]
{
    return DoPreWork();
});

FGraphEventRef MainWork = FFunctionGraphTask::CreateAndDispatchWhenReady([]
{
    return DoMainWork();
}, TStatId(), PreWork);

// 多个先决条件
FGraphEventArray Prerequisites;
Prerequisites.Add(TaskA);
Prerequisites.Add(TaskB);

FFunctionGraphTask::CreateAndDispatchWhenReady([]
{
    // TaskA 和 TaskB 都完成后执行
}, TStatId(), &Prerequisites);

// 批量等待
FGraphEventArray Tasks;
for (int32 i = 0; i < 10; i++)
    Tasks.Add(FFunctionGraphTask::CreateAndDispatchWhenReady([i]() { return Work(i); }));
FTaskGraphInterface::Get().WaitUntilTasksComplete(Tasks);
```

## ParallelFor 数据并行

用于数组元素的并行处理，等同 UE 版 OpenMP。

```cpp
// 基本用法
ParallelFor(Array.Num(), [&](int32 Index)
{
    ProcessElement(Array[Index]);
});

// 标志控制
ParallelFor(Array.Num(), [&](int32 Index)
{
    // 业务逻辑
}, EParallelForFlags::Unbalanced);

// 共享变量保护
int32 Total = 0;
FCriticalSection CS;

ParallelFor(Array.Num(), [&](int32 Index)
{
    FScopeLock Lock(&CS);
    Total += Array[Index].GetValue();
});
```

## TaskGraph 阻塞陷阱

共享 TaskGraph 线程池的阻塞模式会导致引擎卡死。

| 危险模式 | 原因 | 解决方案 |
|---------|------|---------|
| FRunnable 内 Sleep 填充 AsyncTask | 工作线程占着 TaskGraph 槽位 | 用 FRunnable 独立线程 |
| ParallelFor 内嵌 Sleep 等待 | 所有工作线程被占满 | 移除 Sleep，用 ForceSingleThread 调试 |
| AsyncTask 内 Wait 等待另一个 AsyncTask | 线程池死锁 | 用 FGraphEvent 依赖编排 |

本质规律：TaskGraph 工作线程阻塞等待另一个 TaskGraph 任务时形成死锁。
