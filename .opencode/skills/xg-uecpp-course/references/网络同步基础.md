# 网络同步基础

## 概述

UE 原生网络框架以**服务器权威（Server Authority）** 为核心：服务器持有真实状态，客户端反映近似状态。同步分为**属性同步（Replication）** 和**远程过程调用（RPC）** 两条路径。数据走属性同步，行为走 RPC。

## NetMode 与网络架构

```cpp
switch (GetWorld()->GetNetMode())
{
    case NM_Standalone:       // 单人模式
    case NM_DedicatedServer:  // 纯服务器（无本地客户端）
    case NM_ListenServer:     // 服务器 + 本地客户端
    case NM_Client:           // 远程客户端
}
```

## Actor 网络角色体系

每个复制的 Actor 在不同终端上扮演不同角色：

| 角色 | 说明 |
|------|------|
| `ROLE_Authority` | 服务器——拥有完整权限，属性变化自动同步 |
| `ROLE_AutonomousProxy` | 所属客户端——拥有输入权限，可发起 Server RPC |
| `ROLE_SimulatedProxy` | 非所属客户端——只能模拟（位置插值、动画播放） |

```cpp
// Actor 启用复制
bReplicates = true;

// 判断当前角色
if (GetLocalRole() == ROLE_Authority) { /* 服务器逻辑 */ }
if (GetRemoteRole() == ROLE_AutonomousProxy) { /* 对方是拥有客户端的模拟 */ }
```

### 相关性与所有权

| 概念 | 说明 |
|------|------|
| **相关性（Relevancy）** | Actor 是否对某客户端可见（距离、频道等） |
| **所有权（Ownership）** | 哪个客户端拥有此 Actor（通过 `SetOwner(PC)` 在服务器设置） |

## 属性同步

### 注册模式

```cpp
void AXGActor::GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);
    DOREPLIFETIME(AActor, Health);      // 默认复制给所有客户端
    DOREPLIFETIME_CONDITION(AActor, Ammo, COND_OwnerOnly);  // 仅复制给所属客户端
    DOREPLIFETIME_ACTIVE_OVERRIDE(AActor, DynamicVar, bEnabled);  // PreReplication 动态控制
}
```

### 声明与 OnRep

```cpp
UPROPERTY(ReplicatedUsing = OnRep_Health)
float Health;

// 属性变化后自动调用
UFUNCTION()
void OnRep_Health(float OldHealth);
```

### 条件复制

| 条件宏 | 行为 |
|--------|------|
| `COND_None` | 默认，复制给所有连接 |
| `COND_OwnerOnly` | 仅复制给所属客户端 |
| `COND_SkipOwner` | 跳过所属客户端 |
| `COND_SimulatedOnly` | 仅复制给模拟代理 |
| `COND_AutonomousOnly` | 仅复制给自主代理 |
| `COND_InitialOnly` | 仅初始复制一次 |

### 结构体部分复制

通过 `UPROPERTY(NotReplicated)` 标记结构体中不参与同步的字段：

```cpp
USTRUCT()
struct FXGData
{
    UPROPERTY() int32 ServerValue;        // 同步
    UPROPERTY(NotReplicated) int32 LocalCache;  // 不同步
};
```

## RPC 三种调用模式

| RPC 类型 | 调用位置 | 执行位置 | 前置条件 |
|---------|---------|---------|---------|
| **Server** | 拥有 Actor 的客户端 | 服务器 | Actor 归调用客户端所有 |
| **Client** | 服务器 | 指定客户端 | Actor 的 Owner 必须为目标客户端 PlayerController |
| **Multicast** | 服务器 | 所有终端 | 无特殊所有权要求 |

```cpp
UFUNCTION(Server, Reliable, WithValidation)
void Server_AddHealth(int32 Amount);

UFUNCTION(Client, Reliable)
void Client_OnDamageTaken(float NewHealth);

UFUNCTION(NetMulticast, Reliable)
void Multicast_PlayHitEffect();
```

### RPC 验证（WithValidation）

```cpp
UFUNCTION(Server, Reliable, WithValidation)
void Server_AddMoney(int32 InAddMoney);

bool Server_AddMoney_Validate(int32 InAddMoney)
{
    return InAddMoney > 5;  // 验证失败 → 调用丢弃，连接断开
}

void Server_AddMoney_Implementation(int32 InAddMoney)
{
    Money += InAddMoney;
}
```

验证失败时：RPC 被丢弃，客户端连接自动断开。

## 网络休眠（Dormancy）

优化高频更新 Actor 的带宽消耗：

```cpp
// 设置休眠
SetNetDormancy(DORM_DormantAll);

// 唤醒条件：属性变化 / ForceNetUpdate / Relevancy 变化
```

| 状态 | 说明 |
|------|------|
| `DORM_Never` | 从不休眠（默认） |
| `DORM_DormantAll` | 对所有连接休眠 |
| `DORM_Awake` | 唤醒 |
| `DORM_Initial` | 复制一次后自动休眠 |

## DSM 分布式部署

DSM（Dedicated Server Manager）三层架构：

```
DSM (011_XGMultiManage)          ← WebSocket Server，管理 DS 实例
    ├── DS (010_XGMultiGame)     ← 实际游戏服务器，向 DSM 注册
    └── Player (010_XGMultiGame) ← 客户端，从 DSM 获取 DS 列表
```

// DSM 动态拉起新 DS 进程
