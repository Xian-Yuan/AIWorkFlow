# 智能指针详解

## 概述

UE 提供四类智能指针：TSharedPtr（引用计数的共享所有权）、TSharedRef（非空共享引用）、TWeakPtr（弱引用，解决循环引用）、TUniquePtr（独占所有权）。它们在 UE 内存模型（UObject GC 管理）之外为纯 C++ 对象提供自动生命周期管理。

## 四种类型对比

| 类型 | 所有权 | 空值 | 转换 | 开销 |
|------|--------|------|------|------|
| **TSharedPtr** | 共享 | 允许为空 | 可转为 TWeakPtr、TSharedRef（需确保非空） | 引用计数 |
| **TSharedRef** | 共享 | 不允许为空 | 可隐式转为 TSharedPtr | 引用计数 |
| **TWeakPtr** | 不持有 | 允许为空 | 通过 Pin() 获取 TSharedPtr | 无 |
| **TUniquePtr** | 独占 | 允许为空 | Move 语义转让所有权 | 无 |

## TSharedPtr

```cpp
// 创建
TSharedPtr<FMyObject> Ptr = MakeShareable(new FMyObject());

// 访问
if (Ptr.IsValid()) { Ptr->DoSomething(); }

// 引用计数
uint32 RefCount = Ptr.GetSharedReferenceCount();

// 判等
if (Ptr == AnotherPtr) { }
```

## TSharedRef

```cpp
// 创建——必须保证非空
TSharedRef<FMyObject> Ref = MakeShareable(new FMyObject());

// 隐式转为 TSharedPtr
TSharedPtr<FMyObject> Ptr = Ref;

// 反向转换（不安全——需确认非空）
TSharedRef<FMyObject> FromPtr = Ptr.ToSharedRef();
```

## TWeakPtr 与循环引用

### 问题场景

```cpp
class FParent
{
    TSharedPtr<FChild> Child;
};

class FChild
{
    TSharedPtr<FParent> Parent;  // ← 此处用 TSharedPtr 会导致循环引用
};

// Parent 和 Child 互相引用，引用计数永远不为 0，内存泄漏
```

### 修复

```cpp
class FChild
{
    TWeakPtr<FParent> ParentWeak;  // TWeakPtr 不增加引用计数
};

// 使用时 Pin()
if (TSharedPtr<FParent> Parent = ParentWeak.Pin())
{
    Parent->DoSomething();
}
```

### Pin() 语义

- 返回 TSharedPtr，如果对象已被销毁则返回空
- 使用时必须检查有效性
- 不使用时 TWeakPtr 不阻止对象销毁

## TUniquePtr

```cpp
// 创建——Move 语义
TUniquePtr<FMyObject> Unique = MakeUnique<FMyObject>();

// 转移所有权
TUniquePtr<FMyObject> Other = MoveTemp(Unique);  // Unique 变为空

// 使用
if (Unique) { Unique->DoSomething(); }

// 释放
Unique.Reset();  // 清空并销毁对象
Unique = nullptr; // 同上
```

## TSharedFromThis

当对象需要在成员函数中安全地获取指向自身的 TSharedPtr：

```cpp
class FMyNode : public TSharedFromThis<FMyNode>
{
    void DoSomething()
    {
        TSharedPtr<FMyNode> Self = AsShared();  // 安全获取自身 SharedPtr
        RegisterCallback(Self);
    }
};
```

## 与 STL 智能指针对比

| UE | C++ STL | 差异 |
|----|---------|------|
| TSharedPtr | shared_ptr | UE 版本预分配控制块，性能略优 |
| TUniquePtr | unique_ptr | 接口兼容（MakeUnique / MoveTemp） |
| TWeakPtr | weak_ptr | UE 版通过 Pin() 获取 shared_ptr，STL 版通过 lock() |
| TSharedRef | 无直接等价 | STL 无"非空保证"的引用语义 |

## 使用建议

- 默认使用 TObjectPtr 管理 UObject（GC 管理），智能指针只用于非 UObject
- TSharedRef 作为函数参数时表明"此参数必须有效"
- TWeakPtr 是破除循环引用的首选方案，优先于原始指针
- TUniquePtr 适合工厂模式中的独占资源
