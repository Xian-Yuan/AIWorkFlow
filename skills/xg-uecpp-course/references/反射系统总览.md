# 反射系统总览

## 六大宏选择树

```
需要新类型？
├── 需要继承 UObject（GC 管理）→ UCLASS
├── 需要值类型结构体（无多态）→ USTRUCT
├── 需要枚举定义 → UENUM（必须指定 :uint8 底层类型）
└── 需要接口（多继承）→ UINTERFACE（双类模式）

已存在类型中：
├── 需要标记成员变量 → UPROPERTY（最常用）
└── 需要标记成员函数 → UFUNCTION
```

## 宏核心对比

| 宏 | 声明对象 | GENERATED 宏 | 有无 CDO |
|----|---------|-------------|---------|
| UCLASS | 类（UObject/AActor 派生） | GENERATED_BODY() | 有 |
| USTRUCT | 结构体 | GENERATED_USTRUCT_BODY() | 无 |
| UENUM | 枚举 | 无需 | 无 |
| UINTERFACE | 接口 | GENERATED_BODY() | 有（U 类） |
| UPROPERTY | 成员变量 | — | — |
| UFUNCTION | 成员函数 | — | — |

## UINTERFACE 双类模式

```cpp
// U 类 — 反射元数据持有者
UCLASS(MinimalAPI)
class UXGBaseInterface : public UInterface
{
    GENERATED_BODY()
};

// I 类 — C++ 抽象接口
class IXGBaseInterface : public IInterface
{
    GENERATED_BODY()
public:
    virtual void SomeFunction() = 0;
};

// 实现
class AMyActor : public AActor, public IXGBaseInterface
{
    virtual void SomeFunction() override;
};

// 判定方式
Cast<IXGBaseInterface>(Actor);
Actor->ImplementsInterface(UXGBaseInterface::StaticClass());
Actor->Implements<UXGBaseInterface>();
```

## 常见陷阱

- **BlueprintPure = false** — `= false` 不是合法的 BlueprintPure 参数，但不会报编译错，结果函数在蓝图不显示
- **不支持的类型** — uint16/uint32（非 enum）/int8/int16 不能用于 UPROPERTY；嵌套容器 TArray<TArray<>> 不被 UHT 识别
- **UENUM 底层类型** — 必须指定 `: uint8`，否则蓝图中不可见
- **GENERATED_USTRUCT_BODY()** — USTRUCT 必须使用这个，不能使用 GENERATED_BODY()
- **struct 中的 UObject*** — 用 UPROPERTY 修饰时 GC 会追踪，但结构体本身不参与 GC
