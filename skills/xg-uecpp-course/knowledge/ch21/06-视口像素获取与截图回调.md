# 视口像素获取与截图回调

## 概述

通过 UE 的 `FScreenshotRequest` 和 `UGameViewportClient` 机制，在运行时获取当前视口的 RGBA 像素数据，这是 WebP 生成的数据来源。

## 核心机制

### FScreenshotRequest

UE 提供了截图请求结构体，通过代理机制异步获取渲染结果：

- 位置：`Engine/Source/Runtime/Engine/Public/` 下的截图相关模块
- 核心：`FScreenshotRequest::RequestScreenshot()` 发起截图命令
- 回调：通过 `UGameViewportClient::OnScreenshotCaptured()` 添加代理

### GameViewportClient 的截图能力

[UXGSampleWebPOneShotSubsystem.h](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleWebP/Source/XGSampleWebP/Public/Subsystem/XGSampleWebPOneShotSubsystem.h) 中的截图回调：

```cpp
UFUNCTION()
void ScreenShotCallback(int32 InWidth, int32 InHeight, const TArray<FColor>& InColors);
```

回调参数：
| 参数 | 含义 |
|------|------|
| `InWidth` | 截图的像素宽度 |
| `InHeight` | 截图的像素高度 |
| `InColors` | 完整的 RGBA 像素数组（`TArray<FColor>`） |

### 截图代理注册流程

1. 声明 `FDelegateHandle ScreenHandle` 记录注册句柄
2. 获取 `UGameViewportClient`，调用 `OnScreenshotCaptured().AddUObject(this, &ThisClass::ScreenShotCallback)`
3. 调用 `FScreenshotRequest::RequestScreenshot(false)` 发起截图（参数 `false` 表示不显示 UI）
4. 在回调中通过 `OnScreenshotCaptured().Remove(ScreenHandle)` 移除

**为什么是一次性的？**
截图注册后如果不移除，每次截图都会触发回调。对于单张 WebP 生成，只需要截一次，所以绑定后立即 `RequestScreenshot`，回调后就 `Remove`。

### 渲染线程延迟

截图涉及渲染线程（RenderThread），具有一帧延迟：

```
GameThread 发起截图 → RenderThread 渲染 → 下一帧回调返回像素
```

在发起截图前需调用 `FlushRenderingCommands()` 确保渲染命令队列被刷新。

## Subsystem 获取视口尺寸

`FXGSampleWebPCore` 提供静态辅助方法：

```cpp
static bool GetViewportSize(UObject* WorldContextObject, FVector2D& OutViewportSize);
```

内部通过 `GEngine->GameViewport->Viewport->GetSizeXY()` 获取当前视口分辨率。

## 像素数据拷贝的必要性

截图回调返回的 `InColors` 是 `const TArray<FColor>&` 引用。如果将此引用直接传给异步线程，在回调结束后引用可能失效。

安全做法：**在回调中做值拷贝**，将 `TArray<FColor>` 完整复制一份再传给异步编码线程。

## 编辑器 vs 打包运行

- 编辑器中有专用的 `FScreenshotRequest` 工具窗口（High Resolution Screenshot）
- 课程使用的截图接口**在打包后同样可用**，不限于编辑器环境
