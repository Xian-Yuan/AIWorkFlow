# 响应类型体系

## 概述

百度 ERNIE Bot 的响应结构比讯飞更简洁。所有响应类型在 `XGSampleBDRespType.h` 中定义为 USTRUCT，支持蓝图反射。

## 响应类型一览

[全部定义](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Public/Type/XGSampleBDRespType.h)

### FBDErrorInfo——错误信息

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDErrorInfo
{
    int32 error_code;
    FString error_msg;
};
```

百度 API 错误直接在**顶层 JSON** 返回，无需像讯飞那样在 1st/2nd level 嵌套中查找。常见错误码：

| error_code | 含义 |
|------------|------|
| 336001 | 非法参数（JSON 格式错误、缺少必填字段） |
| 336002 | 鉴权失败（AKSK 签名错误） |
| 336003 | 速率限制超限 |
| 336004 | Token 配额不足 |

### FBDStreamMessage——流式中间消息

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDStreamMessage
{
    FString result;                  // 当前 chunk 的文本
    bool is_safety;                  // 安全验证结果
    FXGSampleBDUsage usage;          // token 消耗
    TArray<FXGSampleBDSearchResult> search_info;  // 知识检索引用
};
```

每个流式 chunk 独立解析，`result` 只包含当前增量文本（不是累积结果），需要调用方拼接。

### FBDFinalStreamMessage——流式最终消息

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDFinalStreamMessage
{
    FXGSampleBDHeaderInfo header_info;     // HTTP 响应头信息
    FString total_stream;                  // 流式累积的完整文本
    TArray<FXGSampleBDStreamMessage> stream_messages; // 全部 chunk 记录
};
```

流式完成后组装。`total_stream` 由各 chunk 的 `result` 累积拼接而成。

### FBDNoSteamFinalMessage——非流式最终消息

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDNoSteamFinalMessage
{
    FString id;                              // 请求 ID
    FString object;                          // 对象类型
    int32 created;                           // 创建时间戳
    FString result;                          // 完整响应文本
    bool is_safety;                          // 安全验证
    FXGSampleBDUsage usage;                  // token 消耗
    FXGSampleBDHeaderInfo header_info;       // HTTP 响应头信息
};
```

非流式模式一次性返回的完整响应。

### FBDUsage——Token 消耗

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDUsage
{
    int32 prompt_tokens;            // 输入 token 数
    int32 completion_tokens;        // 输出 token 数
    int32 total_tokens;             // 合计 token 数
};
```

### FBDSearchInfo——知识检索引用

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDSearchResult
{
    int32 index;                    // 引用序号
    FString url;                    // 来源 URL
    FString title;                  // 来源标题
    FString site_info;              // 站点信息
};
```

当请求开启了 `disable_search = false` 且 `enable_citation = true` 时，百度 ERNIE Bot 会从互联网检索知识并返回引用来源。每个 source 包含 URL、标题、索引等信息。

### FBDHeaderInfo——限流 Header

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDHeaderInfo
{
    int32 XRatelimitLimitRequests;      // 请求频率上限
    int32 XRatelimitLimitTokens;        // Token 消耗上限
    int32 XRatelimitRemainingRequests;  // 剩余请求次数
    int32 XRatelimitRemainingTokens;    // 剩余 Token 数
};
```

从 HTTP 响应 Header 中提取的限流信息，字段对应百度 BCE 的 `x-ratelimit-*` 系列 Header。

## 统一响应体——FBDRespInfo

[定义](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Public/Type/XGSampleBDRespType.h#L116-L149)

```cpp
USTRUCT(BlueprintType)
struct FXGSampleBDRespInfo
{
    bool isSuccess;                              // 是否成功
    bool isStream;                               // 是否流式模式
    FString Message;                             // 统一消息文本
    FXGSampleBDErrorInfo errorInfo;              // 错误信息
    FXGSampleBDStreamMessage streamMessage;      // 流式中间消息
    FXGSampleBDNoSteamFinalMessage NoSteamFinalMessage; // 非流式最终消息
    FXGSampleBDFinalStreamMessage FinalStreamMessage;   // 流式最终消息
};
```

配合蓝图节点使用。在 `CallOnUpdate` / `CallOnSuccess` / `CallOnFail` 中构造，填充对应分支的字段，其他字段保持默认值。

## 与讯飞响应类型的对比

| 维度 | 百度 | 讯飞 |
|------|------|------|
| 错误结构 | 顶层 `error_code` + `error_msg` | 1st level `code` + 2nd level `sid` |
| 流式中间 | 单层 `result` + `usage` | `content` + `content_type` + `index` |
| Token 统计 | `usage` 字段 | `payload.usage.text` |
| 引用信息 | `search_info` 数组 | `payload.search_info` |
| 限流信息 | `x-ratelimit-*` Header | 无对应字段 |
| 序列化方式 | FJsonObjectConverter 反射 | 手动遍历 FJsonObject |
