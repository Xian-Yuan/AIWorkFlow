# 非流式响应解析

## 概述

当请求的 `stream = false` 时，百度 API 返回完整的 JSON 响应。在 `OnHttpRespReceived` 回调中一次性解析。

## 回调入口

[实现](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Private/AsyncAction/XGSampleBPAyncAction.cpp#L104-L304)

```cpp
void UXGSampleBDAyncAction::OnHttpRespReceived(
    FHttpRequestPtr HttpRequest,
    FHttpResponsePtr HttpResponse,
    bool bSucceeded)
```

## 解析流程

```
OnHttpRespReceived
  ├─ bSucceeded && Status == Succeeded && ResponseCode == 200?
  │   ├─ YES → 判断 ReqInfo.stream
  │   │   ├─ stream = true  → 流式处理分支 (ErrorSteamData 检查)
  │   │   └─ stream = false → 非流式处理分支
  │   │       ├─ JSON 反序列化失败 → CallOnFail("ParseJson Failed")
  │   │       ├─ 存在 error_code/error_msg → CallOnFail(错误信息)
  │   │       └─ 成功 → 解析 NoSteamFinalMessage → CallOnSuccess
  │   └─ NO → CallOnFail("Http Failed")
  └─ 最终 → RealeaseResources()
```

### 1. HTTP 状态验证

```cpp
if (bSucceeded && HttpRequest->GetStatus() == EHttpRequestStatus::Succeeded
    && HttpResponse->GetResponseCode() == 200)
```

三层验证：通信成功 + 请求状态成功 + HTTP 200。

### 2. JSON 反序列化

```cpp
TSharedPtr<FJsonObject> TextJsonObject;
TSharedRef<TJsonReader<TCHAR>> TextJsonReader =
    TJsonReaderFactory<TCHAR>::Create(ResponseJson);
bool IsDeserialize = FJsonSerializer::Deserialize(TextJsonReader, TextJsonObject);
```

### 3. 错误检测（百度特有）

```cpp
bool bError = TextJsonObject->TryGetField(TEXT("error_code"))
           || TextJsonObject->TryGetField(TEXT("error_msg"));
```

与讯飞不同，百度的错误信息直接在**顶层**，无需多层嵌套检索。

### 4. 成功解析

```cpp
FXGSampleRespNoSteamFinalMessage NoSteamFinalMessage;
FJsonObjectConverter::JsonObjectStringToUStruct(ResponseJson, &NoSteamFinalMessage);
```

使用 `FJsonObjectConverter::JsonObjectStringToUStruct` 反射反序列化，自动匹配 JSON 字段名与 USTRUCT 成员名。

### 5. HTTP Header 提取

从响应 Header 中提取限流信息：

```cpp
FString XRatelimitLimitRequestsStr =
    HttpResponse->GetHeader(TEXT("X-Ratelimit-Limit-Requests"));
if (!XRatelimitLimitRequestsStr.IsEmpty())
    NoSteamFinalMessage.HeaderInfo.XRatelimitLimitRequests =
        FCString::Atoi(*XRatelimitLimitRequestsStr);
```

提取的 Header 字段：

| Header 名称 | 含义 |
|-------------|------|
| X-Ratelimit-Limit-Requests | 请求频率上限 |
| X-Ratelimit-Limit-Tokens | Token 消耗上限 |
| X-Ratelimit-Remaining-Requests | 剩余请求次数 |
| X-Ratelimit-Remaining-Tokens | 剩余 Token 数 |

### 6. 成功/失败回调

```cpp
CallOnSuccess(AsyncID, true, RespInfo.Message, RespInfo);
// 或
CallOnFail(AsyncID, false, RespInfo.Message, RespInfo);
```

## 与讯飞响应解析的对比

| 维度 | 百度 | 讯飞 |
|------|------|------|
| 错误位置 | 顶层 `error_code` + `error_msg` | 嵌套在 1st/2nd level 的 `code` + `message` |
| 反序列化 | FJsonObjectConverter 单步反射 | 手动遍历 + 条件分支 |
| Header 信息 | x-ratelimit-* 限流头 | code/message 状态码 |
| 成功时 | 单个完整响应 | 单个完整响应 |
