# 第二十八章 知识概览

## 章节主题

本章实现百度文心一言（ERNIE Bot）大模型的 HTTP 流式与非流式通讯。核心任务是通过 HTTP 协议与大语言模型交互，支持**流式输出**（逐 token 实时显示）和**非流式输出**（完整响应一次返回）。

## Pipeline 总览

```
[请求] → AKSK 签名 → HTTP POST → 百度 ERNIE Bot API
                                      ↓
                            ┌─────────────────┐
                            │  stream = false  │  → OnResponseReceived → 完整 JSON 解析 → OnSuccess
                            │  stream = true   │  → OnStreamReady → 逐 chunk 解析 → OnUpdate × N → OnSuccess
                            └─────────────────┘
```

## 与第二十七章的关联

- **相同点**：都使用 UBlueprintAsyncActionBase 模式、OpenSSL HMAC-SHA256 加密、FHttpModule 发送请求
- **不同点**：
  - 鉴权方式：Xunfei = URL 签名（Base64），Baidu = Header 签名（Hex）
  - 请求体：Xunfei = 三层嵌套（header/parameter/payload），Baidu = 扁平 JSON（messages 数组）
  - 错误信息：Xunfei = 嵌套在 1st/2nd level，Baidu = 顶层 error_code + error_msg
  - 流式支持：本章新增 OnUpdate 引脚和 OnStreamReady 流式处理管线
- **模块依赖**：XGSampleBDLink 依赖 XGSampleXFLinkLibrary（仅加密基库），不依赖 XGSampleXFLink 模块
