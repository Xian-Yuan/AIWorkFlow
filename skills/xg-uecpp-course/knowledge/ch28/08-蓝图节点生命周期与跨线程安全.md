# 蓝图节点生命周期与跨线程安全

## 概述

HTTP 请求生命周期跨越多个帧，涉及网络回调、线程切换、资源管理等复杂问题。本章节点 `UXGSampleBDAyncAction` 采用与第二十五章相同的生命周期模式，但因流式处理新增了跨线程委托广播的挑战。

## 生命周期

### 1. 创建——静态工厂

```cpp
UXGSampleBDAyncAction* UXGSampleBDAyncAction::XGBDChat(
    UObject* WorldContextObject, ...)
{
    UXGSampleBDAyncAction* Action = NewObject<UXGSampleBDAyncAction>();
    Action->RegisterWithGameInstance(WorldContextObject);
    // 参数赋值...
    return Action;
}
```

### 2. 活跃期——Activate

[Activate](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Private/AsyncAction/XGSampleBPAyncAction.cpp#L40-L54)

```cpp
void UXGSampleBDAyncAction::Activate()
{
    Super::Activate();
    AsyncTask(ENamedThreads::GameThread, [this]() {
        this->Activate_Internal();
    });
    Then.Broadcast(...);
}
```

- `RegisterWithGameInstance` 绑定节点生命周期到 GameInstance，防止节点被 GC 回收
- `Super::Activate()` 必须调用
- `AsyncTask(GameThread, ...)` 将 HTTP 请求推迟到下一帧发送

### 3. 销毁——RealeaseResources

[实现](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Private/AsyncAction/XGSampleBPAyncAction.cpp#L376-L382)

```cpp
void UXGSampleBDAyncAction::RealeaseResources()
{
    if (HttpRequest.IsValid())
    {
        HttpRequest->OnProcessRequestComplete().Unbind();
        HttpRequest->OnStreamReady().Unbind();
        HttpRequest->CancelRequest();
        HttpRequest.Reset();
    }
    SetReadyToDestroy();
}
```

- 解绑所有 HTTP 委托
- 取消 HTTP 请求
- 调用 `SetReadyToDestroy()` 标记销毁

必须在每次 `OnHttpRespReceived` 完成后调用，无论成功或失败。

## 跨线程安全

### 风险来源

`OnStreamReady` 委托在**非游戏线程**回调。UE 的 UObject 系统和蓝图委托广播只能在游戏线程进行：

```cpp
bool UXGSampleBDAyncAction::OnStreamReady(void* Data, int64 Length)
```

### 防御措施

[实现](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Private/AsyncAction/XGSampleBPAyncAction.cpp#L310-L322)

```cpp
bool UXGSampleBDAyncAction::OnStreamReady(void* Data, int64 Length)
{
    if (IsEngineExitRequested()) return false;
    
    TArray<uint8> DataArray;
    DataArray.Append((uint8*)Data, Length);
    
    FFunctionGraphTask::CreateAndDispatchWhenReady(
        [this, DataArray]()
        {
            ContentDataArray.Append(DataArray);
            ParseStreamData();
        },
        TStatId(), nullptr, ENamedThreads::GameThread
    );
    return true;
}
```

| 措施 | 说明 |
|------|------|
| `IsEngineExitRequested()` | 引擎退出时直接返回 false 停止流式传输，避免访问已销毁对象 |
| `FFunctionGraphTask` + `ENamedThreads::GameThread` | 将数据处理调度到游戏线程执行 |
| 值捕获 `DataArray` | `TArray<uint8>` 值拷贝，避免访问原始 `void*` 内存 |
| `return true` | 除非引擎退出，否则始终继续接收数据 |

### 缺少的防御

为确保完全安全，应额外考虑：
- **WeakPtr 捕获**：用 `TWeakObjectPtr<UXGSampleBDAyncAction>` 替代裸 `this` 捕获。如果节点在流式处理中被销毁，WeakPtr 可检测到。
- **线程安全的标记**：使用 `IsInGameThread()` 断言验证执行线程。

## 调用链

```
1. 用户拖入 BDChat 节点
2. 设置参数（API Key, Secret Key, messages, stream 等）
3. 执行 → Activate()
   ├─ Then 广播（"等待中"）
   └─ 下一帧 → Activate_Internal()
       └─ SendHttp()
           ├─ 组装 URL、Body、Headers
           └─ HttpRequest->ProcessRequest()
               ├─ (stream=false) → OnHttpRespReceived → 非流式处理 → RealeaseResources
               │                     ├─ OnSuccess 或 OnFail
               │
               └─ (stream=true)  → OnStreamReady(×N) → FFunctionGraphTask(GameThread)
                                    ├─ ParseStreamData → OnUpdate
                                    └─ 最终 data: 空 → OnSuccess → RealeaseResources
```

## UObject 生命周期要点

| 时机 | 状态 | 说明 |
|------|------|------|
| NewObject 后 | 未注册 | 可能被 GC 回收 |
| RegisterWithGameInstance 后 | 活跃 | 绑定到 GameInstance，防止 GC |
| Activate 后 | 开始请求 | HTTP 请求进行中 |
| 回调触发后 | 准备销毁 | Broadcast 完成 |
| SetReadyToDestroy 后 | 标记销毁 | 等待 GC 回收 |
