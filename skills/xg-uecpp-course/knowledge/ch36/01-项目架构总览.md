# 项目架构总览

## 工程概览

017_XGRPG 是基于虚幻引擎 5 的 GAS（Gameplay Ability System）实战项目，约 7600 行 C++ 代码。项目实现了完整的 RPG 战斗体系，包括近战连击、远程攻击、技能系统、装备系统、背包系统和多人网络同步。

### 项目参考来源

项目架构同时参考了 Lyra 项目、MMORPG 教程和官方 Action RPG 示例的设计模式，在 GAS 框架的基础上进行了适合 RPG 场景的整合。

### 目录命名说明

项目中存在多处目录名拼写不正确的情况，这是项目实际使用的名称，在引用时需保持一致：

| 实际目录名 | 正确拼写 | 所在路径 |
|-----------|---------|---------|
| `Excutions/` | Executions | 伤害执行计算 |
| `NoifyState/` | NotifyState | 动画通知状态 |
| `Noitfy/` | Notify | 动画通知 |
| `FeedBack/` | Feedback | 伤害数字系统 |
| `ChatacterInfo/` | CharacterInfo | 角色属性 UI |
| `ArrtibuteSet` | AttributeSet | 属性集子类名 |

## 系统分层架构

项目按"框架层 → 业务逻辑层 → 表现层"三层组织：

```
框架层（GAS 基础设施）
  ├── AbilitySystemComponent  — 能力系统组件
  ├── GameplayAbility         — 能力基类
  ├── AttributeSet            — 属性集
  └── GameplayTags            — 标签系统
        ↓
业务逻辑层（游戏玩法）
  ├── ComboComponent          — 连击系统
  ├── HealthComponent         — 生命值/死亡管理
  ├── HitBox 系列             — 碰撞伤害检测
  ├── InventoryComponent      — 背包
  ├── EquipmentComponent      — 装备
  └── GameplayEffectExecution — 伤害计算
        ↓
表现层（UI/视觉反馈）
  ├── NumberPopComponent 系列  — 伤害数字
  ├── UI_CharacterInfo        — 角色属性面板
  ├── UI_Inventory/Equipment   — 背包/装备界面
  └── UUI_DamageNum            — 伤害数字控件
```

## 网络架构

- 项目采用客户端-服务器（CS）架构
- ASC 网络同步模式：Full（服务端） / Mixed（客户端自主） / Minimal（只读）
- 技能执行策略：LocalPredicted（本地预测）为主、ServerInitiated（服务端发起）用于死亡特殊能力
- GEEC 仅在服务端执行（`WITH_SERVER_CODE` 宏保护）
- 动画通知不在专属服务器（Dedicated Server）上执行
- 背包/装备组件均标记为 `Replicated`，通过 RPC 同步 UI 刷新
