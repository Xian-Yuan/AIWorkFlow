# 动画通知系统

## 概述

动画通知（AnimNotify / AnimNotifyState）是连接蒙太奇播放与游戏逻辑的桥梁。项目中共有 3 个 AnimNotify 类和 5 个 AnimNotifyState 类，分别位于 `Noitfy/` 和 `NoifyState/` 目录下。

## AnimNotify 类（一次性通知）

### AnimNotify_ApplyGameplayEffect

在蒙太奇播放到特定帧时，在服务端生成一个 GameplayEffect 碰撞盒子（AXGRPGHitBox_ApplyGameEffect），盒子命中目标后应用指定的 GameplayEffect。

```cpp
UCLASS(meta = (DisplayName = "ApplyGameplayEffect"))
class UAnimNotify_ApplyGameplayEffect : public UAnimNotify
{
    // 要应用的 GE 类
    TSubclassOf<UGameplayEffect> EffectClass;

    // 生成的碰撞盒子类
    TSubclassOf<AXGRPGHitBox_ApplyGameEffect> HitObjectClass;

    // 碰撞体相对插槽的位置偏移
    FVector RelativeOffsetLocation;

    // 碰撞体相对插槽的旋转偏移
    FRotator RotationOffset;

    // 存在的生命周期
    float LifeTime;

    // 生成在哪个骨骼插槽上
    FName InSocketName;
};
```

**实现要点**：
- 只在非客户端执行（`!NM_Client`）
- 使用 `SpawnActorDeferred` + `FinishSpawning` 两步式生成
- 生成后通过 `AttachToComponent` 绑定到指定骨骼插槽
- 角色类走 OpenFire 插槽，非角色类额外旋转 Yaw+90°

### AnimNotify_Attack

触发攻击碰撞判定——在蒙太奇的关键帧通知生成碰撞盒子（AXGRPGHitBox），盒子命中目标时应用 BuffTags 标记的 GameplayTag。

```cpp
UCLASS(meta = (DisplayName = "Attack"))
class UAnimNotify_Attack : public UAnimNotify
{
    // 触发哪些 GameplayTag 作为 Buff 标识
    TArray<FGameplayTagContainer> BuffTags;

    // 生成的碰撞盒子类
    TSubclassOf<AXGRPGHitBox> HitObjectClass;

    // 碰撞体位置/旋转/大小参数
    FVector RelativeOffsetLocation;
    FRotator RotationOffset;
    FVector BoxExtent;
    float LifeTime;
    FName InSocketName;
    bool bBind;    // 是否绑定到骨骼
};
```

**与 ApplyGameplayEffect 的区别**：
- Attack 通知生成 AXGRPGHitBox（通用碰撞盒子），携带的是 GameplayTag（标记伤害类型）
- ApplyGameplayEffect 通知生成 AXGRPGHitBox_ApplyGameEffect，携带的是整个 GameplayEffect 类

### AnimNotify_ResetCombo

连击结束时重置 ComboIndex 为 0，确保下次攻击从第一段连击开始。

```cpp

{

};
```

## AnimNotifyState 类（持续状态通知）

### AnimNotifyState_NextCombo（连击窗口控制）

标记一段持续的时间区间作为"连击窗口"：Begin 时消耗连击标记（ResetPressed），End 时检查玩家是否仍在按下攻击键（IsStillPressed），若是则自动进入下一段连击。

```cpp
UCLASS()
class UAnimNotifyState_NextCombo : public UAnimNotifyState
{
    // NotifyBegin：ComboComponent->ResetPressed() — 消耗连击标记
    // NotifyEnd：如果 IsStillPressed() 为 true，调用 Character->ComboMelee()
};
```

**关键逻辑**：排除专用服务器（`!NM_DedicatedServer`），确保只在客户端和监听服务器上执行输入检测。

### AnimNotifyState_AddForce（冲刺冲力）

在 Begin 时对角色施加一次性脉冲（AddImpulse），在 End 时立即停止移动。使用基于角色朝向的方向映射。

```cpp
UCLASS(meta = (DisplayName = "AddForce"))
class UAnimNotifyState_AddForce : public UAnimNotifyState
{
    // 方向向量（在角色本地空间，X=前进, Y=右, Z=上）
    FVector DirectionForce;
    // 冲力大小（默认 180000.f）
    float ForceSize;
};
```

**方向映射逻辑**：DirectionForce 的三个分量分别映射到角色的 ActorForward/Right/Up 向量，实现相对角色朝向的冲力方向。Begin 时一次性施加全部冲力，End 时调用 `StopMovementImmediately()`。

### AnimNotifyState_StopSpeed（速度锁定）

在持续时间内锁定角色移动速度为 0，防止冲刺或技能过程中被外部因素移动。

```cpp
UCLASS(meta = (DisplayName = "StopSpeed"))
class UAnimNotifyState_StopSpeed : public UAnimNotifyState
{
    // NotifyBegin：保存 MaxWalkSpeed/MinAnalogWalkSpeed，设为 0
    // NotifyEnd：恢复保存的原始速度值
};
```

### AnimNotifyState_StopRotation（朝向锁定）

在持续时间内禁止角色朝向跟随控制器旋转，防止冲刺或技能播放过程中角色自动转向。

```cpp
UCLASS(meta = (DisplayName = "StopRotation"))
class UAnimNotifyState_StopRotation : public UAnimNotifyState
{
    // NotifyBegin：bUseControllerRotationYaw = false, bOrientRotationToMovement = false
    // NotifyEnd：bOrientRotationToMovement = true（恢复朝向跟随移动方向）
};
```

### AnimNotifyState_IgnoreInput（输入忽略）

在持续时间内禁用玩家输入，防止动画播放过程中玩家操作干扰。

```cpp
UCLASS(meta = (DisplayName = "IgnoreInput"))
class UAnimNotifyState_IgnoreInput : public UAnimNotifyState
{
    // NotifyBegin：根据网络模式决定是否禁用输入
    // NotifyEnd：恢复输入
};
```

**网络感知实现**：
- 在客户端（NM_Client）：仅对 Autonomous Proxy（本地玩家）禁用输入，不对 Simulated Proxy 操作
- 在独立模式或监听服务器（Standalone/ListenServer）：直接禁用输入
- 不修改专用服务器的输入状态

## 设计原则

1. **单一职责**：每个通知类只做一件事——ApplyGameplayEffect 只负责应用 GE，Attack 只负责触发攻击碰撞，ResetCombo 只负责重置连击
2. **网络感知**：AnimNotifyState 的实现包含网络模式判断，关键操作只在服务端执行，输入相关操作仅在 Autonomous Proxy 上执行
3. **使用 Deferred Spawning**：碰撞盒子类使用 `SpawnActorDeferred` + `FinishSpawning` 模式，确保参数设置完成后再完成生成
4. **连击状态机**：NextCombo 与 ResetCombo 配合，实现"窗口期内输入→进入下一段→结束重置"的连击闭环
