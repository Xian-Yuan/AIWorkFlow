# 独立程序手动打包与部署

## 概述

HTTPServer 独立程序的打包方式与第二十三章 Slate 独立程序类似——没有 UAT 支持，需要手动从引擎编译产物中拷贝必要文件。区别在于 HTTP Server 程序**不需要 Slate 资源**（图标、字体、Shader），打包体积更小。

## 打包步骤

根据字幕 021 的描述，打包过程分为以下步骤：

### 步骤 1：确认编译产物

确保 XGSampleServer 已在开发机上使用源码引擎编译通过。Debug 模式下产物路径：

```
{EngineRoot}/Engine/Binaries/Win64/XGSampleServer-Win64-Debug.exe
```

### 步骤 2：创建打包目录

```
XGHTTPProgram/
└── Engine/
    ├── Binaries/
    │   └── Win64/           ← 可执行文件和 DLL
    ├── Config/              ← 引擎配置
    │   └── BaseEngine.ini
    └── Content/
        └── Internationalization/  ← 国际化资源
```

### 步骤 3：拷贝核心依赖

从引擎编译产物拷贝以下内容到打包目录：
1. `XGSampleServer-Win64-Debug.exe` — 程序主文件
2. 所有相关的 DLL（Core、CoreUObject、HTTPServer 等依赖模块的 DLL）
3. `Config/BaseEngine.ini` — 基础引擎配置

### 步骤 4：配置网络

编辑 `Engine/Config/BaseEngine.ini`，添加网络绑定配置：

```ini
[HTTP]
HttpRetryDelay=1
HttpTimeout=30

[HTTPServer]
DefaultBindAddress=0.0.0.0
```

`DefaultBindAddress=0.0.0.0` 表示监听所有网络接口，使服务器可以被 LAN 和公网访问。

### 步骤 5：启动验证

使用批处理文件启动（而非双击 EXE），以便查看控制台输出：

```batch
@echo off
start Engine\Binaries\Win64\XGSampleServer-Win64-Debug.exe
pause
```

### 步骤 6：部署测试

| 场景 | 说明 |
|------|------|
| 本机测试 | 客户端配置 `ServerURL = http://127.0.0.1:11235/Login` |
| LAN 测试 | 客户端配置 `ServerURL = http://192.168.x.x:11235/Login` |
| 公网部署 | 将程序拷贝到云服务器，配置防火墙开放端口 |

## HTTP Server 的最小依赖

与 Slate 独立程序相比，HTTP Server 的依赖更少：

| 依赖项 | Slate 程序 | HTTP Server | 说明 |
|--------|-----------|-------------|------|
| Engine/Binaries/Win64/ | 需要 | 需要 | 程序文件 + DLL |
| Engine/Config/ | 需要 | 需要 | 引擎配置 |
| Engine/Content/Slate/ | 需要 | **不需要** | 无 UI，无需 Slate 图标资源 |
| Engine/Content/EditorSlate/ | 需要 | **不需要** | 无编辑器图标 |
| Engine/Shaders/ | 需要 | **不需要** | 无渲染管线 |
| Engine/Content/Localization/ | 需要 | 视情况 | 可精简 |
| Engine/ThirdParty/CEF3/ | 需要 | **不需要** | 无 WebBrowser 依赖 |
| Engine/Plugins/ | 部分需要 | **不需要** | 独立程序不依赖插件 |

## 与第二十三章打包方式的对比

| 维度 | 第二十三章 (Slate) | 第二十六章 (HTTPServer) |
|------|-------------------|----------------------|
| 打包子文件夹 | `ExeBinariesSubFolder = "XGPrograms"` | 无（直接在 Win64 下） |
| Slate 资源 | 必须保留 | 不需要 |
| Shader 资源 | 必须保留 | 不需要 |
| CEF3 依赖 | 需要 | 不需要 |
| 打包体积 | 较大 | 较小 |
| 启动方式 | 双击或批处理 | 批处理（需查看控制台输出） |
| 网络配置 | 不需要 | 必须配置 `DefaultBindAddress` |
