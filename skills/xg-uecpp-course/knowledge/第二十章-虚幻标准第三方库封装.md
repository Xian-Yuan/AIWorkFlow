# 第二十章：虚幻标准第三方库封装

## 章节概述

本章讲解 Unreal Engine 中 **标准第三方库封装** 的完整流程，以 `XGSampleTemp` 项目下的 7 个插件为教学载体。核心内容覆盖从基础的插件类型创建到完整的第三方 DLL 集成、加载、错误排查和冲突解决。

本章的核心理念是 **External/Internal 模块拆分**：将第三方库隔离在 `ModuleType.External` 模块中，通过 Build.cs 配置链接预编译的 .lib 和 DLL，避免引擎直接编译第三方源码。

## 关键知识点

### 插件分类维度

- **运行位置**：Runtime 插件（游戏运行时可用）vs Editor 插件（仅编辑器，打包剥离）
- **内容类型**：Code（含 C++ 源码）vs Content Only（仅蓝图资源）
- **功能类型**：空骨架、蓝图函数库、编辑器模式、独立窗口、工具栏按钮、第三方库集成

### 第三方库集成核心步骤

1. 创建 External 模块（`Type = ModuleType.External`）
2. 配置头文件搜索路径（`PublicSystemIncludePaths`）
3. 链接导入库（`PublicAdditionalLibraries` + `.lib` 文件）
4. 声明延迟加载（`PublicDelayLoadDLLs` + DLL 名称）
5. 自动拷贝 DLL（UBT 预构建 C# 脚本）
6. 配置打包依赖（`RuntimeDependencies`）
7. 在 Internal 模块中引用 External 模块
8. 在 `StartupModule()` 中显式加载并验证 DLL

### 常见陷阱

| 问题 | 解决方案 |
|------|---------|
| Windows SDK 宏冲突 | `PushAllowPlatformFile/PopAllowPlatformFile` 包裹第三方头文件 |
| OpenSSL 符号冲突 | 使用引擎内置 OpenSSL，不自购引入 |
| DLL 缺失导致崩溃 | 自动拷贝脚本 + RuntimeDependencies |
| 打包不含 DLL | Build.cs 中配置 RuntimeDependencies |
| 编辑器插件在打包后消失 | 这是正常行为，`Type: "Editor"` 会被剥离 |
