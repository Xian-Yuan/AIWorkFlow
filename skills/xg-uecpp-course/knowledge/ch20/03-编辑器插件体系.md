# 编辑器插件体系

## 概述

编辑器类型的插件（`.uplugin` 中 `Type: "Editor"`）只在编辑器环境下运行，**打包时会被自动剥离**。本章涉及三种编辑器插件：Editor Mode（交互模式）、Editor Standalone Window（独立窗口）、Editor Toolbar Button（工具栏按钮）。

---

## 编辑器插件通用特征

### .uplugin Type 字段

```json
"Modules": [
    {
        "Name": "XGEditorMode",
        "Type": "Editor",
        "LoadingPhase": "Default"
    }
]
```

`Type: "Editor"` 的模块在打包时被自动排除。

### 依赖模块

编辑器插件需要添加以下依赖到 `PrivateDependencyModuleNames`（以 [XGEditorMode.Build.cs](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/002_XGSampleTemp/Plugins/XGEditorMode/Source/XGEditorMode/XGEditorMode.Build.cs) 为例）：

| 模块 | 用途 |
|------|------|
| `Slate` | UI 框架核心 |
| `SlateCore` | Slate 基础类型和控件 |
| `InputCore` | 输入处理 |
| `EditorFramework` | 编辑器框架 |
| `EditorStyle` | 编辑器样式 |
| `UnrealEd` | 编辑器核心功能 |
| `LevelEditor` | 关卡编辑器集成 |
| `InteractiveToolsFramework` | 交互工具框架 |
| `EditorInteractiveToolsFramework` | 编辑器交互工具扩展 |

## Editor Mode 编辑器模式

### 架构层级

```
模块启动 (Module)
  → 注册 EdMode (UEdMode)
    → 注册 Commands (TCommands)
      → 创建 Toolkit (FModeToolkit)
        → 使用 Tool (UInteractiveTool)
```

### 核心类

| 类 | 基类 | 职责 |
|----|------|------|
| `XGEditorModeModule` | `IModuleInterface` | 模块入口，注册 EdMode |
| `XGEditorModeEditorMode` | `UEdMode` | 编辑器模式定义，绑定 Input 和 Tool |
| `XGEditorModeEditorModeCommands` | `TCommands` | 命令注册（快捷键、按钮命令） |
| `XGEditorModeEditorModeToolkit` | `FModeToolkit` | 模式 UI 面板 |
| `XGEditorModeSimpleTool` | `UInteractiveTool` | 简单交互工具 |
| `XGEditorModeInteractiveTool` | `UInteractiveTool` | 复杂交互工具 |

### 组件注册流程

1. **模块启动**：在 `StartupModule()` 中注册 `UEdMode` 子类
2. **Commands 初始化**：`TCommands::Register()` 注册命令列表
3. **EdMode 绑定**：将 Input 行为和 Tool 对象绑定到模式
4. **Toolkit 创建**：`CreateToolkit()` 返回 UI 面板实例

## 编辑器开发难度分级

| 等级 | 说明 |
|------|------|
| Gameplay 开发 | 相对容易，资源丰富 |
| **编辑器开发** | **高于 Gameplay，低于引擎底层** |
| 引擎底层开发 | 难度最高 |

编辑器开发的学习路线：

```
Gameplay 认知 → 资源类熟悉 → Slate UI 学习 → 编辑器开发
```
