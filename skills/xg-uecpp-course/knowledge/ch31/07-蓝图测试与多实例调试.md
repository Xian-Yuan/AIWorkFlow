# 蓝图测试与多实例调试

## 蓝图事件系统

客户端子系统暴露 4 个多播委托，供蓝图绑定：

```cpp
// 连接成功
DECLARE_MULTICAST_DELEGATE_TwoParams(
    FXGWSMClientConnectionConnectedSuccessCallBack,
    FGuid,          // ServerConnectionID
    FGuid);         // ClientConnectionID

// 连接失败
DECLARE_MULTICAST_DELEGATE_TwoParams(
    FXGWSMClientConnectionConnectedErrorCallBack,
    FString,        // Error 信息
    FGuid);         // ClientConnectionID

// 收到消息
DECLARE_MULTICAST_DELEGATE_TwoParams(
    FXGWSMClientConnectionReceiveMessageCallBack,
    FGuid,          // ClientConnectionID
    FString);       // Data（消息内容）

// 连接关闭
DECLARE_MULTICAST_DELEGATE_FourParams(
    FXGWSMClientConnectionClosedCallBack,
    int32,          // StatusCode
    FString,        // Reason
    bool,           // bWasClean
    FGuid);         // ClientConnectionID
```

## 蓝图面板结构

### 服务端控制面板

```
┌─────────────────────────────────┐
│  服务器控制                       │
│  ● 端口输入框（默认 9083）        │
│  ● LinkServer 按钮               │
│  ● UnLinkAll 按钮                │
│                                 │
│  事件绑定：                       │
│  ▲ OnConnected（控制台日志）       │
│  ▲ OnConnectionError（控制台日志） │
│  ▲ OnMessage（控制台打印消息）     │
│  ▲ OnClosed（控制台日志）          │
└─────────────────────────────────┘
```

### 客户端控制面板

```
┌─────────────────────────────────┐
│  客户端控制                       │
│  ● URL 输入框（ws://127.0.0.1:9083）
│  ● LinkServer 按钮               │
│  ● UnLinkAll 按钮                │
│  ● 消息输入框 + Send 按钮         │
│                                 │
│  事件绑定：                       │
│  ▲ OnConnectedSuccess（显示 ID）  │
│  ▲ OnConnectedError（提示错误）   │
│  ▲ OnMessage（显示消息内容）      │
│  ▲ OnClosed（显示关闭原因）       │
└─────────────────────────────────┘
```

### 事件绑定方式

```cpp
// 在蓝图 BeginPlay 或 Construct 中
UXGWSMClientSubsystem* ClientSubsystem = 
    GetGameInstance()->GetSubsystem<UXGWSMClientSubsystem>();

if (ClientSubsystem)
{
    ClientSubsystem->ClientConnectionConnectedSuccessCallBack.AddDynamic(
        this, &UMyClass::OnConnectedSuccess);
    // ... 同理绑定其他 3 个委托
}
```

## 多编辑器实例测试

### 测试架构

```
实例 1（编辑器窗口 1）：服务端
  ├── 启动 XGSampleWSM Server（端口 9083）
  ├── 绑定 4 个服务端事件
  └── 等待客户端连接

实例 2（编辑器窗口 2）：客户端
  ├── 连接 ws://127.0.0.1:9083
  ├── 绑定 4 个客户端事件
  ├── 发送测试消息
  └── 观察回调

实例 3（编辑器窗口 3）：第二个客户端（可选）
  ├── 连接同一端口
  ├── 发送不同消息
  └── 验证多连接管理
```

    或

## Postman WebSocket 测试

### 测试步骤

```
Postman 测试 WebSocket 流程：

1. 启动游戏中的服务端（端口 9083）

2. Postman 新建 WebSocket 请求：
   ws://127.0.0.1:9083

3. 发送初始化消息（Action 类型需手动编码）
   ┌─────────────────────────────────────────────────────────┐
   │ 原始 JSON（需 Base64 编码后作为二进制发送）：               │
   │ {"Code":0,"Action":1,"ClientConnectionID":"...",        │
   │  "ServerConnectionID":"00000000-0000-0000-0000-000000000000",
   │  "Description":"ClientCallServerInit","Data":""}         │
   │                                                         │
   │ 注意：Action=1 对应 ClientCallServerInit                  │
   └─────────────────────────────────────────────────────────┘

4. 等待服务端回复 ServerResponseClientInit
   → 获取 ServerConnectionID

5. 发送心跳消息
   Action=3（ClientCallTick）
   → 收到 ServerResponseTick

6. 发送业务消息
   Action=5（ClientMessageToServer）
   → 收到 ServerMessageToClient（回显）

7. 发送关闭请求
   Action=7（ClientReqQuit）
   → 收到 ServerAllowClientQuit
```

### ActionType 数值对照表

| ActionType | 数值 | 说明 |
|-----------|------|------|
| None | 0 | 无操作 |
| ClientCallServerInit | 1 | 初始化请求 |
| ServerResponseClientInit | 2 | 初始化响应 |
| ClientCallTick | 3 | 心跳 |
| ServerResponseTick | 4 | 心跳响应 |
| ClientMessageToServer | 5 | 消息发送 |
| ServerMessageToClient | 6 | 消息转发 |
| ClientReqQuit | 7 | 客户端请求退出 |
| ServerAllowClientQuit | 8 | 允许退出 |
| ServerReqQuit | 9 | 服务端请求退出 |
| ClientAllowServerQuit | 10 | 允许服务端退出 |

### Postman 测试注意事项

| 注意事项 | 说明 |
|---------|------|
| Base64 编码 | Postman 发送前需手动将 JSON 转为 Base64 |
| 超时控制 | 10s 内无操作服务端会断开连接 |
| 心跳维持 | 测试长时间连接需持续发送心跳 |
| 初始 GUID | ServerConnectionID 初始化消息中填全零 GUID |
| 回显测试 | 发送 MessageToServer 会收到原内容回显 |
| 关闭验证 | 发送 ClientReqQuit 后需确认收到 ServerAllowClientQuit |

## 完整通讯流程验证

```
1. 启动服务端（端口 9083）
   日志: [LinkServer] Port:[9083]
   日志: [OnWebSocketServerConnected]

2. 客户端连接
   日志: [OnConnected] TargetClientConnectionID:[GUID_A]
   日志: [CallInitServerConnection] Send ClientCallServerInit

3. 初始化握手
   服务端收到 Init → 回复 ServerResponseClientInit
   客户端收到 → 保存 ServerConnectionID → 状态 → Tick
   日志: [OnServerResponseClientInit]
   蓝图: OnConnectedSuccess 广播

4. 心跳维持（每 3s）
   客户端: Send ClientCallTick
   服务端: Reply ServerResponseTick
   客户端: UpdateIntervalTime()

5. 消息收发
   客户端: Send ClientMessageToServer(Data="Hello")
   服务端: Reply ServerMessageToClient(Data="Hello")
   蓝图: OnMessage 广播（显示 "Hello"）

6. 正常关闭
   客户端: Send ClientReqQuit
   服务端: Reply ServerAllowClientQuit
   客户端: 状态 → Quit → Cleanup
   蓝图: OnClosed 广播
```
