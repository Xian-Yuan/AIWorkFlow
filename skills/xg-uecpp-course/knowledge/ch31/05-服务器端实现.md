# 服务器端实现

## 架构总览

```
UXGWSMServerSubsystem（UGameInstanceSubsystem + FTickableGameObject）
  │
  ├── WebSocketServer（IWebSocketServer*）
  │     └── 端口绑定 + 连接监听
  │
  ├── ServerConnections（TMap<FGuid, FXGWSMServerConnection>）
  │     ├── Connection_A（TSharedPtr<WebSocket> + 双GUID + WaitClientToInit → Tick）
  │     ├── Connection_B
  │     └── ...
  │
  ├── Tick(float DeltaTime)
  │     └── TickServerConnections()
  │           └── 遍历所有连接 → Tick() → 超时检测(10s) → 释放
  │
  └── OnMessage(FGuid ClientCollectionID, FGuid ServerConnectionID, FString Message)
        └── 解码校验 → Switch(ActionType) 分发
```

## 服务器创建与初始化

```cpp
void UXGWSMServerSubsystem::LinkServer(int32 InPort)
```

初始化流程：

```
1. FModuleManager::LoadModuleChecked("WebSocketNetworking")
   → 加载 WebSocketNetworking 模块

2. FWebSocketServer->CreateWebSocketServer(InPort)
   → 创建 IWebSocketServer*，绑定端口

3. 绑定 Filter 回调
   Server->OnFilterConnection().BindUObject(...)
   → 返回 true 接受连接，false 拒绝

4. 绑定 OnConnected 回调
   Server->OnConnected().BindUObject(...)
   → 新连接到达时触发
```

> `IWebSocketServer` 为**原生指针**（非智能指针），需手动管理生命周期。Subsystem Deinitialize 时需释放。

## 连接状态机（服务端）

依据代码中 `EXGWSMServerConnectionStatus` 枚举：

```
新连接到达
    │
    ▼
┌────────────────┐
│ WaitClientToInit│ ← 等待客户端发送 ClientCallServerInit
└───────┬────────┘
        │ 收到 ClientCallServerInit + Code == 0
        ▼
┌──────────┐
│   Tick   │ ← 正常通信状态
│          │   收到心跳 → 不更新状态
│          │   收到消息 → 不更新状态
└─────┬────┘
      │
      ├── 收到 ClientReqQuit → PrepareToQuit
      │     → 回复 ServerAllowClientQuit → 客户端断开 → Quit
      │
      ├── 收到 ClientAllowServerQuit → Quit
      │
      ├── 累计超时(OutOfTime) → Quit
      │
      └── 服务端主动调用 UnLink → Quit
      │
      ▼
┌──────────┐
│   Quit   │ → 释放 WebSocket 资源 → 从 TMap 移除
└──────────┘
```

### 状态枚举

```cpp
UENUM()
enum class EXGWSMServerConnectionStatus : uint8
{
    None,               // 初始/未使用
    WaitClientToInit,   // 等待客户端初始化
    Tick,               // 正常通信
    PrepareToQuit,      // 准备退出（已发送或收到退出请求）
    Quit,               // 已退出
    OutOfTime,          // 超时
    Max,
};
```

## 连接接受：OnConnected 回调

当客户端 WebSocket 连接到达时触发：

```
收到新连接 (FWebSocketClientConnectionClient*)
  │
  ├── 生成 FGuid ServerConnectionID
  │
  ├── 创建 FXGWSMServerConnection
  │   ├── 存储 WebSocket 指针（ServerWebSocket）
  │   ├── 配对双 GUID（CollectionID + ServerConnectionID）
  │   └── 设置状态 = WaitClientToInit
  │
  ├── 绑定连接级回调
  │   ├── OnError
  │   ├── OnClosed
  │   └── OnMessage
  │
  └── 存入 ServerConnections TMap
      (ServerConnections.Add(ServerConnectionID, Connection))
```

### 连接可用性概念

```cpp
// 连接是否"可用"——只检查 WebSocket 指针是否有效
bool FXGWSMServerConnection::IsAvailable() const
{
    return ServerWebSocket != nullptr;
}
```

可用性判断**不依赖心跳状态**——只要 WebSocket 指针未被销毁，连接就视为可用。这是一个内部安全检查（供容器遍历时判断），不暴露给外部使用。

## 消息接收与分发（009 核心内容）

### OnMessage → 解码校验 → Switch 分发

```
OnMessage(ClientCollectionID, ServerConnectionID, RawMessage)
  │
  ├── 根据 ServerConnectionID 查找对应连接
  │   → 未找到则返回
  │
  ├── FJsonObjectConverter::JsonObjectStringToUStruct<FXGWSMMessage>
  │   → bool bParse
  │
  ├── 安全校验：
  │   if (bParse == false || Message.Code != 0)
  │   {
  │       UnLinkPointedServerConnection(ServerConnectionID);
  │       return;
  │   }
  │   // 解析失败或 Code != 0 → 直接断开连接
  │
  └── 通过校验 → Switch(Message.Action) 分发
        │
        ├── ClientCallServerInit    → 初始化处理
        ├── ClientCallTick          → 回复 ServerResponseTick
        ├── ClientMessageToServer   → 回复 ServerMessageToClient
        ├── ClientReqQuit           → 回复 ServerAllowClientQuit
        └── ClientAllowServerQuit   → 状态 Quit
```

### Switch 分发实现

```cpp
// 服务端只处理客户端发起的 ActionType
switch (Message.Action)
{
case EXGWSMActionType::ClientCallServerInit:
    // 设置状态 Tick → 回复 ServerResponseClientInit
    break;

case EXGWSMActionType::ClientCallTick:
    // 回复 ServerResponseTick
    break;

case EXGWSMActionType::ClientMessageToServer:
    // 回复 ServerMessageToClient
    break;

case EXGWSMActionType::ClientReqQuit:
    // 状态 PrepareToQuit → 回复 ServerAllowClientQuit
    break;

case EXGWSMActionType::ClientAllowServerQuit:
    // 状态 Quit → 关闭连接
    break;

default:
    break;
}
```

### 回复消息的 GUID 填充规则

服务端回复消息时，始终填入双 GUID：

```
FXGWSMMessage ReplyMessage;
ReplyMessage.ClientConnectionID = Incoming.ClientConnectionID;
ReplyMessage.ServerConnectionID = Incoming.ServerConnectionID;
// 客户端通过对比这两个 GUID 确认属于同一连接
```

## Tick 与心跳超时

### TickServerConnections

```cpp
void UXGWSMServerSubsystem::TickServerConnections(float InDeltaTime)
{
    for (auto& TmpPair : ServerConnections)
    {
        FXGWSMServerConnection& Connection = TmpPair.Value;
        Connection.Tick(InDeltaTime);
    }

    // 遍历完成后，清理状态为 Quit 的连接
    for (auto It = ServerConnections.CreateIterator(); It; ++It)
    {
        if (It.Value().Status == EXGWSMServerConnectionStatus::Quit)
        {
            It.RemoveCurrent();
        }
    }
}
```

> 注意：遍历过程中不可原地删除元素，需标记后二次清理。

### 超时参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 超时阈值 | 10s | 超过此时间未收到客户端消息 |
| 检测频率 | 每帧 Tick 一次 | FTickableGameObject::Tick |
| 超时动作 | 状态 → Quit | 下一帧清理时从 TMap 移除 |
| 心跳方向 | client→server | 客户端主动发送 ClientCallTick |

## 连接容器管理

### 数据结构

```cpp
TMap<FGuid, FXGWSMServerConnection> ServerConnections;
```

### 容器操作

| 操作 | API | 时机 |
|------|-----|------|
| Add | `Add(ServerConnectionID, Connection)` | OnConnected 回调 |
| Remove（指定） | `Remove(ServerConnectionID)` | 连接正常关闭或超时 |
| Remove（全部） | `Empty()` | Subsystem Deinitialize 或 LinkServer 重新绑定前 |
| Find | `Find(ServerConnectionID)` | 收到消息时根据 GUID 定位连接 |

### 迁移清理

当连接从旧容器转移到新容器时（如多容器场景），必须在旧容器中删除条目，否则旧容器会保留悬空引用：

```
1. 新容器: ServerConnections2.Add(ConnectionID, Connection)
2. 旧容器: ServerConnections.Remove(ConnectionID)  ← 必须
```

### 服务器停止时的清理

```cpp
void UXGWSMServerSubsystem::UnLinkAllServerConnection()
{
    // 逐个关闭 WebSocket
    for (auto& TmpPair : ServerConnections)
    {
        // Release WebSocket resource
    }
    ServerConnections.Empty();
}
```

`LinkServer` 在重新绑定新端口前应调用此方法清理旧连接。

## 关键注意事项

| 注意点 | 说明 |
|--------|------|
| `IWebSocketServer` 非智能指针 | 原生指针，需在 Deinitialize 中手动删除 |
| 手动 Tick 的必要性 | `IWebSocketServer` 无自动内部循环，需 Subsystem Tick 驱动 |
| 二次清理模式 | 遍历 TMap 时不可原地删除，需在遍历完成后统一移除 Quit 状态的连接 |
| Code != 0 直接断开 | 防御性编程，不信任上层错误码 |
| 连接可用性 | 判断标准为 WebSocket 指针是否存在，不依赖心跳状态 |
| 平台限制 | WebSocketNetworking 仅 Win/Mac/Linux |
| 端口冲突 | `CreateWebSocketServer` 失败时 LinkServer 应返回失败状态 |
| 迁移清理 | 连接从一个容器迁移到另一个时，必须从旧容器移除 |
| 文件命名 | `XGWSMSeverConnection` 拼写为 **Sever**（非 Server），保持与代码一致 |
