# 插件架构与子系统设计

## 插件基本结构

`XGSampleWSM` 是一个自包含的 UE 插件，提供 WebSocket 服务器端和客户端双角色通信能力。

### .uplugin 配置

```json
{
    "FileVersion": 3,
    "FriendlyName": "XGSampleWSM",
    "Version": 1,
    "VersionName": "1.0",
    "CreatedBy": "XiaoGang",
    "Category": "WebSocket",
    "Description": "XiaoGang Sample WebSocket Message",
    "Plugins": [
        { "Name": "WebSocketNetworking", "Enabled": true }
    ],
    "Modules": [
        {
            "Name": "XGSampleWSM",
            "Type": "Runtime",
            "LoadingPhase": "Default"
        }
    ]
}
```

> 依赖 `WebSocketNetworking` 插件以使用 `IWebSocketServer`。

### Build.cs 配置

```csharp
using UnrealBuildTool;

public class XGSampleWSM : ModuleRules
{
    public XGSampleWSM(ReadOnlyTargetRules Target) : base(Target)
    {
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        PublicDependencyModuleNames.AddRange(new string[] {
            "Core",
            "WebSockets",       // 客户端 IWebSocket
            "WebSocketNetworking", // 服务端 IWebSocketServer
            "Json",             // JSON 序列化/反序列化
            "JsonUtilities",
        });

        // 服务端模块限制：仅限 Windows/Mac/Linux
        if (Target.Platform == UnrealTargetPlatform.Win64 ||
            Target.Platform == UnrealTargetPlatform.Mac ||
            Target.Platform == UnrealTargetPlatform.Linux)
        {
            // Server 相关代码在此平台可用
        }
    }
}
```

| 依赖模块 | 用途 |
|---------|------|
| `WebSockets` | 客户端 `IWebSocket` 创建和通信 |
| `WebSocketNetworking` | 服务端 `IWebSocketServer` 创建和通信 |
| `Json` / `JsonUtilities` | `FJsonObjectConverter` JSON 序列化 |

## UGameInstanceSubsystem 模式

服务器端和客户端均采用 `UGameInstanceSubsystem` 作为生命周期容器。

### 优势

| 特性 | 说明 |
|------|------|
| 生命周期 | 随 GameInstance 创建/销毁自动管理 |
| 蓝图访问 | 通过 `GetGameInstanceSubsystem` 蓝图节点直接调用 |
| 单例保证 | 每个 GameInstance 只有一个实例 |
| Tick 支持 | 实现 `FTickableGameObject` 接口获得 Tick 能力 |

### 服务端子系统的 Tick 实现

```cpp
// XGWSMServerSubsystem 继承 FTickableGameObject
void UXGWSMServerSubsystem::Tick(float DeltaTime)
{
    TickServerConnections(DeltaTime);   // 遍历容器，调用每个连接的 Tick
}

bool UXGWSMServerSubsystem::IsTickable() const
{
    return !IsTemplate();               // CDO 不 Tick
}

TStatId UXGWSMServerSubsystem::GetStatId() const
{
    RETURN_QUICK_DECLARE_CYCLE_STAT(UXGWSMServerSubsystem, STATGROUP_Tickables);
}
```

### 关键设计决策

```
为什么服务器需要手动 Tick？

IWebSocketServer 没有自动的内部 Tick 循环机制。
必须由外部（Subsystem）定期调用才能：
1. 处理新连接的 Accept
2. 触发已连接客户端的回调
3. 执行各连接的心跳超时检测

这与 IWebSocket（客户端）不同——客户端 WebSocket
引擎内部已经处理了这些轮询。
```

## 连接状态枚举

```cpp
UENUM(BlueprintType)
enum class EXGWSMConnectionStatus : uint8
{
    None                UMETA(DisplayName = "None"),
    InitServerConnection UMETA(DisplayName = "InitServerConnection"),
    Tick                UMETA(DisplayName = "Tick"),
    PrepareToQuit       UMETA(DisplayName = "PrepareToQuit"),
    Quit                UMETA(DisplayName = "Quit"),
    OutOfTime           UMETA(DisplayName = "OutOfTime"),

```

服务器端使用 `WaitClientToInit` / `Tick` / `PrepareToQuit` / `Quit` / `OutOfTime` 五态状态。

## 三模块拆分计划（概念）

课程在字幕 015 中探讨了将单模块拆分为三模块的设计方案，当前代码尚未实施，作为后续重构方向记录：

```
XGSampleWSM（当前：单模块）
  ├── 所有代码在同一个模块中
  ├── 平台白名单在 Build.cs 的条件编译中区分
  └── 问题：WebSocketNetworking 不支持移动端，导致移动端无法编译

XGWSM（计划：三模块）
  ├── XGWSMMessage（公共消息模块）
  │   ├── Type/（枚举、结构体）
  │   ├── Util/（Base64 编解码）
  │   └── 无平台限制（所有平台可用）
  │
  ├── XGWSMClient（客户端模块）
  │   ├── 依赖 XGWSMMessage
  │   ├── ClientConnection + ClientSubsystem
  │   ├── 使用标准 WebSockets 模块
  │   └── 无平台限制（手机也可用）
  │
  └── XGWSMServer（服务端模块）
      ├── 依赖 XGWSMMessage
      ├── ServerConnection + ServerSubsystem
      ├── 使用 WebSocketNetworking 插件
      └── 平台限制：仅 Windows/Mac/Linux
```

| 拆分维度 | 当前（单模块） | 计划（三模块） |
|---------|--------------|--------------|
| 编译时平台过滤 | `#if` 或条件编译 | 模块级平台白名单 |
| 移动端支持 | 不可编译 | Client 模块可用 |
| 依赖关系 | 强耦合 | 消息类型共享 |
