# 消息协议与编解码

## 消息结构体

所有通信消息使用 `FXGWSMMessage` 结构体定义，通过 `FJsonObjectConverter` 序列化为 JSON 字符串传输。

```cpp
// Type/XGWSMType.h
USTRUCT()
struct FXGWSMMessage
{
    GENERATED_USTRUCT_BODY()

    UPROPERTY()
    int32 Code = -1;

    UPROPERTY()
    EXGWSMActionType Action = EXGWSMActionType::None;

    UPROPERTY()
    FGuid ClientConnectionID;

    UPROPERTY()
    FGuid ServerConnectionID;

    UPROPERTY()
    FString Description = TEXT("");

    UPROPERTY()
    FString Data = TEXT("");
};
```

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `Code` | `-1` | 业务状态码；0 表示正常，非 0 表示异常 |
| `Action` | `None` | 消息动作类型，决定消息目的和处理逻辑 |
| `ClientConnectionID` | 空 GUID | 客户端连接唯一标识 |
| `ServerConnectionID` | 空 GUID | 服务端连接唯一标识 |
| `Description` | 空字符串 | 描述文本 |
| `Data` | 空字符串 | JSON 字符串载荷，可反序列化为不同类型的业务数据结构 |

### 传输编码

传输原始数据为 `TArray<uint8>`，实际内容为 JSON 字符串的 Base64 编码：

```
发送端：FString(JSON) → UTF-8 → FBase64::Encode → uint8[] → IWebSocket::Send
接收端：uint8[] → Base64字符拼接 → FBase64::Decode → UTF-8 → FString(JSON) → FJsonObjectConverter → FXGWSMMessage
```

## ActionType 枚举

```cpp
// Type/XGWSMType.h
UENUM()
enum class EXGWSMActionType : uint8
{
    None,                       // 无操作

    ClientCallServerInit,       // 客户端→服务端：初始化连接
    ServerResponseClientInit,   // 服务端→客户端：确认初始化成功

    ClientCallTick,             // 客户端→服务端：心跳
    ServerResponseTick,         // 服务端→客户端：心跳响应

    ClientMessageToServer,      // 客户端→服务端：发送业务消息
    ServerMessageToClient,      // 服务端→客户端：转发消息

    ClientReqQuit,              // 客户端→服务端：请求断开
    ServerAllowClientQuit,      // 服务端→客户端：允许客户端断开

    ServerReqQuit,              // 服务端→客户端：请求断开
    ClientAllowServerQuit,      // 客户端→服务端：允许服务端断开

    Max,
};
```

### 消息动作与方向

| ActionType | 值 | 方向 | 阶段 | 说明 |
|-----------|----|------|------|------|
| `None` | 0 | - | - | 无操作 |
| `ClientCallServerInit` | 1 | client→server | 初始化 | WebSocket 建立后客户端发送初始请求 |
| `ServerResponseClientInit` | 2 | server→client | 初始化 | 服务端确认连接，配对 GUID |
| `ClientCallTick` | 3 | client→server | 心跳 | 客户端每 3s 发送一次心跳 |
| `ServerResponseTick` | 4 | server→client | 心跳 | 服务端心跳响应，客户端据此重置累计时间 |
| `ClientMessageToServer` | 5 | client→server | 消息 | 客户端发送业务消息到服务端 |
| `ServerMessageToClient` | 6 | server→client | 消息 | 服务端转发消息到客户端 |
| `ClientReqQuit` | 7 | client→server | 断开 | 客户端主动请求断开 |
| `ServerAllowClientQuit` | 8 | server→client | 断开 | 服务端同意客户端断开 |
| `ServerReqQuit` | 9 | server→client | 断开 | 服务端请求断开连接 |
| `ClientAllowServerQuit` | 10 | client→server | 断开 | 客户端同意服务端断开 |

### 服务端处理的消息子集

服务端只处理客户端发起的 ActionType：

```
ClientCallServerInit    (1) → 初始化连接
ClientCallTick          (3) → 心跳
ClientMessageToServer   (5) → 业务消息
ClientReqQuit           (7) → 客户端请求断开
ClientAllowServerQuit  (10) → 允许服务端断开
```

## 消息处理的安全校验

服务端收到消息后的解码与校验流程：

```
收到原始 uint8[] 数据
  │
  ├── Base64 解码 → FString(JSON)
  │
  ├── FJsonObjectConverter::JsonObjectStringToUStruct<FXGWSMMessage>
  │   → 返回 bool bParse（是否成功解析）
  │
  ├── 校验：
  │   if (!bParse || (bParse && Message.Code != 0))
  │   {
  │       // 解析失败 或 Code != 0 → 断开连接
  │       UnLinkPointedServerConnection(InTargetServerConnectionID);
  │       return;
  │   }
  │
  └── 校验通过 → 按 ActionType 分发处理
```

> 关键安全策略：即使用 JSON 解析成功，只要 `Code != 0` 就视为异常消息，直接断开连接。这是防御性编程——上层错误不应被下层继续处理。

## Base64 编解码

编解码操作位于 `Util/XGWSMUtil.h/.cpp`，作为蓝图函数库的静态方法暴露。

### XGMessageEncode（编码 → 发送）

```cpp
TArray<uint8> UXGWSMUtilBPLibrary::XGMessageEncode(const FString& InMessage)
```

编码流程：
```
FString（UE 内部 UTF-16）
  → FTCHARToUTF8（转换为 UTF-8 多字节）
  → FBase64::Encode（UTF-8 字节数组 → Base64 字符串）
  → 逐字符将 Base64 字符串转化为 uint8 存入 TArray<uint8>
  → 返回 TArray<uint8>（用于 IWebSocket::Send）
```

### 两种解码入口

| 解码函数 | 参数 | 使用场景 |
|---------|------|---------|
| `XGMessageDecode(TArray<uint8>&)` | 累积缓冲区 | 碎片数据重组后解码 |
| `XGMessageDecode(const uint8*, int32)` | 原始指针 + 长度 | OnBinaryMessage 首次到达直接解码 |

两个函数本质相同：将 `uint8[]` 逐字节拼接为 Base64 字符串，`FBase64::Decode` 解码为 UTF-8 字节，再 `FUTF8ToTCHAR` 转为 FString。

### 关键细节

| 细节 | 说明 |
|------|------|
| 为什么逐字节转换 | `FBase64::Encode` 返回 `FString`，需手动转为 `TArray<uint8>` 用于二进制发送 |
| 编码标准 | UE 内置 `FBase64` 模块，符合标准 Base64 |
| 异常处理 | `Code != 0` 时直接断开，不解包 `Data` 字段 |
