# 独立程序暴力打包

## 概述

独立程序打包没有 UAT（Unreal Automation Tool）的标准流程支持（不像游戏项目有 Project Launcher）。本章展示了一种**手动暴力打包法**——将引擎编译产物中必要的部分拷贝出来，通过逐文件排除法精简到最小依赖。

## 暴力打包思想

```
1. 从引擎编译目录拷贝全部产物
2. 逐个删除文件夹/文件
3. 每次删除后启动程序，测试是否正常
4. 记录"删了就会崩溃"的最小依赖集合
5. 保留最小依赖，其余全部删除
```

这种方法**非常耗时**（每次删除+验证可能 20 分钟~1 小时），但它是目前最可靠的手动打包独立程序方式。

## 打包产物目录结构

```
XG Slate Programs/
└── Engine/
    ├── Binaries/
    │   └── Win64/              ← 程序 DLL/EXE（仅保留 Win64）
    ├── Config/                 ← 引擎配置（可大量精简）
    ├── Content/                ← 必需资源
    │   ├── Slate/              ← Slate 图标/样式资源
    │   └── EditorSlate/        ← 编辑器 Slate 图标
    ├── Plugins/                ← 插件（可大量精简）
    ├── Shaders/                ← Shader 资源
    └── ThirdParty/             ← 第三方库
```

## 必须保留的依赖清单（UE 5.4.2）

### 基础依赖
- `Engine/Binaries/Win64/` — 程序可执行文件和 DLL
- `Engine/Config/` — 配置文件（可精简）

### Slate/UI 资源
- `Engine/Content/Slate/` — Slate 图标、样式资源
- `Engine/Content/EditorSlate/` — 编辑器图标（如缺，Slate 图标会找不到）

### Shader
- `Engine/Shaders/` — Standard Renderer Shader（SlateViewer 依赖）

### 第三方库
- **`Engine/ThirdParty/CEF3/`**（UE 5.4 新增依赖）— Chromium Embedded Framework，浏览器/WebBrowser 相关，体积很大。SlateViewer 在 UE 5.4 依赖此项（可能是为像素流 PixelStreaming 引入）

### 国际化
- `Engine/Content/Localization/` — 国际化和文本语言资源

## 可安全删除的内容

| 目录 | 说明 |
|------|------|
| `Engine/Binaries/` 下其他平台目录 | Linux/Mac/Android 等 |
| `Engine/Content/Movies/` | 启动动画 |
| `Engine/Content/Slate/` 中非图标资源 | 可进一步精简 |
| `Engine/Plugins/` 中不相关的插件 | 如仅保留核心 Slate 插件 |
| `Engine/Source/` | 源码不需要 |
| `Engine/Intermediate/` | 中间文件 |

## 版本差异

| 版本 | CEF3 依赖 | 说明 |
|------|----------|------|
| UE 5.0.3 | 不需要 | SlateViewer 模板约 100MB+ |
| UE 5.4.2 | **需要** | 新增 CEF3 依赖，体积显著增大 |

## 注意事项

- 打包后的程序依赖 VC++ 运行时（需目标机器已安装）
- Debug 模式依赖 PDB 符号文件
- 最低目标机器可能是 Windows 10+
- 打包产物仍然较大（通常 100~200MB），可进一步用 7z/LZMA 压缩
