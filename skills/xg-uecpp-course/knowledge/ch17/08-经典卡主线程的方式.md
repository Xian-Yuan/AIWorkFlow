# 经典卡主线程的方式

## 概述

UE 的 TaskGraph 线程池被**引擎内部系统**和**用户代码**共享。如果用户代码长时间占用 TaskGraph 工作线程，会导致引擎的其他系统（物理、音频、动画等）无法获得执行线程，造成全局卡顿甚至看起来像是"主线程被卡住"。本章总结三种经典的阻塞方式及解决方案。

## 根本原因

TaskGraph 线程池的大小是有限的（通常等于 CPU 核心数）。以下系统共享同一个线程池：

```
         ┌─────────────────────────────┐
         │      TaskGraph 线程池        │
         │  (4-8 个工作线程)            │
         ├─────────────────────────────┤
         │  用户 AsyncTask              │
         │  用户 ParallelFor            │
         │  FGraphEvent 任务             │
         │  FAsyncTask                  │
         │  物理引擎                    │
         │  音频系统                    │
         │  动画更新                    │
         └─────────────────────────────┘
```

当用户代码占满所有线程时，所有系统都需要等待。

## 方式一：FAsyncTask + TaskGraph 滥用

```cpp
// 阻塞方式 — 连续创建 100 个长时间任务
void UXGThreadSubsystem::BlockThreadPool()
{
    for (size_t i = 0; i < 100; i++)
    {
        AsyncTask(ENamedThreads::AnyThread, []() {
            FPlatformProcess::Sleep(2);  // 每个任务阻塞 2 秒
        });
    }
}
```

后果：
- 100 个任务全部提交到 TaskGraph 线程池
- 工作线程被全部占满（每个线程 Sleep 2 秒）
- 引擎物理/音频/动画系统无法获取工作线程
- 整个引擎卡顿

"你把整个线程池都干掉了" —— 课程原话

## 方式二：ParallelFor + 长时间任务

```cpp
// 阻塞方式 — ParallelFor 在 TaskGraph 执行长时间任务
void UXGThreadSubsystem::BlockThreadPool2()
{
    ParallelFor(100, [](int32 Index) {
        UE_LOG(LogTemp, Display, TEXT("ParallelFor:%d"), Index);
        FPlatformProcess::Sleep(2);  // 每个迭代阻塞 2 秒
    });
}
```

后果：
- ParallelFor 在 TaskGraph 线程池执行
- 每个迭代 Sleep 2 秒
- 工作线程被全部占满

## 方式三：Async + TaskGraph 嵌套

```cpp
// 阻塞方式 — AsyncTask(AnyThread) + ParallelFor 嵌套
void UXGThreadSubsystem::BlockThreadPool3()
{
    AsyncTask(ENamedThreads::AnyThread, []() {
        // 在 AnyThread 中再开 ParallelFor
        ParallelFor(100, [](int32 Index) {
            UE_LOG(LogTemp, Display, TEXT("ParallelFor:%d"), Index);
            FPlatformProcess::Sleep(2);
        });
    });
}
```

这种方式同样会卡住线程池，因为 ParallelFor 最终也是在 TaskGraph 工作线程中执行。

## 解决方案

### 方案 A：使用 FRunnable 创建独立线程

FRunnable 创建的线程是**操作系统独立线程**，不占用 TaskGraph 线程池：

```cpp
// ✅ 正确：使用独立线程
XGSimpleRunnable = MakeShared<FXGSimpleRunnable>(TEXT("XGThread000"));
XGSimpleThreadPtr = FRunnableThread::Create(XGSimpleRunnable.Get(), *XGSimpleRunnable->GetThreadName());
```

适用于：需要长期运行、持续工作的后台任务。

### 方案 B：使用 Async(EAsyncExecution::Thread)

```cpp
// ✅ 正确：使用独立线程
for (int32 i = 0; i < 100; i++)
{
    Async(EAsyncExecution::Thread, [i]() {
        FPlatformProcess::Sleep(2.0f);  // 独立线程，不阻塞 TaskGraph
    });
}
```

适用于：少量、长时间运行的独立任务。

### 方案 C：自己实现线程池

对于需要批量管理线程的场景，可以实现自己的线程池，避免占用引擎的共享线程池。

## 判断标准

| 场景 | 是否阻塞 TaskGraph | 推荐方案 |
|------|-------------------|----------|
| 短期小任务（< 10ms） | 否 | AsyncTask / ParallelFor |
| 长期运行任务 | 是 | FRunnable 独立线程 |
| 大量并发长时间任务 | 是 | 自定义线程池 |
| 需要获取计算结果 | 取决于实现 | Async(EAsyncExecution::Thread) + TFuture |
