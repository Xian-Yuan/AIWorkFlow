# 商城指南文档关键规则

## 概述

Epic 官方提供了详细的 Marketplace Guidelines 文档，规定了所有上架商品必须遵守的内容政策、图片规则、代码规范和文件命名约定。提交前必须逐条核对。

---

## 内容政策（Content Policies）

- 插件内容不得违反 Epic 内容政策
- 不得包含色情、暴力、政治敏感内容
- 不得包含恶意代码、后门、数据收集功能
- 必须使用最新的引擎版本兼容性测试

---

## 图片规则

- 截图必须是**纯 UE 引擎截图**，不含外部软件的 UI
- 不能包含 Windows 任务栏、桌面、浏览器等操作系统 UI 元素
- 所有图片中的文字必须使用英文
- 图片需清晰展示插件功能，不要过度修饰
- 主图尺寸：1920×1080

---

## EngineVersion 字段

- 位于 `.uplugin` 文件中
- **不要手动填写**，Apex Launcher 打包时自动生成
- 必须确保 `EngineVersion` 与实际打包所用的引擎版本一致

---

## 文件版权声明（Copyright Notice）

这是初学者最常犯的错误，**99% 的首次提交者会忘记**。

### 规则要求

- `.h`、`.cpp`、`.cs` 文件的**头部必须包含版权声明**
- 版权声明格式需符合 Epic 规范或开发者自有版权格式
- `.uplugin` 文件**不允许包含注释**（`//` 或 `/* */`），JSON 解析器会失败

### 常见错误

```
// 错误示例：在 .uplugin 中写注释
{
    "FileVersion": 3,
    // "EngineVersion": "5.3"  // ← 会导致 JSON 解析失败
}
```

---

## IWYU（Include What You Use）

- 每个源文件只包含它实际需要的头文件
- 不得包含未使用的头文件
- 减少不必要的依赖，加快编译速度
- 商城审核会检查代码质量，IWYU 是评审标准之一

---

## 文件/目录命名规范

- 目录名和文件名**不能包含空格**
- 建议使用 `PascalCase` 或 `camelCase`
- 命名要有语义，反映文件内容
- 避免使用特殊字符（`@`、`#`、`$`、`%` 等）

---

## 第三方库归属要求

使用第三方库时必须：

1. 提交 **Third-Party Software Attribution Form**
2. 列出使用的每个第三方库
3. 包含以下信息：
   - 库名称
   - 开源协议（如 MIT、Apache 2.0、BSD 等）
   - 官方项目地址（URL）
   - 作者/维护者联系方式
4. **Openssl 被明确标记为风险库**，使用它可能导致审核失败

---

## 技术支持要求

- 必须提供**合理的技术支持**渠道
- 回复用户问题的时间不应过长
- 如果插件有严重 Bug 需要及时修复更新
- 商城对长期不维护的插件有下架处理机制

---

## 常见审核拒绝原因总结

| 原因 | 说明 |
|------|------|
| 图片/描述含非英文内容 | 首次提交必须全英文 |
| 缺少版权声明 | 99% 首次提交者遗漏 |
| 标题含商标名 | 不能包含 UE、Apex、国家名称等 |
| 第三方库未声明 | 使用 Openssl 等库未填写归属表单 |
| `.uplugin` 含注释 | JSON 标准不允许注释 |
| 代码质量不达标 | IWYU 不合规、命名不规范等 |
