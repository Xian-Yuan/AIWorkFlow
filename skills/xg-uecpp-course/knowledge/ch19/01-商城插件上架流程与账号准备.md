# 商城插件上架流程与账号准备

## 概述

将 UE 插件发布到虚幻商城，需要完成 Apex 卖家账号注册、Hyperwallet/PayPal 收款设置、理解插件放置位置与类型区别、掌握 Apex Launcher 打包流程。

---

## Apex 卖家账号注册

| 步骤 | 说明 |
|------|------|
| 注册 Apex 账号 | 访问 Apex 官网（apex.run），注册开发者账号 |
| 关联 Epic Games 账号 | 在 Apex 后台关联已有的 Epic Games 开发者账号 |
| 申请 Marketplace Merchant | 在 Apex 卖家后台（Seller Dashboard）提交 Marketplace Merchant 申请 |
| 设置收款方式 | 绑定 Hyperwallet 或 PayPal |

### Hyperwallet / PayPal 收款设置

- 中国开发者推荐使用 Hyperwallet
- 绑定银行卡时建议使用**招商银行一卡通（储蓄卡）**，支持 Swift Code 国际汇款
- PayPal 可作为备选方案

## 插件放置位置

| 位置 | 路径示例 | 适用场景 |
|------|---------|---------|
| 项目 Plugins 目录 | `项目根目录/Plugins/插件名/` | 自写项目插件 |
| 引擎 Plugins 目录 | `引擎根目录/Engine/Plugins/Marketplace/插件名/` | 商城下载安装的插件 |

- 商城下载的插件自动安装到引擎 Plugins 目录下的 `Marketplace/` 子目录
- 自己的项目插件放在项目 `Plugins/` 目录

---

## 插件类型

| 类型 | 说明 |
|------|------|
| **Source Code 插件** | 源码可见，可在编辑器中编译，开发者可修改 |
| **Binary（DLL）插件** | 预编译的二进制文件，源码不可见，不可修改 |

- 商城提交时根据是否包含源码选择对应类型
- Binary 插件适合不想公开源代码的场景

## Apex Launcher 卖家后台

### 打包方式

| 方式 | 说明 |
|------|------|
| **Package Plugin** | 仅打包插件本身，不包含项目 |
| **Package Project** | 打包整个项目（含插件），用于验证插件在打包后项目中能正常工作 |

- 正式提交前，两种打包方式都应测试
- 打包后还需运行 `Run Packaged Build` 验证打包结果

### EngineVersion 自动生成

- 不要手动编写 `EngineVersion` 字段
- Apex Launcher 在打包时会**自动生成**对应的引擎版本号
- 手动填写可能导致版本号与打包产物不匹配

### MarketplaceURL

- 插件创建后，商城会分配唯一的商品页面 URL
- 将此 URL 填入 `.uplugin` 文件的 `MarketplaceURL` 字段
- 用户从商城下载后可通过此链接访问商品详情页

---

## Seller Dashboard 操作流程

1. 登录 Apex 卖家后台
2. 创建新产品 → 选择 "Plugin"
3. 上传打包产物
4. 配置商品详情（标题、描述、图片、定价、分类）
5. 设置版本支持范围
6. 提交审核
