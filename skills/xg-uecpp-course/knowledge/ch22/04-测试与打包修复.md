# 展示篇测试与打包修复

## 概述

对动态 WebP 加载展示进行完整流程测试，验证从本地 `.webp` 文件 → 解码 → 纹理创建 → Tick 逐帧切换 → 场景显示的全链路。

## 测试流程

### 1. 准备测试素材

- 使用第 21 章生成的 `.webp` 动态图作为测试素材
- 确保 `.webp` 文件路径可访问

### 2. 蓝图测试

1. 在关卡蓝图中绑定键盘事件
2. 获取 `UXGSampleWebpShowMultiSubsystem`
3. 调用 `LoadWebp` 传入文件路径和完成回调
4. 回调中接收 `UTexture2D`，创建 Material Instance Dynamic，应用到场景中的 Plane
5. 观察纹理是否逐帧切换

### 3. 场景准备

- 放置一个 `Plane` Actor 作为展示面
- 准备动态材质，包含 `Texture2D` 参数
- 可放置旋转物体（Cube）观察动画效果

## 常见问题

### 问题 1：加载后纹理不更新

**原因**：
- `Tick()` 未被调用 → 检查 `FTickableGameObject` 的三个必须方法是否实现完整
- `IsTickable()` 返回了 `false` → 检查 `LoadAndShowStatus` 状态

### 问题 2：纹理更新时闪烁或不正确

**原因**：
- `UpdateTextureRegions` 的 `SrcPitch` 计算错误（应为 `Width × 4`）
- 像素格式不匹配（`FColor` 应为 `PF_B8G8R8A8`）
- 未持有 `UTexture2D*` 指针导致被 GC

### 问题 3：打包后加载失败

**原因**：
- `.webp` 文件未被打包进游戏 → 检查文件是否在 Content 目录或使用绝对路径
- 第三方库在 Shipping 配置下链接失败 → 确认 `.lib` 文件包含所有构建配置

## 完美方案 vs 简化方案

### 简化方案（本章采用）

- 每次 Tick 通过 `UpdateTextureRegions` 更新纹理
- 适用于少量帧（<100 帧）
- 优点：实现简单，无需深入理解 UE 媒体框架

### 完美方案（需独立研究）

- 基于 `IMediaPlayer` / `FMediaIOCapture` 框架
- 支持音视频同步、推拉流
- 需要深入理解 UE 底层媒体管线、时间戳同步、缓冲区管理
- 适用于专业音视频应用开发
