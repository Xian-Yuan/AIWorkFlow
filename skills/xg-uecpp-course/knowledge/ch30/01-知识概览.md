# 第三十章：WebSocket 语音合成（TTS）知识概览

## 章节说明

本章基于 WebSocket 协议实现**文本转语音（TTS/Text to Speech）** 功能。与第二十九章（STT 语音识别）刚好相反的数据流向——读入文本，输出音频。

| 对比维度 | 第二十九章（STT） | 第三十章（TTS） |
|---------|-----------------|----------------|
| 数据方向 | 麦克风 → PCM → 服务器 | 文本 → 服务器 → 音频数据 |
| 节点类型 | UGameInstanceSubsystem + BPLibrary | UBlueprintAsyncActionBase |
| 鉴权 | HMAC-SHA1（URL 参数） | HMAC-SHA256（AssembleAuthUrl） |
| WebSocket URL | `ws://rtasr.xfyun.cn/v1/ws` | `ws://tts-api.xfyun.cn/v2/tts` |
| 音频处理 | float→int16→uint8 发送 | Base64 解码 → USoundWave |
| 输出 | 文本（识别结果） | 音频（播放 + WAV 文件） |

## 整体 Pipeline

```
蓝图调用 → XGXunFeiTextToSpeech
   ↓ Activate
Create WebSocket（ws://tts-api.xfyun.cn/v2/tts）
   ↓ OnConnected
发送 JSON（包含 Base64 编码的文本）
   ↓ OnMessage（多次触发）
解析 JSON → 检查 code == 0
   ↓
Base64 解码 → 累积音频数据到 TArray<uint8>
   ↓ status == 2（最后一帧）
┌──────────────────────────────────────┐
├─ 创建 USoundWave（内存播放）          │
├─ 保存 .wav 文件到磁盘（可选）         │
└──────────────────────────────────────┘
   ↓
回调蓝图（OnSoundWaveSuccess / OnSoundWaveFail / WavFileSuccess / WavFileFail）
```

## 边界说明

- 本章处理**TTS 语音合成**，与第二十九章 STT 处理同一插件的不同子系统
- TTS 采用 UBlueprintAsyncActionBase 模式（与第二十七章 HTTP 节点相同）
- 输出通道：内存播放（USoundWaveProcedural）和/或 WAV 文件存储
- 文本编码：FString（UTF-16）→ UTF-8 → Base64 传递给服务器
