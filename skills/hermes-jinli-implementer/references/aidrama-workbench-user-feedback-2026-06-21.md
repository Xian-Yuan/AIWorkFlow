# AIDramaProducer 前端工作流改进需求（爸爸原始反馈）

**来源**: Codex 会话 `019ee423` (2026-06-20 ~ 2026-06-21)
**上下文**: Preproduction Workbench UI 实现完成后，爸爸试用后的完整反馈

## 核心定位问题

爸爸明确指出：**这个工具的产出，是一套可以在生图网站生成的分镜图和视频生成网站的提示词**。当前实现偏离了这个定位。

## 完整改进清单

### 1. 语言与交互
- 所有内容必须中文，当前中间对话区域仍是英文
- 产出 JSON 需要一键复制按钮，不能只下载文件
- 中间对话区应接入 AI 对话：用户输入自定义内容 → AI 思考后回答

### 2. 视频分析 → 模板抽象流程
- 分析抖音/B站等主流视频链接
- 本地模型提取内容 → API 聪明模型做结构/转折/分镜/导演/编剧分析
- 抽象总结成模板，不同视频类型整理不同模板
- 用户可以在基础模板上创作，或让 AI 在模板基础上创作再修改

### 3. 提示词产出目标（关键）
- **只需要两个产出**：
  1. 分镜提示词 → 生成分镜图像（给生图网站）
  2. 视频生成提示词 → 包含视觉设定等，和分镜图像一起给视频生成模型
- 提示词不能太分散，用户不想拼接
- 提示词角色定义太多是否有问题？参考字节/腾讯/B站的创作平台

### 4. 左侧创作流程
- 点击哪一步就跳转到哪一步
- 支持重新编辑、增加设置
- 可选供应商、配置 API Key

### 5. 布局改进
- 右侧创意简报无效 → 与左侧创作流程合并
- 左侧策略下放子栏展示创作内容
- 提示词区域太小 → 可拖动边框
- 参考 ByteDance/Tencent/Bilibili 的创作平台

### 6. 提示词质量（关键批评）
- 当前提示词太空泛，问问题没问到点子上
- 例：说"喜剧"类型 → 应追问：漫才？小品？相声？什么形式？
- 需要更具体：笑点类型、整体节奏、多少秒一个包袱、多少秒一个反转、整体结构
- AI 应联网搜梗/热点/爆款视频，提供灵感，与用户对话共创

## 参考方向

爸爸要求先搜索了解成熟的 AI 漫剧工作流：
- 别人的提示词基础模板
- 提示词格式和约束规范
- 字节/腾讯/Bilibili 创作平台的分镜和视频提示词生成功能

## 对未来规划的影响

这些反馈将作为下一轮 AIDramaProducer 前端工作流 redesign 的核心需求输入。任何相关规划任务都应先读取此文件。
