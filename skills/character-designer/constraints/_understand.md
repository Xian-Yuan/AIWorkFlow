# 理解增强规则

> **元认知层**：指导 AI 如何深入理解约束，而非约束本身。
> 在约束加载后、设计确认前按 L1→L2→L3 顺序执行。
> 与 `_rules.md`（冲突解决）互补——`_understand.md` 处理"如何理解"，`_rules.md` 处理"冲突怎么办"。

---

## L1: 语义具象化

### 目的

每个 MUST 从抽象描述变为具体画面。

### 规则

1. 加载约束文件后，逐条读取 MUST
2. **量化值**：直接使用，取中间偏安全值（如 `≥60%` → 约 `65%`）
3. **抽象描述**：必须具象化。判断是否多义：
   - 单义 → 直接具象化
   - 多义 → 列出 2-3 个具体方向，结合已有特征选最优，**必要时追问用户**

### 具象化示例

| 抽象 MUST | 单义具象化 | 多义方向 |
|-----------|-----------|---------|
| 服装以轻便灵活为主 | 布料+轻皮甲，无金属片 | — |
| 至少有1处不对称设计 | — | ①披风单侧(左肩有右肩无) ②腰带偏斜 ③肩甲左大右小 ④手套不对称 |
| 剑身笔直 | 直线双刃，无弯曲 | — |
| 身体至少有1处机械改造 | — | ①义肢(手臂/腿) ②数据接口(后颈/太阳穴) ③植入物(眼球/芯片) ④外骨骼 |
| 火焰必须有动势 | 向上燃烧，非静止 | — |
| 面部覆盖≥50% | — | ①下半脸面罩 ②上半脸兜帽 ③全脸面具(只露眼) |

### 多义时的选择逻辑

```
列出方向 → 查已选特征：
  → 如果有强关联（如已选"暗影力量"→面罩优先遮上半脸）
  → 如果没有明显偏好 → 列出 2-3 选项，让用户选
```

## L2: 交叉理解

### 目的

每加载一个约束后，推理它如何改变已有约束的视觉表达。

### 规则

```
新约束 {B} 加载后 → 对每个已加载约束 {A} 做交叉理解：

1. 查 SYNERGY 段（已加载文件的 ## SYNERGY）
   → 有则直接用

2. 查 _rules.md 冲突表
   → 有冲突则按规则解决

3. 都不匹配 → AI 自己推理一个合理的交互效果
   → 用 1 句话说明
```

### 输出格式

```
[A+B] 交互效果描述（1句话）
```

### 示例

```
已选: assassin (MUST:轻装/暗色, FORBID:暖色/重甲, SYNERGY:暗影+匕首)
已选: 7头身 (MUST:腿长≥躯干×1.3, 腰线清晰, SYNERGY:无)

新选: fire (MUST:火焰可视化/暖色, FORBID:冷色火焰, SYNERGY:火焰元素同源)
  → 步骤1: 查 SYNERGY → assassin 无 fire 协同
  → 步骤2: 查 _rules.md → R20(刺客+暖色→降为暗暖色, ≤30%)
  → 步骤3: AI推理 → 火焰调整为暗焰形态(深红+暗金)，出现在武器刃口/手部，30%占比，不破坏潜行感
  → 输出: assassin+fire → 暗焰形态，深红暗金，武器刃口/手部微光，30%占比

新选: cold (MUST:站姿直立, 眼神不闪躲, FORBID:笑容/过度装饰, SYNERGY:暗色系)
  → 步骤1: 查 SYNERGY → cold+暗色系 有协同
  → 步骤2: cold(直立)+assassin(前倾) → 无冲突规则但姿势矛盾
  → 步骤3: AI推理 → 直立但微前倾(身体直立、头部微低、眼神平视)，既保持冷酷又蓄势
  → 输出: [cross] assassin+cold → 身体直立+头部微前倾+眼神平视=冷峻蓄势姿态
```

## L3: 视觉转译

### 目的

将抽象约束描述转为**画面语言**——有具体位置、形态、比例。

### 转译公式

```
有什么(What) + 在哪里(Where) + 怎么样(How)
```

### 转译对照表

| 抽象描述 | 画面语言 |
|---------|---------|
| 站姿前倾 | 重心落前脚掌，上身前倾≈15°，膝盖微屈 |
| 低重心 | 两脚分开≈肩宽，髋部下沉，小腿受力感 |
| 不对称披风 | 左肩覆披风至肘关节，右肩裸露/轻甲 |
| 剪影优先 | 远看轮廓可读：武器外轮廓+体型特征+头饰通用项 |
| 元素统一 | 主元素纹样/颜色出现在 ≥2 处(武器+衣摆/护甲+披风) |
| 面罩≥50% | 下半脸覆面罩，鼻梁以下遮挡，露眼 |
| 火焰动势 | 向↑燃烧，底部深红→中部橙→顶部金，边缘不规则 |
| 直立挺拔 | 双肩等高，脊柱直线，下颌微收 |
| 反握匕首 | 刀尖朝下/后，握柄在拳中，手腕外翻 |
| 机械改造 | 机械部分与肉体交界处可见接口/接线/铆钉 |
| 轻盈感 | 衣摆/披风有飘动角度(≈15°)，布料薄可见身形轮廓 |
| 重量感 | 护甲边缘有厚度(≥2mm)，地面接触处有阴影/凹陷 |

### 转译时机

在生成提示词前，将所有抽象约束转译为画面语言，融入 prompt-compressor 输出。

## 执行流程图

```
约束加载
  │
  ▼
L1 语义具象化 ──→ 多义? ──→ 追问用户
  │ (单义/已具象)
  ▼
L2 交叉理解 ──→ 对每个已加载约束做融合推理
  │              SYNERGY → _rules.md → AI推理
  ▼
L3 视觉转译 ──→ 抽象→画面语言
  │
  ▼
进入下一步设计 / 冲突检测 / 提示词生成
```
