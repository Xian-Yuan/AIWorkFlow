# 模块配置与插件依赖

## 概述

ControlFlows 是 UE5 的实验性插件，不是引擎内置模块。使用前需要在 `.uproject` 文件中启用插件，并在目标模块的 `.Build.cs` 中添加模块依赖。

## 启用 ControlFlows 插件

### .uproject 配置

在项目的 `.uproject` 文件的 `Plugins` 数组中添加：

```json
{
    "Name": "ControlFlows",
    "Enabled": true
}
```

完整示例：

```json
{
    "FileVersion": 3,
    "EngineAssociation": "5.4",
    "Category": "",
    "Description": "",
    "Modules": [
        {
            "Name": "XGSampleDemo",
            "Type": "Runtime",
            "LoadingPhase": "Default"
        }
    ],
    "Plugins": [
        {
            "Name": "ControlFlows",
            "Enabled": true
        }
    ]
}
```

### .Build.cs 依赖

在目标模块的 `.Build.cs` 文件的 `PublicDependencyModuleNames` 中添加 `"ControlFlows"`：

```csharp
PublicDependencyModuleNames.AddRange(new string[] {
    "Core",
    "CoreUObject",
    "Engine",
    "InputCore",
    "ControlFlows"
});
```

## 插件说明

- ControlFlows 是 UE5 的实验性插件，源自 Lyra 项目
- 提供 `FControlFlow`、`FControlFlowNodeRef`、`FControlFlowStatics` 等核心类型
- 包含在引擎发布版中（非第三方插件），只需在项目配置中启用即可
- 相关头文件在引擎 `Source/Runtime/Experimental/ControlFlows/` 目录下

## 所需头文件

在代码中 `#include` 以下文件以使用 ControlFlows API：

```cpp
#include "ControlFlow.h"
#include "ControlFlowNode.h"     // FControlFlowNodeRef
#include "ControlFlowManager.h"  // FControlFlowStatics
```

此外，线程跳跃模式需要：

```cpp
#include "Async/Async.h"         // AsyncTask
```

文件持久化需要：

```cpp
#include "Misc/FileHelper.h"
#include "Misc/Paths.h"
```

## 验证配置

配置完成后，编译项目验证：

1. `#include "ControlFlow.h"` 是否可被正确引用
2. `FControlFlowStatics::Create` 是否可解析
3. `FControlFlowNodeRef` 是否为已知类型
