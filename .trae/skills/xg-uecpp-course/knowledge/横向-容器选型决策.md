# 容器选型决策

## 概述

UE 提供三种核心容器：TArray、TMap、TSet。本章覆盖第 4~6 章（TArray/TMap/TSet）的内容，将三个独立章节的容器知识整合为统一的选型框架。

## 三种容器选型决策

```
需要有序序列？
  ├── 是 → TArray
  └── 否 → 需要键值映射？
             ├── 是 → TMap
             └── 否 → 仅需要集合存在性？
                        ├── 是 → TSet
                        └── 否 → 是否已用 TArray？
                                  ├── 是 → TArray（转换开销<重写代码）
                                  └── 否 → TArray（默认容器）
```

## 核心特性对比

| 维度 | TArray | TMap | TSet |
|------|--------|------|------|
| 底层结构 | 动态数组（连续内存） | 哈希表 + 桶 | 哈希表 + 桶 |
| 元素形式 | 单一类型 | TPair<Key, Value> | 单一类型（元素即键） |
| 顺序 | 有序（插入序） | 无序 | 无序 |
| 随机访问 | O(1) 索引 | O(1) 键查值 | O(1) 元素查找 |
| 插入 | O(1) 末尾 / O(n) 中间 | O(1) 平均 | O(1) 平均 |
| 查找 | O(n) | O(1) 键 / O(n) 反向 | O(1) |
| 移除 | O(n) | O(1) 平均 | O(1) 平均 |
| 元素唯一性 | 否（AddUnique 可去重） | 键唯一 | 元素唯一 |
| 扩容行为 | 连续扩容（首次4，之后每次+16） | Rehash | Rehash |
| 内存连续性 | 连续 | 非连续 | 非连续 |
| 指针/引用稳定性 | 扩容时失效 | 插入/移除后可能失效 | 插入/移除后可能失效 |
| 使用频率 | 最高 | 中 | 低 |

## 增删改查 API 对比

### 添加元素

| 操作 | TArray | TMap | TSet |
|------|--------|------|------|
| 添加一个 | `Add(Val)` | `Add(Key, Val)` | `Add(Val)` |
| 原地构造 | `Emplace(Args...)` | `Emplace(Key, Val)` | `Emplace(Args...)` |
| 合并容器 | `Append(Other)` | `Append(Other)` | `Append(Other)` |
| 去重添加 | `AddUnique(Val)` | 默认唯一（键重复覆盖） | 默认唯一 |
| 默认构造 | `AddDefaulted()` | — | — |
| 指定位置 | `Insert(Val, Index)` | — | — |
| 指定位置构造 | `EmplaceAt(Index, Args...)` | — | — |

### 查询元素

| 操作 | TArray | TMap | TSet |
|------|--------|------|------|
| 存在性 | `Contains(Val)` | `Contains(Key)` | `Contains(Val)` |
| 查找索引/指针 | `Find(Val)` 返回索引 | `Find(Key)` 返回值指针 | `Find(Val)` 返回元素指针 |
| 获取引用 | `operator[]` | `operator[]`（键不存在时崩溃） | `operator[FSetElementId]` |
| 默认值 | — | `FindRef(Key)` 返回副本 | — |
| 不存在时创建 | — | `FindOrAdd(Key)` | — |
| 谓词查找 | `FindByPredicate(Pred)` | — | — |
| 反向查找 | — | `FindKey(Val)` O(n) | — |

### 移除元素

| 操作 | TArray | TMap | TSet |
|------|--------|------|------|
| 按值/键移除 | `Remove(Val)` | `Remove(Key)` | `Remove(Val)` |
| 按索引/ID 移除 | `RemoveAt(Index)` | — | `Remove(FSetElementId)` |
| 谓词移除 | `RemoveAll(Pred)` | — | — |
| 全部清除 | `Empty()` / `Reset()` | `Empty()` / `Reset()` | `Empty()` / `Reset()` |
| 快速移除（不保序） | `RemoveAtSwap(Index)` | — | — |

### 排序

| 操作 | TArray | TMap | TSet |
|------|--------|------|------|
| 不稳定排序 | `Sort()` / `QuickSort()` | `KeySort(Pred)` / `ValueSort(Pred)` | `Sort(Pred)` |
| 稳定排序 | `StableSort()` | — | `StableSort(Pred)` |
| 堆排序 | `HeapSort()` | — | — |

## 堆操作（TArray 独有）

| 操作 | 说明 |
|------|------|
| `Heapify()` | 将数组转换为最小堆 |
| `HeapPush(Val)` | 插入堆 |
| `HeapPop(TopOut)` | 移除堆顶并输出 |
| `HeapRemoveTop()` | 仅移除堆顶 |
| `HeapTop()` | 只读查看堆顶 |

| 操作 | TArray | TMap | TSet |

| 缩容 | `Shrink()` | `Shrink()` | `Shrink()` |

## 自定义类型要求

| 容器 | 需要重载 | 额外说明 |
|------|---------|---------|
| TArray | `operator==`（查找/比较） | 排序需要 `operator<` |
| TMap（键） | `operator==` + `GetTypeHash()` | 或自定义 KeyFuncs |
| TSet（元素） | `operator==` + `GetTypeHash()` | 或自定义 KeyFuncs |

### 自定义 KeyFuncs 对比

| 成员 | TMap | TSet |
|------|------|------|
| KeyInitType | 键类型 | 元素类型本身 |
| ElementInitType | TPair<Key, Value> | 元素类型本身 |
| GetSetKey | 从 TPair 中提取 Key | 返回元素本身 |
| Matches | 比较两个 key | 比较两个元素 |
| GetKeyHash | 计算 key 的哈希 | 计算元素的哈希 |

## 迭代方式对比

三种容器的迭代模式统一：

```cpp
// 方式一：for-range（推荐）
for (const auto& Elem : Array) { }
for (const auto& Elem : Map) { Elem.Key; Elem.Value; }
for (const auto& Elem : Set) { }

// 方式二：索引/迭代器
for (int32 i = 0; i < Arr.Num(); ++i) { }                    // TArray 独有
for (auto It = Container.CreateConstIterator(); It; ++It) { }  // 三种容器通用
```

### 迭代中安全移除

| 场景 | 安全做法 |
|------|---------|
| TArray 正向遍历移除 | ❌ 索引错位 |
| TArray 反向遍历移除 | ✅ 从后往前 |
| TMap 迭代中移除 | ✅ `It.RemoveCurrent()`（CreateIterator 模式） |
| TSet 迭代中移除 | ✅ `It.RemoveCurrent()` |

## 其他容器

### TMultiMap

当需要单个 Key 对应多个 Value 时使用 TMultiMap。与 TMap 的差异：

| 特性 | TMap | TMultiMap |
|------|------|-----------|
| 键唯一性 | 唯一 | 可重复 |
| Add(Key, Val) | 键存在则覆盖 | 键存在则新增条目 |
| Remove(Key) | 移除一个条目 | 移除该键所有条目 |
| 使用频率 | 高 | 低 |

### TMap 与 TSet 的选择

当只关心"元素是否存在"而不需要关联值时，使用 TSet 比 TMap<Key, NullType> 更直接。TSet 的语义清晰地表达了"集合"意图。

## 选型建议

### 默认选型策略

| 需求 | 推荐容器 |
|------|---------|
| 需要按索引访问、保持顺序 | TArray |
| 需要按键快速查找值 | TMap |
| 只需要快速集合存在性检查 | TSet |
| 不确定、通用场景 | TArray |
| 从一个容器迁移 | TArray（转换成本最低） |

### 性能敏感场景

| 场景 | 推荐做法 |
|------|---------|
| 频繁中间插入/移除 | 考虑 TArray 反序操作 + Add，或评估是否可无序（Swap 移除） |
| 大量元素+频繁查找 | TMap/TSet（哈希 O(1) vs 线性 O(n)） |
| 需要稳定指针/引用 | 在扩容/rehash 后重新获取，或使用指针容器（TSharedPtr） |
| 小量元素（< 50） | TArray 线性查找即可，哈希表开销可能更大 |
| 需要排序输出 | TArray（直接内置排序 API） |

### 容器转换

| 转换 | 方式 |
|------|------|
| TSet → TArray | `Set.Array()`（直接返回副本） |
| TMap → TArray（键） | `Map.GenerateKeyArray(OutArr)` |
| TMap → TArray（值） | `Map.GenerateValueArray(OutArr)` |
| TArray → TSet | `TSet<T>(Array)`（构造去重集合） |
| TArray → TMap | 逐元素 `Map.Add()` 或 Append |
