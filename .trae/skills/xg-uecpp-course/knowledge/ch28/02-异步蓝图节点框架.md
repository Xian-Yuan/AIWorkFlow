## 概述

本章的异步蓝图节点 `UXGSampleBDAyncAction` 封装了百度 ERNIE Bot HTTP 交互的全流程。与第二十五章（HTTP 时间节点）框架相同，但增加了**流式输出引脚 OnUpdate**。

## 四引脚设计

节点输出四个动态多播委托，每个签名相同：

```cpp
DECLARE_DYNAMIC_MULTICAST_DELEGATE_FourParams(FXGSampleBDLinkDelegate,
    FGuid, AsyncID,
    bool, bResult,
    FString, Message,
    FXGSampleBDRespInfo, RespInfo);
```

| 引脚 | 触发时机 | 说明 |
|------|----------|------|
| Then | Activate 时立即触发 | 标识节点已启动，参数为"等待中" |
| OnSuccess | 请求成功完成 | 非流式/流式结束后触发 |
| OnUpdate | 流式模式下每收到一个 chunk | 实时输出中间结果 |
| OnFail | 请求失败 | 网络错误、API 错误、JSON 解析失败 |

## Activate 执行模式

[实现](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleBDLink/Private/AsyncAction/XGSampleBPAyncAction.cpp#L40-L54)

```cpp
void UXGSampleBDAyncAction::Activate()
{
    Super::Activate();
    AsyncTask(ENamedThreads::GameThread, [this]() {
        this->Activate_Internal();
    });
    Then.Broadcast(AsyncID, false, TEXT("..."), FXGSampleBDRespInfo());
}
```

Activate 中通过 `AsyncTask(ENamedThreads::GameThread, ...)` 将实际执行推迟到下一帧。这种**错帧执行**确保外部代码（如 Blueprint 的构造逻辑）在 Activate_Internal 发送 HTTP 请求之前完成执行。

## AsyncID 追踪

每个节点在构造时生成唯一 GUID：

```cpp
FGuid AsyncID = FGuid::NewGuid();
```

该 ID 随所有委托广播传递，用于多节点并发场景下区分响应归属。
