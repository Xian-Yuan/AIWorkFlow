# 智能指针概述与类型体系

UE 提供了四类智能指针，定义在 `Templates/SharedPointer.h` 中。与 C++11 STL 的 `std::shared_ptr` / `std::unique_ptr` 相比，UE 版本在内存布局和调试体验上有显著差异。

## 四类智能指针对比

| 类型 | 所有权 | 可为空 | 可复制 | 引用计数 | 典型用途 |
|------|--------|--------|--------|----------|---------|
| `TSharedPtr` | 共享 | 是 | 是 | 强引用 | 共享对象生命周期 |
| `TSharedRef` | 共享 | **否** | 是 | 强引用 | 保证对象一定存在 |
| `TWeakPtr` | 观察 | 是 | 是 | **弱引用** | 打破循环引用 |
| `TUniquePtr` | 独占 | 是 | **否** | 无 | 资源独占，性能最优 |

## 引用计数机制

每个被智能指针管理的对象内部维护两个计数器：

- **SharedCount（强引用计数）**：TSharedPtr / TSharedRef 引用该对象的次数。强引用归零时对象立即析构
- **WeakCount（弱引用计数）**：TWeakPtr 引用该对象的次数。弱引用不影响对象生命周期，但会阻止引用计数控制块本身被释放

对象析构条件：`SharedCount == 0 && WeakCount > 0` → 对象析构，但引用计数控制块保留
控制块释放条件：`SharedCount == 0 && WeakCount == 0` → 对象和控制块同时释放

## 线程安全模式

```cpp
// 默认：非线程安全（ESPMode::NotThreadSafe），性能最优
TSharedPtr<FSmartPtrStruct> DefaultPtr = MakeShared<FSmartPtrStruct>();

// 线程安全模式
TSharedPtr<FSmartPtrStruct, ESPMode::ThreadSafe> SafePtr =
    MakeShared<FSmartPtrStruct, ESPMode::ThreadSafe>(998);
```

| 模式 | 枚举值 | 适用场景 |
|------|--------|---------|
| `NotThreadSafe` | 默认 | 单线程使用或手动加锁 |
| `ThreadSafe` | `ESPMode::ThreadSafe` | 跨线程共享同一智能指针 |
| `Fast` | `ESPMode::Fast` | 与 NotThreadSafe 同义 |
| `FastMT` | `ESPMode::FastMT` | 已弃用，行为等同 ThreadSafe |

## 与 STL 智能指针的差异

| 维度 | UE 智能指针 | C++ std 智能指针 |
|------|------------|-----------------|
| 内存布局 | 内联控制块，一次分配 | 分离控制块，两次分配 |
| 调试 | UE 日志/断言集成 | 需额外工具 |
| 线程安全 | 模板参数控制 | 通过引用计数原子操作 |
| 自定义删除器 | Lambda / 仿函数 | Deleter 模板参数 |
