# Actor 网络休眠与复制流程

## Actor 网络休眠（Dormancy）

网络休眠是一种优化机制，允许高频更新的 Actor 在不需要同步时暂停更新，大幅降低网络带宽消耗。

### 休眠状态

| 状态 | 说明 |
|------|------|
| `DORM_Never` | 从不休眠（默认） |
| `DORM_DormantAll` | 对所有连接进入休眠，不再发送复制更新 |
| `DORM_Awake` | 唤醒状态 |
| `DORM_Initial` | 初始状态，复制一次后自动休眠 |

### 设置休眠

```cpp
// 使 Actor 进入休眠
SetNetDormancy(DORM_DormantAll);

// 唤醒 Actor（通常通过属性修改触发）
```

### 唤醒机制

休眠中的 Actor 在以下情况下自动唤醒：

- 属性值在服务器上被修改（触发脏标记）
- 调用 `ForceNetUpdate()`
- Actor 被重新设置为相关（Relevancy 变化）

### 使用场景

- 静态环境物体（休眠后不更新位置）
- 拾取物（仅在可拾取状态变化时更新）
- 需要频繁更新但大部分时间无变化的属性

## 属性复制流程

### 同步路径

```
服务器属性变化
    → 脏标记（Dirty Flag）设置
    → 网络驱动程序检测到变化
    → 序列化 + 压缩
    → 通过网络发送
    → 客户端反序列化
    → 属性更新 + OnRep 回调（如果声明）
```

### 属性同步的特性

- **可靠传输**：属性同步使用可靠通道，客户端最终会反映服务器值
- **跳过中间值**：客户端不保证收到每个中间值，只反映最新值
- **增量更新**：仅同步变化的值

## RPC 排序规则

| 规则 | 说明 |
|------|------|
| 跨 Actor 的 RPC | 不可靠——不同 Actor 使用不同的连接通道 |
| 同一 Actor 的 Reliable RPC | 保持调用顺序 |
| 执行优先级 | 稳定 RPC > 属性同步 > 不稳定 RPC |

跨 Actor 的 RPC 不可靠是因为每个 Actor 有独立的通道，通道间没有顺序保证。

## 连接通道（Connection Channel）

- 每个客户端通过一个 PlayerController 建立连接通道
- 每个复制的 Actor 对每个连接有一个通道
- 通道负责属性同步和 RPC 的有序传递
- 所有权决定哪些通道处理 Client RPC
