## 安装源码版本引擎

### 前置条件

- Epic 账号已在 GitHub 获得 EpicGames 组织的访问授权
- GitHub 账号需要配置 Token（PAT）用于认证
- 已正确安装 VS2022 和对应工具链（见上文）

### 获取源码

**方式一：Git Clone**

```bash
git clone -b 5.4 https://github.com/EpicGames/UnrealEngine.git
```

- 优点：可随时拉取最新提交
- 缺点：初次克隆速度较慢（国内网络问题）

**方式二：Download ZIP**

- 从 GitHub 仓库页面直接下载源码压缩包
- 推荐使用 **7-Zip** 解压（Windows 自带解压对大文件容易卡死）
- 下载大小约 500-600MB

### 源码编译流程

1. **运行 Setup.bat**
   - 自动检查并下载第三方依赖库
   - 可通过修改 `Setup.bat` 中的参数开启**多线程下载**，提升下载速度
   - 支持断点续传
   - 下载完成后会请求管理员权限进行本地安装校验

2. **运行 GenerateProjectFiles.bat**
   - 生成 Visual Studio 解决方案文件（`.sln`）
   - 需要已正确安装 VS 和工具链，否则会报错

3. **在 VS 中编译**
   - 打开生成的 `UE5.sln`
   - 解决方案配置选择 **DebugGame Editor**（或 Development Editor）
   - 右键 UE5 设为启动项目
   - 右键 → 生成（Build），**不要点 Clean 或 Rebuild**（会导致数小时重编）
   - 编译项目数：UE 5.4 约 6700 个项目

### 常见编译问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 内存不足 / 堆栈溢出 | 并发编译数过高，内存不够（<16GB） | 限制 `max_concurrency`（如 7），设置单核内存上限（如 512MB） |
| 生成 VS 文件时报错 | VS 版本或工具链版本不匹配 | 确认 VS 版本和工具链与引擎要求一致 |
| Setup.bat 下载慢 | 国内网络 / GitHub 下载限速 | 开启多线程下载（修改 Setup.bat） |

**UBT 并发控制参数示例**（在 `BuildConfiguration.xml` 中配置）：

```xml
<Configuration>
  <ParallelExecutor>
    <MaxProcessorCount>7</MaxProcessorCount>
    <MemoryPerAction>512</MemoryPerAction>
  </ParallelExecutor>
</Configuration>
```

- `MaxProcessorCount`：最大并发数
- `MemoryPerAction`：每个编译动作分配的内存（MB）

### 编译后

- 源码引擎占用磁盘空间约 **240-300GB**
- 编译完成后可在 VS 中直接启动引擎
- 首次启动较慢（加载大量 DLL），属正常现象
- 启动时可观察输出日志，了解引擎模块加载顺序

### 引擎版本选择建议

- UE 官方一般维护最新的三个主版本
- **推荐选择中间版本**（如 5.3），既有稳定性又有较新的功能
- 避免使用 `.0` 版本（如 5.0.0、5.4.0），可能存在较多早期 Bug
- 避免追逐最新版本，引擎底层 API 可能在不同小版本间发生不兼容变更

### 关于源码引擎的说明

- 本课程第一阶段**不需要使用源码引擎**，发布版本引擎即可满足学习需求
- 源码引擎将在课程第二、三阶段涉及引擎深度定制时使用
- 源码引擎可用于开发独立程序（Slate 独立小程序等）

## Git 版本管理

### Git 安装

- 从 Git 官网下载安装程序（推荐使用迅雷等工具加速）
- 安装时一路默认选项即可

### UE 项目 Git 配置

**关键配置文件：**

| 文件 | 用途 |
|------|------|
| `.gitignore` | 忽略 UE 项目的中间产物（Intermediate、Binaries、Saved、DerivedDataCache 等） |
| `.gitattributes` | 声明大文件的 Git LFS 存储规则 |

### 常用 Git 命令

```bash
# 配置用户信息
git config --global user.email "your@email.com"
git config --global user.name "Your Name"

# 跟踪文件
git add .

# 提交
git commit -m "commit message"

# 查看日志
git log

# 查看状态
git status
```

### 重要配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| `core.ignorecase` | `false` | 关闭大小写忽略，避免 UE 项目因大小写导致文件溯源问题 |
| 中文文件名支持 | 在 `.git/config` 中配置 `quotepath = false` | 使 Git 正确显示中文文件名 |

### 分支管理概念

- `master/main`：主分支，保持稳定，不直接修改
- `develop`：开发分支，日常开发主阵地
- 各功能分支（feature branch）：独立开发，完成后合并回 develop

### 版本管理的作用

- 支持版本回退（代码写坏了可以回退到之前的正常版本）
- 支持多人协同开发
- 初级阶段**可以暂不深入** Git，但建议掌握 add、commit、log 等基本操作
