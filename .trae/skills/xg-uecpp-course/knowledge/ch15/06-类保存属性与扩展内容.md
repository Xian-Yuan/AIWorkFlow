# 类保存修改后的属性与扩展内容

## 通过 CDO 保存属性到 INI

除了 GConfig 手动读写，UObject 类还提供了 `SaveConfig` 方法用于将 CDO 的属性写回 INI。

```cpp
void AXGINIActor::ModifyConfigVariable()
{
    AXGINIActor* Settings = GetMutableDefault<AXGINIActor>();
    Settings->MyConfigVariable = 420;

    FString PathToConfigFile = FString::Printf(
        TEXT("%sDefaultGame.ini"), *FPaths::SourceConfigDir());
    Settings->SaveConfig(CPF_Config, *PathToConfigFile);
}
```

### 流程

| 步骤 | 代码 | 说明 |
|------|------|------|
| 1. 获取 CDO | `GetMutableDefault<AXGINIActor>()` | 获取当前类的默认对象 |
| 2. 修改属性 | `Settings->MyConfigVariable = 420` | 修改 CDO 上的 Config 属性 |
| 3. 写路径 | `FPaths::SourceConfigDir()` | 构造目标 INI 路径（DefaultGame.ini） |
| 4. 保存 | `SaveConfig(CPF_Config, *PathToConfigFile)` | 将 Config 标记的属性写回 INI |

### SaveConfig 参数

`SaveConfig(uint64 Flags, const TCHAR* Filename, ...)`

| 参数 | 说明 |
|------|------|
| `CPF_Config` | 指定只保存标记了 CPF_Config 标志的属性 |
| `Filename` | 目标 INI 文件路径（留空则按 UCLASS(config = X) 自动选择） |

### 与 GConfig 方式的区别

| 方式 | 原理 | 适用场景 |
|------|------|---------|
| `SaveConfig(CPF_Config, path)` | CDO 反射序列化 | 保存 UObject 子类的标记属性 |
| `GConfig->SetInt/Flush` | 直接操作 INI 字符串 | 任意段名、无需 UObject 上下文 |
| `XGSampleSettings->SaveConfig` | UDeveloperSettings 子类的 CDO 保存 | DeveloperSettings 的持久化 |

## 未覆盖的扩展内容

以下内容在该章节中没有深入讲解，属于配置系统的扩展知识：

### Console Variables（控制台变量）

通过 `FAutoConsoleVariableRef` 声明可在控制台中实时调整的变量：

```cpp
static TAutoConsoleVariable<int32> CVarShadowQuality(
    TEXT("r.ShadowQuality"),
    1,
    TEXT("阴影质量: 0=低, 1=中, 2=高"),
    ECVF_Scalability);
```

- 在控制台输入 `r.ShadowQuality 2` 实时修改
- `ECVF_Scalability` 表示该变量参与可扩展性设置
- 适用于运行时调试和性能调优

### 命令行参数

在启动时通过命令行传递参数：

```cmd
MyGame.exe -ResX=1920 -ResY=1080 -NoSound
```

C++ 中通过 `FParse::Value` 解析：

```cpp
int32 ResX;
FParse::Value(FCommandLine::Get(), TEXT("ResX="), ResX);
```

### Exec 命令

在 UObject 子类中重写 `Exec()` 方法，可以自定义控制台命令：

```cpp
virtual bool Exec(UWorld* InWorld, const TCHAR* Cmd, FOutputDevice& Ar) override;
```

### Cheat Manager

通过 `CheatManager` 类注册作弊命令，用于调试和测试：

```cpp
UCLASS()
class UMyCheatManager : public UCheatManager
{
    GENERATED_BODY()

    UFUNCTION(exec)
    void GodMode();
};
```

## 章节总结

UE 的配置系统提供了从开发到发布的全链路支持：

1. **开发阶段** — DeveloperSettings 在 Project Settings UI 中可视化编辑
2. **C++ 接口** — GetMutableDefault + SaveConfig 进行编程读写
3. **INI 系统** — 多层级的配置覆盖，GGameIni 自动选择当前层
4. **GConfig** — 底层手动读写，灵活但需要手动管理
5. **安全** — 密钥泄露风险与 pak 加密防护
6. **扩展** — Console Variables、命令行、Exec、Cheat Manager
