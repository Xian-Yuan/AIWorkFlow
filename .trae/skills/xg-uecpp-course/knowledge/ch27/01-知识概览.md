# 第二十七章 知识概览

## 章节目标

实现"摄像头活体检测"功能：通过 UE 的 Media Framework 打开摄像头捕获视频流，将视频帧渲染到 RenderTarget2D 并生成为 PNG 二进制数据，然后通过 HTTP POST 请求将图片上传到科大讯飞静默活体检测 API，最终解析检测结果（人脸是否匹配、置信度分数、人脸矩形区域等）。

## Pipeline 总览

```
┌─────────────────────────────────────────────────────────────┐
│                    XGSamplePicture Plugin                     │
│  Camera → MediaPlayer → MediaTexture → Material →           │
│  DrawMaterialToRenderTarget → RenderTarget2D                │
│       ↓                                                     │
│  UXGSamplePictureActionAction (Async Blueprint Node)        │
│  ReadPixels → AlphaPremultiply → FImageUtils::ExportToPNG   │
│       ↓ (PNG Binary Data)                                   │
│                    XGSampleXFLink Plugin                      │
│  UXGSampleXFSilentBiopsyAyncAction (Async Blueprint Node)   │
│  Base64Encode → BuildJSON → SignURL(SHA256) → HTTP POST     │
│       ↓                                                     │
│  科大讯飞 静默活体检测 API                                    │
│       ↓                                                     │
│  Parse Response → 1st-level auth check → 2nd-level result   │
│  → BP Delegate (bPassed, Score, FaceRect)                    │
└─────────────────────────────────────────────────────────────┘
```

## 三种 HTTP 上传方式对比

| 方式 | 描述 | 本章使用 |
|------|------|----------|
| Base64 in JSON | 图片二进制 Base64 编码后放入 JSON body 的 `image` 字段 | **是（讯飞主流方式）** |
| X-WWW-Form-UrlEncoded | 二进制数据放在 body，参数在 header | 已淘汰 |
| Multipart/Form-Data | 最复杂，需要处理 boundary 分隔 | 仅在服务端强制要求时使用 |
