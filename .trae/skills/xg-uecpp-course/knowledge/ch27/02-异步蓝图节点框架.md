# 异步蓝图节点框架

本章的两个插件（XGSamplePicture 和 XGSampleXFLink）都使用 `UBlueprintAsyncActionBase` 子类实现异步蓝图节点，模式一致。

## 核心类继承结构

```
UBlueprintAsyncActionBase
  ├── UXGSamplePictureActionAction    （摄像图 → PNG）
  └── UXGSampleXFSilentBiopsyAyncAction （讯飞活体检测 HTTP）
```

## 标准实现模式

### 1. 类声明（.h）

```cpp
UCLASS(meta = (HideThen = true))
class XGSAMPLEPICTURE_API UXGSamplePictureActionAction : public UBlueprintAsyncActionBase
{
    GENERATED_BODY()
```

关键属性：
- `HideThen = true` — 隐藏默认的 `Then` 执行引脚，统一通过命名委托引脚控制流程
- `BlueprintInternalUseOnly = "true"` — 隐藏工厂函数的标准执行引脚，防止在蓝图右键菜单中直接调用

### 2. 委托声明

```cpp
DECLARE_DYNAMIC_MULTICAST_DELEGATE_FourParams(FXGSamplePictureDelegate,
    FGuid, AsyncID,
    bool, bResult,
    FString, Message,
    const TArray<uint8>&, OutData);
```

两个插件各定义了三个委托引脚：
| 引脚 | 触发时机 |
|------|----------|
| `Then` | Activate() 时立即触发，通知蓝图"任务已启动" |
| `OnSuccess` | 异步操作成功完成时触发 |
| `OnFail` | 异步操作失败时触发 |

> 注：SilentBiopsy 的 `FXGSampleXFSilentBiopsyDelegate` 第四个参数为 `FXGSampleXFSilentBiopsyRespInfo` 结构体（包含检测结果），而非 `TArray<uint8>`。

### 4. Activate() 流程

```cpp
void UXGSamplePictureActionAction::Activate()
{
    Super::Activate();  // 必须调用父类
    
    AsyncTask(ENamedThreads::GameThread, [this]() {
        this->Activate_Internal();
    });
    
    Then.Broadcast(AsyncID, false, TEXT("XGSamplePictureActionAction is just started,please wait to finish"), {});
}
```

代码见 [XGSamplePictureAsyncAction.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSamplePicture/Source/XGSamplePicture/Private/XGSamplePictureActionAction.cpp#L41-L53) L41-53 和 [SilentBiopsyAsyncAction.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSampleLink/Source/XGSampleXFLink/Private/AsyncAction/XGSampleXFSilentBiopsyAyncAction.cpp#L39-L57) L39-57。

关键要点：
- **必须**调用 `Super::Activate()` — 子类如果覆盖 Activate() 而忘记调用父类，HTTP 回调将不执行
- 实际工作放在 `Activate_Internal()` 中执行，通过 `AsyncTask` 确保在 GameThread 执行
- `Then` 委托立即广播表示任务已启动

关键要点：
- `SetReadyToDestroy()` — 标记 UObject 可被销毁，不等同于立即销毁
- 每次成功或失败路径的末尾必须调用，否则 UObject 内存泄漏
- 对于 SilentBiopsy：还需 `Clear()` 清空委托，重置所有成员变量

### 6. OnSuccess/OnFail 的线程安全

```cpp
void UXGSamplePictureActionAction::CallOnSuccess(FString InMessage, TArray<uint8> InPNGData)
{
    FXGSamplePictureDelegate TempDelegate = OnSuccess;
    AsyncTask(ENamedThreads::GameThread, [=]() {
        TempDelegate.Broadcast(TempGuid, true, InMessage, InPNGData);
    });
}
```

代码见 [XGSamplePictureAsyncAction.cpp](file:///d:/UPS/GitLab/XGUnrealNote/Courses/UE-CPP-Basic-to-Advanced/code/001_XGSampleDemo/Plugins/XGSamplePicture/Source/XGSamplePicture/Private/XGSamplePictureActionAction.cpp#L155-L167) L155-167。

关键要点：
- 通过值捕获拷贝委托对象，避免捕获期对象被释放
- `AsyncTask(ENamedThreads::GameThread, ...)` 确保广播在 GameThread 执行
