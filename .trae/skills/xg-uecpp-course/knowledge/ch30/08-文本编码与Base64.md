# 文本编码与 Base64

## 编码链路

```
FString (UTF-16 LE) → UTF-8 → Base64 → JSON → WebSocket Send
```

## FString → UTF-8

`FString` 在 Windows 平台内部存储为 UTF-16 LE（每个字符 2 字节）。网络传输需要 UTF-8 编码（每个 ASCII 字符 1 字节，中文 3 字节）。

```cpp
// TCHAR_TO_UTF8 宏将 FString 转换为 UTF-8 编码的 char* 字符串
FString InText = TEXT("你好世界");
const char* UTF8Str = TCHAR_TO_UTF8(*InText);
// 输出："\xe4\xbd\xa0\xe5\xa5\xbd\xe4\xb8\x96\xe7\x95\x8c"（12 字节）
```

### TCHAR_TO_UTF8 实现原理

```cpp
// UE 内部实现（简化）
// FTCHARToUTF8 是一个转换辅助类，管理转换缓冲区的生命周期
FTCHARToUTF8 Converter(*InText);
const char* UTF8Str = Converter.Get();
// 转换器在析构时自动释放临时内存
```

## UTF-8 → Base64

```cpp
// FString → UTF-8 → Base64
FString UTF8Str = TCHAR_TO_UTF8(*InText);
FString Base64Text = FBase64::Encode(UTF8Str);

// 或者一步到位：
FString Base64Text = FBase64::Encode(TCHAR_TO_UTF8(*InText));
```

### Base64 编码原理

Base64 将每 3 字节（24bit）编码为 4 个 ASCII 字符，每个字符对应 6bit：

```
原始字节:   [0xE4] [0xBD] [0xA0]  (3 字节)
二进制:     11100100 10111101 10100000
分组 6bit:  111001 001011 110110 100000
Base64 表:  '5'    'L'    '2'    'g'
```

编码后体积增加约 33%（3 字节 → 4 字符）。

## 在 Blueprint 中的使用

由于 `FBase64::Encode` 和 `TCHAR_TO_UTF8` 都是 C++ 层 API，Blueprint 调用时需要通过函数库封装：

```cpp
// 在 XGSampleTTSAsyncAction 内部自动处理
void UXGSampleTTSAsyncAction::OnWebSocketConnected(const FString& InStr)
{
    // 文本编码转换
    FString UTF8Text = TCHAR_TO_UTF8(*InText);
    ReqInfo.data.text = FBase64::Encode(UTF8Text);

    // 发送 JSON
    FString JsonStr;
    FJsonObjectConverter::UStructToJsonObjectString(ReqInfo, JsonStr);
    Socket->Send(JsonStr);
}
```

## 注意事项

| 问题 | 说明 |
|------|------|
| 中文编码 | 确保文本使用 UTF-8 编码，否则服务器端可能无法正确解析 |
| 文本长度 | 讯飞 TTS 对文本长度有限制（通常不超过 1000 字） |
| 特殊字符 | 换行符、制表符等控制字符可能需要转义或过滤 |
| 编码一致性 | `tte` 字段需与编码方式匹配，使用 UTF8 时 tte 应为 "UTF8" |

### 编码检测

```cpp
// 如果文本包含非 ASCII 字符，需要确认编码正确
bool bHasNonASCII = false;
for (const auto& Char : InText)
{
    if (Char > 127)
    {
        bHasNonASCII = true;
        break;
    }
}
// 中文等非 ASCII 字符通过 TCHAR_TO_UTF8 自动处理
```
