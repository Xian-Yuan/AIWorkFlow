# 日志运行时控制与多线程安全

## 运行时日志级别控制

### 控制台命令方式

在编辑器 Output Log 的控制台输入框或游戏内控制台中：

```
LogXGSample Verbose
```

- `LogXGSample` — 目标日志类别名称
- `Verbose` — 设置为 Verbose 级别，也可以设为 `Display`、`Log`、`Warning`、`Error`、`VeryVerbose`

执行后，所有低于或等于该级别的日志都会输出。例如设置为 `Verbose` 后，`UE_LOG(LogXGSample, Verbose, ...)` 才会输出（默认 `Log` 级别时 Verbose 是不输出的）。

**仅当前会话有效**，引擎重启后恢复为默认级别。

### INI 配置文件方式

在 `DefaultEngine.ini` 的 `[Core.Log]` 段中配置：

```ini
[Core.Log]
LogXGSample=Verbose
```

- 持久化配置，重启引擎后仍然生效
- 格式为 `类别名=级别名`
- 可以配置多个类别

## 多线程日志安全

**UE_LOG 不能在非 GameThread 上直接调用**。在 Worker Thread 中调用 UE_LOG 可能导致竞态条件或崩溃。

### 安全方案：AsyncTask 回推

```cpp
void AXGLogActor::PringLogString(FString InStr)
{
    AsyncTask(ENamedThreads::GameThread, [InStr]() {
        UE_LOG(LogXGSample, Error, TEXT("Async--[%s]"), *InStr);
    });
}
```

| 元素 | 说明 |
|------|------|
| `AsyncTask` | UE 的异步任务调度函数 |
| `ENamedThreads::GameThread` | 指定在 GameThread 上执行 Lambda |
| `[InStr]` | 按值捕获参数（线程安全，复制副本） |
| Lambda 体 | 在此 Safe Zone 内调用 UE_LOG |

**工作原理**：Lambda 被排队到 GameThread 的下一个 Tick 周期执行，而不是立即执行。因此日志打印有轻微延迟（最多一帧），但保证了线程安全。

**适用场景**：任何可能在 Worker Thread 中触发的日志（如 FRunnable 的 Run、文件 IO 回调、网络接收回调等）。

## 调试辅助工具概览

UE 提供了一系列开发期调试工具，与日志系统互补：

| 工具 | 功能 | 入口 |
|------|------|------|
| VS 断点调试 | 设置 F9 断点，F10/F11 步过/步入 | VS IDE |
| 条件断点 | 仅在满足条件时中断（如 `MyInt > 100`） | VS IDE 右键断点 |
| Blueprint 调试器 | 类似 VS 的断点/单步调试 | 编辑器工具栏 |
| Widget Reflector | 定位 Slate UI 元素 | 编辑器 Command 窗口输入 `WidgetReflector` |
| Stat 命令 | 性能分析 | 控制台输入 `stat game`、`stat gpu` |
| 屏幕百分比 | 性能优化时调整渲染分辨率 | 控制台输入 `r.ScreenPercentage 50` |
