# 反射宏体系总览

## 概述

UE 的反射宏体系是连接 C++ 静态代码与引擎运行时能力的核心桥梁。这套体系跨越多章知识——第二章（引擎架构/UBT/UHT 管线）提供了编译期框架，第三章（基础概念/宏体系）提供了具体编码规范。本总览将两章知识整合为单一全景视图。

## 编译管线（第二章 ↔ 第三章）

```
源码中的 UE 宏（UCLASS/UPROPERTY/UFUNCTION/...）
    │
    ▼
UHT（UnrealHeaderTool）编译期解析
    │  ├── 解析 UCLASS/UPROPERTY 等宏的参数
    │  ├── 为每个 UE 类型生成 .generated.h / .gen.cpp
    │  └── 生成反射元数据（类信息、属性偏移、函数签名）
    │
    ▼
Generated 文件嵌入
    │  ├── #include "xxx.generated.h" 必须在头文件末尾
    │  ├── GENERATED_BODY() / GENERATED_USTRUCT_BODY() 宏展开
    │  └── 位置：Intermediate/Build/Win64/Editor/Inc/项目名/UHT/
    │
    ▼
运行时反射能力
    ├── 类信息：StaticClass() / ClassPrivate / UClass 元数据
    ├── 属性访问：FindPropertyByName() / 反射设置/获取值
    ├── 函数调用：ProcessEvent() / CallFunctionByNameWithArguments()
    ├── 序列化：SaveGame / 网络复制 / 编辑器存档
    └── GC 管理：AddReferencedObjects / FUObjectReferenceFinder
```

### 关键知识链

| 知识环节 | 所属章节 | 学习要点 |
|---------|---------|---------|
| Module/Build.cs | 第二章 | 声明依赖模块、Public/Private 分离 |
| UHT 管线 | 第二章 | 编译时宏解析、生成反射代码 |
| 宏声明 | 第三章 | UCLASS/UPROPERTY/UFUNCTION/USTRUCT/UENUM/UINTERFACE 六大宏 |
| Generated 文件 | 第三章 | .generated.h 包含位置、GENERATED_BODY() |
| 运行时能力 | 第三章 | StaticClass()、属性/函数反射操作 |

## 宏体系全景

六个 UE 反射宏按用途分类：

| 宏 | 声明对象 | 核心特性 | GENERATED 宏 | 有无 CDO |
|----|---------|---------|-------------|---------|
| UCLASS | 类（继承 UObject/UActor） | BlueprintType/Blueprintable/MinimalAPI | GENERATED_BODY() | 有 |
| UPROPERTY | 成员变量 | 可见性/编辑性/复制/序列化 | —（不独立存在） | — |
| UFUNCTION | 成员函数 | 蓝图调用/网络同步/控制台 | —（不独立存在） | — |
| USTRUCT | 结构体 | 值类型、蓝图画读写、Break/Make 节点 | GENERATED_USTRUCT_BODY() | 无 |
| UENUM | 枚举定义 | uint8 底层约束、UMETA、位掩码 | 无（C++ enum） | 无 |
| UINTERFACE | 接口类 | 双类模式（U类+I类）、Cast/ImplementsInterface | GENERATED_BODY() | 有（U类） |

## UCLASS

### 说明符

| 说明符 | 用途 | 示例 |
|--------|------|------|
| BlueprintType | 可在蓝图中作为变量类型使用 | `UCLASS(BlueprintType)` |
| Blueprintable | 可在蓝图中被继承 | `UCLASS(Blueprintable)` |
| MinimalAPI | 仅导出类型本身，不导出类方法 | `UCLASS(MinimalAPI)` |

### 代码示例

```cpp
// BlueprintType — 可在蓝图中作变量类型（XGBaseObject.h）
UCLASS(BlueprintType)
class XGSAMPLEDEMO_API UXGBaseObject : public UObject
{
    GENERATED_BODY()
};

// Blueprintable — 蓝图可继承（XGPropertyActor.h）
UCLASS(Blueprintable)
class AXGPropertyActor : public AActor
{
    GENERATED_BODY()
};
```

## UPROPERTY

UPROPERTY 是所有反射宏中使用频率最高、说明符体系最庞杂的一个。

### 可见性与编辑性

| 说明符 | 运行时 | 编辑器 | 默认值编辑 | 实例编辑 |
|--------|--------|--------|----------|---------|
| （无修饰） | ✓ | ✗ | ✗ | ✗ |
| BlueprintReadOnly | ✓（只读） | ✗ | ✗ | ✗ |
| BlueprintReadWrite | ✓（读写） | ✗ | ✗ | ✗ |
| EditDefaultsOnly | ✓ | ✓（只读） | ✗ | ✗ |
| EditInstanceOnly | ✓ | ✓（只读） | ✓ | ✗ |
| EditAnywhere | ✓ | ✓（只读） | ✓ | ✓ |
| VisibleDefaultsOnly | ✓ | ✓（只读） | ✗ | ✗ |
| VisibleInstanceOnly | ✓ | ✓（只读） | ✓ | ✗ |
| VisibleAnywhere | ✓ | ✓（只读） | ✓ | ✓ |
| BlueprintReadWrite + EditAnywhere | ✓（读写） | ✓（读写） | ✓ | ✓ |

### Meta 说明符

| Meta | 用途 | 示例 |
|------|------|------|
| Bitmask | 将 int32 属性变为位掩码下拉列表 | `Meta = (Bitmask)` |
| BitmaskEnum | 将位掩码与指定枚举关联 | `Meta = (BitmaskEnum = "EColorBits1")` |
| EditCondition | 条件控制是否可编辑 | `Meta = (EditCondition = "bEdite")` |
| DisplayName | 编辑器中显示的名称 | `Meta = (DisplayName = "姓名")` |

### 行为说明符

| 说明符 | 用途 |
|--------|------|
| Category | 在编辑器细节面板中的分组名 |
| Replicated | 网络复制 |
| Transient | 不参与序列化 |
| SaveGame | 参与存档保存 |

### Meta 与说明符的协同

| 模式 | 写法 | 效果 |
|------|------|------|
| 仅 Meta | `UPROPERTY(Meta = (Bitmask))` | 位掩码，蓝图中不可见 |
| Meta + 蓝图 | `UPROPERTY(BlueprintReadWrite, Meta = (Bitmask))` | 位掩码，蓝图可见 |
| Meta + 编辑 | `UPROPERTY(EditAnywhere, Meta = (EditCondition = "bEdite"))` | 可编辑，有条件控制 |

## UFUNCTION

### 蓝图相关

| 说明符 | 执行引脚 | 实现方式 | 适用场景 |
|--------|---------|---------|---------|
| BlueprintCallable | 有 | C++ 实现 | C++ 实现逻辑，蓝图调用 |
| BlueprintPure | 无 | C++ 实现 | 纯计算函数，无副作用 |
| BlueprintImplementableEvent | 有 | 蓝图必须实现 | 蓝图定义行为，C++ 仅声明 |
| BlueprintNativeEvent | 有 | C++ 实现 + 蓝图覆写 | 既有默认实现又可扩展 |

### 网络相关

| 说明符 | 执行位置 | 可靠性 |
|--------|---------|--------|
| Client | 客户端 | 配合 Reliable/Unreliable |
| Server | 服务端 | 配合 Reliable/Unreliable |
| NetMulticast | 所有客户端 | 配合 Reliable/Unreliable |
| Reliable | — | 保证到达，用于关键逻辑 |
| Unreliable | — | 不保证到达，用于频繁更新 |

### 其他

| 说明符 | 用途 |
|--------|------|
| Exec | 控制台命令入口 |
| CallInEditor | 编辑器中显示为触发按钮 |
| Const | 标记函数不修改状态 |

### 参数传递模式

| 写法 | 蓝图引脚 | 说明 |
|------|---------|------|
| `const Type&` | 输入 | 标准 C++ 常量引用 |
| `Type&` | 输出 | 蓝图中显示为输出引脚 |
| `TArray<Type>&` | 输入 | C++ 中修改不影响蓝图侧（默认拷贝语义） |
| `UPARAM(ref) TArray<Type>&` | 输入+引用 | C++ 中修改反映回蓝图 |

## USTRUCT

USTRUCT 声明值类型结构体，与 UCLASS 的关键差异：

| 维度 | USTRUCT | UCLASS |
|------|---------|--------|
| 继承 | 不支持继承（无多态） | 支持继承链 |
| 生命周期 | 栈/堆分配，无 GC | UObject GC 管理 |
| 复制 | 逐值拷贝 | 引用/指针传递 |
| CDO | 无 | 有默认对象 |
| GENERATED | `GENERATED_USTRUCT_BODY()` | `GENERATED_BODY()` |
| UPROPERTY | 支持 | 支持 |
| UFUNCTION | 不支持 | 支持 |
| 网络复制 | 作为 UPROPERTY 成员复制 | 单独注册复制 |

```cpp
// 基本结构体（XGBaseStruct.h）
USTRUCT(BlueprintType)
struct FXGBaseStruct
{
    GENERATED_USTRUCT_BODY()

    UPROPERTY(BlueprintReadOnly, EditAnywhere)
    FString Name;
};
```

## UENUM

### 基本枚举

```cpp
UENUM(BlueprintType)
enum class EMYUENUM : uint8  // 必须指定底层类型
{
    Value1,
    Value2
};
```

### UMETA 枚举值

```cpp
UENUM(BlueprintType)
enum class EXGActorType : uint8
{
    Hero     UMETA(DisplayName = "Hero"),
    Monster  UMETA(Hidden),           // 在蓝图中隐藏
    NPC      UMETA(DisplayName = "NPC")
};
```

### 位掩码枚举

```cpp
UENUM(Meta = (Bitflags))
enum class EColorBits1 : uint8
{
    Red,
    Green,
    Blue
};
```

```cpp
UENUM(Meta = (Bitflags, UseEnumValuesAsMaskValuesInEditor = "true"))
enum class EColorBits2 : uint8
{
    Red   = 0x01,
    Green = 0x02,
    Blue  = 0x04
};
```

### 约束

- 必须指定底层类型（通常为 `uint8`），否则无法暴露给蓝图
- `UMETA(Hidden)` 隐藏指定值
- `UseEnumValuesAsMaskValuesInEditor = "true"` 使编辑器使用实际掩码值
- 非位掩码枚举可同时应用 `BlueprintType`

## UINTERFACE

### 双类模式

UINTERFACE 在 UHT 编译时会生成两个类：

| 类 | 命名规则 | 用途 |
|---|---------|------|
| U 类 | `UXxx`（继承 UInterface） | 反射元数据持有者，同 UObject 体系 |
| I 类 | `IXxx`（继承 IInterface） | C++ 抽象接口，包含纯虚函数 |

```cpp
// XGBaseInterface.h
UCLASS(MinimalAPI)
class UXGBaseInterface : public UInterface
{
    GENERATED_BODY()
};

class XGSAMPLEDEMO_API IXGBaseInterface : public IInterface
{
    GENERATED_BODY()
public:
    virtual void SomeFunction() = 0;  // 纯虚函数
};
```

### 接口判定

| 方式 | 说明 |
|------|------|
| `Cast<IXGActorTypeInterface>(Actor)` | C++ 隐式转换风格 |
| `Actor->ImplementsInterface(UXGActorTypeInterface::StaticClass())` | 运行时反射检查 |
| `Actor->Implements<UXGActorTypeInterface>()` | C++ 模板风格 |

### 多接口实现

```cpp
// AInterfaceActor 同时实现两个接口（InterfaceActor.h）
UCLASS()
class AInterfaceActor : public AActor,
    public IXGHealthInterface,
    public IXGActorTypeInterface
{
    GENERATED_BODY()
public:
    // 实现 IXGHealthInterface
    virtual void Heal_Implementation(float Amount) override;

    // 实现 IXGActorTypeInterface
    virtual EXGActorType GetActorType_Implementation() override;
};
```

## 常见陷阱

### 1. BlueprintPure = false

`UFUNCTION(BlueprintPure = false)` 是一个拼写错误——`BlueprintPure` 参数应为布尔值，但 UFUNCTION 该参数只接受 `true`。`= false` 的写法不产生语法错误，但导致该函数在蓝图中不显示，难以排查。

### 2. 不支持的类型

下列 C++ 类型不能用于 UPROPERTY：
- `uint16`、`uint32`（非 TEnumAsByte）、`int8`、`int16`
- 嵌套容器：`TArray<TArray<int32>>`（编译器支持 11 标准但不被 UHT 识别）

### 3. uint32 位域的正确写法

错误：`uint32 bIsHungry : 1`
正确：`uint32 bIsHungry : 1` + 配合 `UPROPERTY()`（但 uint32 位域在 UHT 中实际不支持）

### 4. 枚举底层类型

UENUM 必须指定 `: uint8` 底层类型，否则蓝图中不可见。

### 5. GENERATED_USTRUCT_BODY() vs GENERATED_BODY()

USTRUCT 必须使用 `GENERATED_USTRUCT_BODY()`，不能使用 `GENERATED_BODY()`。

### 6. struct 中持有 UObject 指针

结构体可以包含 `UPROPERTY()` 修饰的 UObject 指针成员（用于 GC 追踪），但结构体本身不参与 GC。
