## 一、四种代码组织形式

### 1.1 主项目内 C++ 文件

功能直接放在 `001_XGSampleDemo/Source/XGSampleDemo/` 下，按章节编号建立子目录：

```
Source/XGSampleDemo/
├── 025_HttpTime/    # 第 25 章 HTTP
├── 026_Net/         # 第 35 章 网络复制
├── 027_RPC/         # 第 35 章 RPC
└── 025_WebP/        # 第 22 章 WebP 展示
```

**适用场景**：功能简单（单个 .h/.cpp 对），不需要跨项目复用，不需要额外模块拆分。

### 1.2 主项目 + 插件

功能封装为插件，放在 `001_XGSampleDemo/Plugins/` 下：

```
Plugins/
├── XGSampleWebP/      # 第 21 章：WebP 编解码第三方库封装
├── XGSampleClient/    # 第 26 章：HTTP 客户端
├── XGSamplePicture/   # 第 27 章：摄像头捕获 + PNG 导出
├── XGSampleLink/      # 第 27~30 章：第三方 AI 链接（百度/讯飞）
│   ├── XGSampleBDLink/    # 百度文心一言（第 28 章）
│   ├── XGSampleXFLink/    # 讯飞 STT/TTS（第 29~30 章）
│   └── XGSampleXFLinkLibrary/  # 加密库（共享）
├── XGSampleWSM/       # 第 31 章：WebSocket 消息管理
├── XGSampleEMail/     # 第 32 章：TCP 邮件发送
└── XGSampleTool/      # 工具函数库
```

**插件模板**：所有插件使用 `Blank` 模板创建，从零构建，而非用 `Content Only` 或 `Blueprint Library` 模板。

### 1.3 独立程序

独立程序是脱离 Editor 的可执行程序，放在 `013_独立程序源码/` 下：

```
013_独立程序源码/
├── XGBlankProgram/   # 空白模板（第 23/26 章参考）
├── XGSampleServer/   # HTTP 服务器（第 26 章）
└── XGSlateSample/    # Slate UI 程序（第 23 章对比参考）
```

另有单独的独立程序项目（不在 013 目录下）：

```
007_UEProgram1/       # 第 23 章：Slate 独立程序（UE 5.4.2）
008_UEProgram2/       # 第 26 章参考：HTTP 独立程序
```

**独立程序的特征**：
- 有自己的 `.Build.cs` 和 `.Target.cs`
- 使用 `INT32_MAIN_INT32_ARGC_TCHAR_ARGV()` 作为入口
- `PlatformWorkarounds.cpp` 处理 Windows 平台兼容
- 模块引用通常不包含 `Engine`，仅依赖 `Core`/`Networking`/`HTTP` 等

### 1.4 独立游戏项目

完整的游戏项目，有自己的 `.uproject`、独立的 Module 和 Assets：

```
010_XGMultiGame/      # 多人游戏项目（引用 XGSampleWSM 插件）
011_XGMultiManage/    # 多人管理项目（引用 XGSampleWSM 插件）
016_XGNetDemo/        # 网络演示项目
017_XGRPG/            # GAS 案例项目（引用 GameplayAbilities + ModularGameplay）
```

## 二、插件阶段演变

### 2.1 阶段一：独立功能插件（第 21 章）

```cpp
// XGSampleWebP — 独立第三方库封装插件
// 目标：编译 libwebp 静态库，提供 C++ 调用接口
// 结构：Internal（隐藏实现）+ External（暴露接口）模块拆分
```

### 2.2 阶段二：客户端功能插件（第 26 章）

```cpp
// XGSampleClient — HTTP 客户端插件
// 结构：Subsystem + BPLibrary + Type 三级
// 模式：BPLibrary 提供蓝图入口，Subsystem 管理请求生命周期
```

### 2.3 阶段三：功能分离双插件（第 27 章）

```
两个插件各司其职：
XGSamplePicture     → 摄像头捕获 + RenderTarget2D → PNG
XGSampleLink/XFLink → HMAC-SHA256 签名 + HTTP POST + 响应解析
```

### 2.4 阶段四：插件内多模块（第 28~30 章）

```cpp
// XGSampleLink 插件内多模块
// BDLink — 百度文心一言（XGSampleBDLink 模块）
// XFLink — 讯飞 STT/TTS（XGSampleXFLink 模块）
// XFLinkLibrary — 加密库共享（ThirdParty 模块）

// 模块间共享的加密库
ThirdParty/XGSampleXFLinkLibrary/
├── Private/XGSampleXFLinkBase.cpp/.h     // OpenSSL 封装
├── Public/XGSampleXFLinkLibrary.cpp/.h   // 签名工具
└── XGSampleXFLinkLibrary.Build.cs
```

### 2.5 阶段五：全功能自建插件（第 31 章）

```cpp
// XGSampleWSM — WebSocket 服务端 + 客户端双角色插件
// 7 个目录：Connection/、Subsystem/、Type/、Util/、Log/、BPLibrary、Module
// 双 Subsystem：XGWSMClientSubsystem + XGWSMServerSubsystem
```

### 2.6 阶段六：全静态 C++ 插件（第 32 章）

```cpp
// XGSampleEMail — 脱离 UObject 体系的纯 C++ 插件
// AsyncAction：TSharedFromThis（不是 UBlueprintAsyncActionBase）
// Subsystem：C++ 单例（不是 UGameInstanceSubsystem）
// Build.cs：不含 Engine 模块
```

## 三、插件共享模式

### 3.1 同一项目内共享

`XGSampleLink` 插件内的加密库 (`XGSampleXFLinkLibrary`) 被 `BDLink` 和 `XFLink` 两个模块共同引用：

```
XGSampleLink/
├── XGSampleBDLink/              → 百度签名（HMAC-SHA256）
├── XGSampleXFLink/              → 讯飞签名（HMAC-SHA256 / HMAC-SHA1）
└── ThirdParty/XGSampleXFLinkLibrary/  → OpenSSL 封装（供前两者使用）
```

### 3.2 跨项目共享

**XGSampleWSM 插件**被三个项目同时引用：

| 项目 | 用途 |
|------|------|
| 001_XGSampleDemo | 第 31 章：演示 WebSocket Server/Client 基础功能 |
| 010_XGMultiGame | 多人游戏客户端，通过 WSM 与分布式服务器通信 |
| 011_XGMultiManage | 多人管理服务端，通过 WSM 管理多容器连接 |

.uproject 中的引用方式：

```json
// 010_XGMultiGame.uproject
"Plugins": [
    {"Name": "XGSampleWSM", "Enabled": true}
]

// 011_XGMultiManage.uproject
"Plugins": [
    {"Name": "XGSampleWSM", "Enabled": true}
]
```

插件物理位置：`001_XGSampleDemo/Plugins/XGSampleWSM/`，其他项目通过 `ENGINE_DIR/../../` 相对路径引用，或通过 `AdditionalPluginDirectories` 搜索路径。

## 四、独立程序模式

### 4.1 创建流程

```
1. 复制空白模板（XGBlankProgram）→ 重命名为新程序名
2. 修改 .Build.cs：按需添加模块依赖（HTTP/Networking/Core 等）
3. 实现主入口 INT32_MAIN_INT32_ARGC_TCHAR_ARGV()
4. 实现平台兼容处理（PlatformWorkarounds.cpp）
5. 手动打包：拷贝可执行文件 + 最小依赖 DLL
```

### 4.2 独立程序 vs 插件

| 维度 | 插件 | 独立程序 |
|------|------|---------|
| 生命周期 | 随 UE 项目（Editor/Game）启动 | 独立进程 |
| 依赖 | 依赖主项目 Module 体系 | 独立 Build.cs |
| 蓝图 | 可通过 BPLibrary 暴露接口 | 无蓝图 |
| 打包 | 随主项目打包 | 单独手动打包 |
| 适用场景 | 游戏内功能 | 纯服务端 / 后台工具 |

### 4.3 独立程序的三种应用

| 案例 | 程序 | 用途 | 模块依赖 |
|------|------|------|---------|
| 第 23 章 | UEProgram1 | Slate UI 展示 | Core, Slate, SlateCore |
| 第 26 章 | XGSampleServer | HTTP 登录注册服务器 | Core, HTTPServer, JSON |
| 第 26 章对比 | XGSlateSample | Slate 独立模板（对比参考） | Core, Slate, SlateCore |

## 五、多项目拓扑

整个第 21~36 章的 12 个案例形成了三级项目拓扑：

```
第一级：主项目 001_XGSampleDemo
  ├── 自身 C++ 文件（源生代码）
  ├── Plugins/XGSampleWebP（第三方库封装）
  ├── Plugins/XGSampleClient（HTTP 客户端）
  ├── Plugins/XGSamplePicture（摄像头）
  ├── Plugins/XGSampleLink（AI 平台集成）
  ├── Plugins/XGSampleWSM（WebSocket 通信）
  ├── Plugins/XGSampleEMail（TCP 邮件）
  └── Plugins/XGSampleTool（通用工具）

第二级：独立程序
  ├── 007_UEProgram1（Slate 独立程序）
  ├── 013_XGSampleServer（HTTP 独立服务器）
  └── 008_UEProgram2（HTTP 独立程序模板）

第三级：独立游戏项目
  ├── 010_XGMultiGame（多人游戏客户端 → 引用 XGSampleWSM）
  ├── 011_XGMultiManage（多人管理服务端 → 引用 XGSampleWSM）
  ├── 016_XGNetDemo（网络演示 → 无插件依赖）
  └── 017_XGRPG（GAS 案例 → 引用 GameplayAbilities + ModularGameplay）
```

**插件共享流向**：

```
XGSampleWSM 插件 ← 001_XGSampleDemo（第 31 章演示）
                ← 010_XGMultiGame（多人游戏）
                ← 011_XGMultiManage（多人管理）

加密库（XFLinkLibrary） ← XGSampleBDLink（百度签名）
                         ← XGSampleXFLink（讯飞签名）
```

## 六、选型决策

### 何时选择哪种组织形式

```
功能需要跨项目复用？
  ├─ 是 → 需要脱离引擎运行？
  │        ├─ 是 → 独立程序（TSharedFromThis AsyncAction + 无 Engine 依赖）
  │        └─ 否 → 插件（选择模板：Blank 从零构建）
  └─ 否 → 功能复杂度高？
           ├─ 是 → 插件（便于模块化管理）
           └─ 否 → 主项目内 C++ 文件（单文件，简洁直接）

插件是否需要暴露给蓝图？
  ├─ 是 → BPLibrary + Subsystem + Delegate 三层架构
  └─ 否 → 纯 C++ 静态接口（如加密库）

插件是否需要同时支持服务端和客户端？
  ├─ 是 → 插件内双 Subsystem 设计（如 XGSampleWSM）
  └─ 否 → 单 Subsystem 即可
```

### 各组织方式的特征总结

| 组织方式 | 复用性 | 复杂度 | 蓝图支持 | 引擎依赖 | 典型例子 |
|---------|--------|--------|---------|---------|---------|
| 主项目内 C++ 文件 | 无 | 低 | 需额外暴露 | 全引擎 | 第 25 章：XGSampleHttpTime |
| 插件（UObject 体系） | 中 | 中 | 原生支持 | UE 项目 | 第 26 章：XGSampleClient |
| 插件（全静态 C++） | 高 | 高 | 通过 BPLibrary | 无 | 第 32 章：XGSampleEMail |
| 独立程序 | 高 | 高 | 不支持 | 最小依赖 | 第 23 章：UEProgram1 |
| 独立游戏项目 | 中 | 高 | 原生支持 | 全引擎 | 第 36 章：XGRPG |
