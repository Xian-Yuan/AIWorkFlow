# 服务端框架搭建与独立程序入口

## 项目创建方式

XGSampleServer 从 `XGBlankProgram` 模板复制而来。BlankProgram 是 UE 源码引擎中最简的独立程序模板，位于 `Engine\Source\Programs\BlankProgram\`。

相比于第二十三章从零搭建 Slate 独立程序，本章的服务端更接近 BlankProgram 的原生形态——控制台程序、无 UI、靠手工循环驱动。

## 工程配置

### Build.cs

路径：[XGSampleServer.Build.cs](../code/013_独立程序源码/XGSampleServer/XGSampleServer.Build.cs)

```csharp
PublicDependencyModuleNames.AddRange(new string[] { "Core", "CoreUObject", "Engine" });
PrivateDependencyModuleNames.AddRange(new string[] {
    "Projects", "ApplicationCore",
    "HTTP", "HTTPServer", "Json", "JsonUtilities",
});
```

与服务端核心业务相关的模块依赖：
- `HTTPServer` — 提供 `IHttpServerModule` / `IHttpRouter` / `FHttpServerRequest` 等 HTTP 服务端类型
- `HTTP` — 提供 `FHttpModule`（虽然服务端本身不发出 HTTP 请求，但 `HTTPServer` 可能间接依赖）
- `Json` + `JsonUtilities` — 提供 `FJsonObjectConverter` 做 JSON ↔ UStruct 序列化

对比第二十三章 Slate 依赖的是 `AppFramework` / `Slate` / `SlateCore` / `StandaloneRenderer`。

### Target.cs

路径：[XGSampleServer.Target.cs](../code/013_独立程序源码/XGSampleServer/XGSampleServer.Target.cs)

```csharp
Type = TargetType.Program;
LinkType = TargetLinkType.Monolithic;
LaunchModuleName = "XGSampleServer";
bIsBuildingConsoleApplication = true;
bCompileAgainstCoreUObject = true;
bCompileAgainstApplicationCore = true;
bBuildDeveloperTools = false;
bBuildWithEditorOnlyData = false;
bCompileAgainstEngine = false;
bCompileICU = false;
```

关键配置：
- `TargetType.Program` — 声明为独立程序而非游戏/编辑器
- `TargetLinkType.Monolithic` — 单体编译（所有代码链接到一个 EXE）
- `bIsBuildingConsoleApplication = true` — 控制台程序，有命令行窗口
- `bCompileAgainstCoreUObject = true` — 需要 UObject 支持（因为用了 UXGSampleServerHttpObject）
- `bCompileICU = false` — 跳过 ICU 国际化库编译，加速构建

对比第二十三章 Slate 程序的 Target.cs，差异在于：
- XGSlateSample 是 `bIsBuildingConsoleApplication = false`（Windows GUI）
- XGSlateSample 有 `ExeBinariesSubFolder = "XGPrograms"`（打包子文件夹）

## 主入口文件

路径：[XGSampleServer.cpp](../code/013_独立程序源码/XGSampleServer/Private/XGSampleServer.cpp)

### 入口宏

```cpp
IMPLEMENT_APPLICATION(XGSampleServer, "XGSampleServer");
INT32_MAIN_INT32_ARGC_TCHAR_ARGV()
{
    // ...
}
```

这是所有 UE 独立程序的标配入口模式。`IMPLEMENT_APPLICATION` 声明模块入口点，`INT32_MAIN_INT32_ARGC_TCHAR_ARGV()` 创建控制台主函数签名。第二十三章 Slate 程序使用的是 `WinMain`（Windows 应用程序入口），这是唯一的入口差异。

### 初始化序列（5 步）

1. **GEngineLoop.PreInit** — 初始化引擎子系统（任务图、Tick、核心模块）
2. **INI 配置加载** — `FXGSampleServerConfigManage::Get()->Init()` 读取自定义 INI
3. **创建服务端单例** — `UXGSampleServerHttpObject::Get()->Init()` 创建 UObject 单例
4. **绑定端口并启动监听** — 在上一步 Init() 内部调用 `Server->StartAllListeners()`
5. **进入主循环** — `while (!IsEngineExitRequested())` 循环

```cpp
GEngineLoop.PreInit(ArgC, ArgV);
UE_LOG(LogXGSampleServer, Display, TEXT("XGSampleServer PreInit Success"));

FXGSampleServerConfigManage::Get()->Init();
UE_LOG(LogXGSampleServer, Display, TEXT("XGSampleServer Init Config Success"));

UXGSampleServerHttpObject::Get()->Init();
UE_LOG(LogXGSampleServer, Display, TEXT("XGSampleServer Init Server Success"));

while (!IsEngineExitRequested()) { /* 主循环 */ }

FXGSampleServerConfigManage::Get()->Destory();
UXGSampleServerHttpObject::Get()->Destory();
GEngineLoop.Exit();
```

### 主循环结构

```cpp
while (!IsEngineExitRequested())
{
    FPlatformProcess::Sleep(0.01f);
    if (!IsEngineExitRequested())
    {
        FTaskGraphInterface::Get().ProcessThreadUntilIdle();
        FTSTicker::GetCoreTicker().Tick(STATS);
        GFrameCounter++;
    }
}
```

三个核心组件：
- `Sleep(0.01f)` — 控制帧率约 100 FPS（比第二十三章 Slate 的 `1.0f/60` 更紧凑）
- `FTaskGraphInterface::Get().ProcessThreadUntilIdle()` — 处理任务图积压任务
- `FTSTicker::GetCoreTicker().Tick()` — 驱动核心 Tick 系统

此外每 10 秒进行一次自检日志输出（通过 `GFrameCounter % 1000 == 0` 判断）。

### 关闭序列

```cpp
BeginExitIfRequested();
```

通过控制台 Ctrl+C 触发 `BeginExitIfRequested()` 设置退出标志，主循环自然退出后执行：
1. `FXGSampleServerConfigManage::Get()->Destory()` — 销毁配置单例
2. `UXGSampleServerHttpObject::Get()->Destory()` — 销毁 HTTP 服务端
3. `GEngineLoop.Exit()` — 引擎退出

对比第二十三章，Slate 程序使用 `FSlateApplication::Shutdown()` 关闭 Slate 应用，而纯控制台程序不需要这一步骤。

## PlatformWorkarounds

路径：[PlatformWorkarounds.cpp](../code/013_独立程序源码/XGSampleServer/Private/PlatformWorkarounds.cpp)

```cpp
FString GFileRootDirectory;
FString GSandboxName;
```

这两个全局变量是 UE 独立程序的启动修复手段，在 BlankProgram 和 XGSlateSample 中同样存在。它们在 PreInit 前被设置，用于绕过独立程序下文件系统路径的初始化和沙箱目录缺失问题。

## 目录结构

```
XGSampleServer/
├── XGSampleServer.Build.cs
├── XGSampleServer.Target.cs
├── Resources/Windows/
│   └── BlankProgram.ico          ← 程序图标（与 BlankProgram 共享）
└── Private/
    ├── XGSampleServer.cpp         ← 主入口 + 主循环
    ├── XGSampleServer.h           ← 仅包含 CoreMinimal.h
    ├── PlatformWorkarounds.cpp    ← 路径修复全局变量
    ├── Config/
    │   ├── XGSampleServerConfig.h
    │   └── XGSampleServerConfig.cpp
    ├── Log/
    │   └── LogXGSampleServer.h
    ├── ServerObject/
    │   ├── XGSampleServerHttpObject.h
    │   └── XGSampleServerHttpObject.cpp
    ├── Type/
    │   ├── XGSampleServerHttpType.h
    │   ├── XGSampleServerRequestType.h
    │   ├── XGSampleServerResponseType.h
    │   └── XGSampleServerServerType.h
    └── Util/
        ├── XGSampleServerUtil.h
        └── XGSampleServerUtil.cpp
```

对比第二十三章 XGSlateSample 的目录结构，差异主要体现在：
- **ServerObject/** — 替代了 XGSlateSample 的 **Core/** 目录，核心单例从 `FXGSSPCore`（原生 C++ 类）变为 `UXGSampleServerHttpObject`（UObject 子类）
- **Type/** — 大量 USTRUCT 定义文件，XGSlateSample 没有对应的类型目录
- **无 Slate/** 目录 — 不需要 UI 控件
