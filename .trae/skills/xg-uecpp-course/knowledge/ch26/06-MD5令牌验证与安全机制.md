# MD5 令牌验证与安全机制

## 概述

本章实现了基于 MD5 哈希的令牌（Token）验证机制，用于客户端和服务端之间的身份校验。令牌设计为**可配置迭代次数**，增加破解难度。

## Token 生成算法

路径：[XGSampleServerUtil.h](../code/013_独立程序源码/XGSampleServer/Private/Util/XGSampleServerUtil.h)，[XGSampleServerUtil.cpp](../code/013_独立程序源码/XGSampleServer/Private/Util/XGSampleServerUtil.cpp)

```cpp
FString UXGSampleServerUtil::EncryptionToken(const FString& InToken, int32 InMD5Num)
{
    FString Token = InToken;
    for (int32 i = 0; i < InMD5Num; i++)
        Token = FMD5::HashAnsiString(*Token);
    return Token;
}
```

算法逻辑：
1. 从 INI 读取基准 Token 字符串（如 `"XiaoGangNB"`）
2. 从 INI 读取 MD5 迭代次数（如 `5`）
3. 对 Token 进行 `InMD5Num` 次连续 MD5 哈希
4. 每次哈希的输出作为下一次哈希的输入
5. 返回最终的 Token 字符串

`FMD5::HashAnsiString()` 是 UE 内置的静态方法，以 ANSI 字符串形式计算 MD5。

## HTTP Header 传递

Token 通过 HTTP 请求头的 `XGLoginServerToken` 字段传递：

```
POST /Login HTTP/1.1
Host: 127.0.0.1:11235
Content-Type: application/json
XGLoginServerToken: A1B2C3D4E5F6...
```

## 客户端 Token 计算

客户端在发送请求前计算 Token，通过 INI 配置读取相同的 Token 基准值和 MD5 迭代次数：

```cpp
// 客户端 XGSampleClientSettings.h
FString Token = "XiaoGangNB";
int32 MD5Num = 5;

// 发送请求时计算 Token
FString CurrentToken = UXGSampleServerUtil::EncryptionToken(
    UXGSampleClientSettings::Get()->Token,
    UXGSampleClientSettings::Get()->MD5Num
);
```

## 服务端 Token 验证

服务端在每个请求到达时重新计算 Token 并与请求头比较：

```cpp
FString UXGSampleServerHttpObject::GetCurrentToken()
{
    auto& ConfigInfo = FXGSampleServerConfigManage::Get()->GetInfo();
    return UXGSampleServerUtil::EncryptionToken(ConfigInfo.Token, ConfigInfo.MD5Num);
}
```

通过每次请求时重新计算（而不是缓存 Token），确保 INI 配置变更后立即生效。

## 安全设计要点

| 设计 | 说明 |
|------|------|
| 可配置 Token 值 | Token 明文保存在 INI 中，可随时更换 |
| 可配置迭代次数 | `MD5Num` 控制哈希链长度，增加彩虹表破解成本 |
| 链式哈希 | 每次哈希结果作为下一次输入，形成哈希链 |
| 每次请求重新计算 | 配置变更后无需重启服务端 |
| 无缓存机制 | Token 不驻留内存，避免内存 dump 泄露 |

## 密码存储

用户密码同样使用 MD5 存储，不保存明文：

```cpp
UserInfo.Password = FMD5::HashAnsiString(TEXT("123456"));
```

这确保了即使 UserList 内存被 dump，也无法直接获取用户原始密码。

## 安全局限性

当前实现存在以下需要注意的安全局限：

- Token 明文存储在 INI 文件中：任何能访问服务器文件系统的人都能读取 Token 基准值
- MD5 作为密码哈希已不推荐：MD5 已被证明存在碰撞漏洞，生产环境建议使用 SHA256 或 bcrypt
- HTTP 明文传输：未使用 HTTPS，Token 和密码在传输过程中可能被中间人截获
- Token 无有效期：Token 在服务器运行期间保持不变，不做过期刷新

本章重点在于演示 UE 独立程序的 HTTP 通信和自定义安全机制的设计思路，并非生产级安全解决方案。
