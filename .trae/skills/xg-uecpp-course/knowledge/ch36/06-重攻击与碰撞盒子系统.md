# 重攻击与碰撞盒子系统

## 概述

重攻击通过碰撞盒子（HitBox）系统实现伤害检测，区别于轻攻击的直接 GE 应用方式。碰撞盒子是一个在蒙太奇播放到特定帧时生成的 Actor，用于检测攻击范围内的敌人。

## HitBox 架构

项目包含两个碰撞盒子类，分别对应不同复杂度：

### AXGRPGHitBox（简单碰撞盒子）

适用于简单近战范围检测：

```cpp
class AXGRPGHitBox : public AActor
{
    USceneComponent* HitCollisionRootComponent;  // 根组件
    UBoxComponent* HitDamage;                     // 碰撞检测盒子
    TArray<TWeakObjectPtr<AXGRPGCharacterBase>> AttackedTarget;  // 已攻击目标
    
    void HandleDamage();                           // 重叠事件回调
    void SetBoxExtent(FVector);                    // 设置盒子大小
    bool IsExist(AXGRPGCharacterBase*);             // 判重
};
```

### AXGRPGHitBox_ApplyGameEffect（进阶碰撞盒子）

支持更多碰撞类型和弹道移动：

```cpp
class AXGRPGHitBox_ApplyGameEffect : public AActor
{
    USceneComponent* HitCollisionRootComponent;
    UBoxComponent* HitDamage;
    UProjectileMovementComponent* ProjectileMovement;   // 弹道组件
    EXGRPGHitCollisionType HitCollisionType;             // 碰撞类型
    TSubclassOf<UGameplayEffect> EffectClass;            // 要应用的 GE
    
    void PreInitCollision(AActor*);                      // 预初始化
    void HandleDamage();                                  // 重叠事件回调
};
```

## 碰撞类型枚举（EXGRPGHitCollisionType）

| 枚举值 | 描述 | 典型用途 |
|-------|------|---------|
| `HITCOLLISIONTYPE_SHORT_RANGE_ATTACK` | 近程攻击 | 重攻击近战 |
| `HITCOLLISIONTYPE_DIRECT_LINE` | 无障碍直线 | 穿透性攻击 |
| `HITCOLLISIONTYPE_LINE` | 非跟踪直线 | 手枪子弹 |
| `HITCOLLISIONTYPE_TRACK_LINE` | 跟踪类型 | 追踪弹 |
| `HITCOLLISIONTYPE_RANGE_LINE` | 范围伤害（投掷） | 丢手雷 |
| `HITCOLLISIONTYPE_RANGE` | 范围伤害（自爆） | 自爆技能 |
| `HITCOLLISIONTYPE_CHAIN` | 链条类型 | 持续伤害 |

## 重攻击流程

```
重攻击 GA 激活
    → PlayMontage（重攻击蒙太奇）
        → AnimNotify 触发
            → SpawnActor<AXGRPGHitBox>（生成碰撞盒子）
                → 用 SetBoxExtent 设置碰撞区域大小和位置
                → 启动 OnBeginOverlap 检测
                    → 检测到 Actor 进入碰撞区域
                        → 判断是否已对该目标生效（IsExist）
                        → 判重通过后应用 GE → 触发伤害
```

## 动画通知驱动

碰撞盒子的生成时机由动画通知精确控制：

- `AnimNotify_Attack`：在蒙太奇的特定帧通知 GA 生成碰撞盒子
- 碰撞盒子的持续时间为动画通知区的长度

这种基于动画帧的精确控制避免了计时器的不确定性，确保伤害判定与动画表现同步。

## 判重机制

```cpp
bool AXGRPGHitBox::IsExist(AXGRPGCharacterBase* Target)
{
    for (auto& Attacked : AttackedTarget)
    {
        if (Attacked.IsValid() && Attacked.Get() == Target)
            return true;
    }
    return false;
}
```

`AttackedTarget` 数组使用 `TWeakObjectPtr` 存储已攻击目标，确保当目标被销毁时不会导致悬挂指针。每次 OnBeginOverlap 时先判重，避免在一次攻击中对同一目标多次造成伤害。

## 位置与范围设置

碰撞盒子通过 `SetHitDamageRelativePosition` 和 `SetBoxExtent` 两个方法配置：

- 相对位置：相对于角色根骨骼的偏移（由蒙太奇中的 Animation Notify 传递）
- 范围：UBoxComponent 的尺寸（可根据攻击类型动态调整）

## 注意事项

- 碰撞盒子是瞬时生成的 Actor，生命周期有限
- 仅在专属服务器上处理伤害逻辑，客户端不生成碰撞盒子的伤害
- 判重机制对多段伤害（如链条类型 HITCOLLISIONTYPE_CHAIN）很重要，需确保 `AttackedTarget` 适时清空
- `TWeakObjectPtr` 自动处理目标销毁场景，但不影响对象的 GC 时序
- 碰撞盒子应在蒙太奇结束后立即销毁或设置很短的 InitialLifeSpan
