# 连击系统

## 组件架构

`XGRPGComboComponent` 继承自 `UActorComponent`，负责管理连击索引、输入窗口和蒙太奇播放。

### 核心状态

```cpp
class XGRPGComboComponent : public UActorComponent
{
    int32 ComboIndex;          // 当前连击索引（0, 1, 2, ...）
    bool bNextComboAble;       // 是否允许下一段连击
    bool bInputTriggered;      // 输入是否已触发
    TArray<UAnimMontage*> LightAttackMontages;  // 轻攻击蒙太奇数组
    // ...
};
```

### 关键设计决策

连击组件**不开启网络复制**。这是一个有意的设计选择——连击的状态完全由本地决定，服务端只验证最终执行的伤害结果。这样做避免了网络时序带来的连击判定问题（如玩家在客户端点了 3 下，但网络延迟导致服务端只收到 2 下）。

## 连击流程

```
玩家按下攻击键
    → InputTag 匹配 → TryActivateAbility
        → GA 激活 → PlayMontage（连击蒙太奇[ComboIndex]）
            → AnimNotifyState_NextCombo（打开输入窗口）
                → 玩家再次按下攻击键
                    → ComboIndex++ → PlayMontage（下一段蒙太奇）
            → AnimNotifyState_NextCombo 结束（关闭窗口）
        → 蒙太奇结束
            → 如果 ComboIndex 小于数组长度：等待下次输入
            → 否则：重置 ComboIndex
```

## 关键动画通知

### AnimNotifyState_NextCombo（连击窗口控制）

在蒙太奇中标记一个时间段（Notify State），在这段时间内允许下一段连击触发。范围之外按下攻击键不会触发连击。

### AnimNotify_Attack（实际攻击判定）

当蒙太奇播放到特定帧时触发，通知 GA 执行伤害判定逻辑（生成碰撞盒子或应用 GE）。

### AnimNotify_ResetCombo（连击重置）

当连击完成或被中断时，将 ComboIndex 重置为 0。

## 轻攻击 vs 重攻击

| 特性 | 轻攻击 | 重攻击 |
|------|--------|--------|
| 触发方式 | 按一次攻击键 | 长按或单独按键 |
| 连击机制 | 多段连击（ComboIndex++） | 单次攻击 |
| 伤害检测 | 通过 GE 直接应用 | 生成碰撞盒子（HitBox） |
| 动画 | 多段蒙太奇 | 独立蒙太奇 |
| 可被打断 | 是 | 否 |

## 槽位（Slot）

**动画蓝图中的 Slot 节点是必须的**，否则蒙太奇无法播放。如果角色播放了蒙太奇但看不到动画效果，首先检查动画蓝图中是否添加了对应 Slot 节点并与蒙太奇的 Slot Name 一致。

## 注意事项

- 连击组件不开启复制，避免网络时序问题
- 蒙太奇数组顺序必须与连击索引一一对应
- 必须在动画蓝图中添加匹配的 Slot 节点
- 连击窗口通过 `AnimNotifyState` 控制，而不是通过计时器
- 专属服务器上动画通知不会执行，需要 `GetNetMode != NM_DedicatedServer` 排除
