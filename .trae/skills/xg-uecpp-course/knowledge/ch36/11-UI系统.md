# UI 系统

## UI 架构概览

项目 UI 系统采用 C++ 编写逻辑、蓝图配置表现的混合模式。UI 基类构成如下：

```
UUI_Base（通用基类）
├── UUI_GameMain（主游戏界面）
├── UUI_CharacterInfo（角色属性面板）
├── UUI_DamgeNumber（伤害数字控件）
├── UUI_SlotBase（插槽基类）
│   ├── UUI_InventorySlot（背包插槽）
│   └── UUI_EquipmentSlot（装备插槽）
├── UUI_InventoryPanel（背包面板）
├── UUI_EquipmentPanel（装备面板）
└── UUI_ICONDragDrog（拖拽图标）
```

## UI 基类（UUI_Base）

通用基类提供所有 UI 子类共用的基础设施：

```cpp
class UUI_Base : public UUserWidget
{
    // 通用方法
    virtual void NativeConstruct() override;
    virtual void NativeDestruct() override;
    virtual void NativeTick(const FGeometry& MyGeometry, float InDeltaTime) override;
};
```

## 角色属性面板（UUI_CharacterInfo）

### C++ 定义

```cpp
class UUI_CharacterInfo : public UUI_Base
{
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Health;      // 生命值
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Mana;        // 法力值
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Stamina;     // 耐力值
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Attack;      // 攻击力
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Defense;     // 防御力
    UPROPERTY(meta = (BindWidget))
    UTextBlock* Exp;         // 经验值
};
```

### BindWidget 宏

`BindWidget` 是 UPROPERTY 的元数据标识，自动将 C++ 变量与蓝图中同名控件绑定。蓝图侧必须在 Widget Designer 中创建同名 TextBlock 控件，运行时系统自动匹配。

### 每帧属性读取

```cpp
void UUI_CharacterInfo::NativeTick(const FGeometry& MyGeometry, float InDeltaTime)
{
    Super::NativeTick(MyGeometry, InDeltaTime);
    
    AXGRPGCharacterBase* Character = GetOwningCharacter();
    if (!Character) return;
    
    UXGRPGAbilitySystemComponent* ASC = Character->GetAbilitySystemComponent();
    if (!ASC) return;
    
    const UXGRPGCharacterArrtibuteSet* AttributeSet = ASC->GetSet<UXGRPGCharacterArrtibuteSet>();
    if (!AttributeSet) return;
    
    Health->SetText(FText::FromString(
        FString::SanitizeFloat(AttributeSet->GetHealth())));
    Mana->SetText(FText::FromString(
        FString::SanitizeFloat(AttributeSet->GetMana())));
    // ...
}
```

这种每帧读取方式实现简单，不需要绑定事件委托。缺点是性能开销大于事件驱动方式，但在角色属性面板更新的场景下可接受。

### 生命周期方法

| 方法 | 对应蓝图事件 | 用途 |
|------|-------------|------|
| `NativeConstruct` | Construct | 初始化控件、绑定事件 |
| `NativeTick` | Tick | 每帧更新属性显示 |
| `NativeDestruct` | Destruct | 清理资源、解绑事件 |

**必须调用父类版本**，否则代码不执行且很难排查：

```cpp
void UUI_CharacterInfo::NativeConstruct()
{
    Super::NativeConstruct();  // 必须调用
    // 自定义初始化逻辑...
}
```

### 获取 Owner Character

```cpp
AXGRPGCharacterBase* UUI_Base::GetOwningCharacter() const
{
    if (APlayerController* PC = GetOwningPlayer())
    {
        return Cast<AXGRPGCharacterBase>(PC->GetPawn());
    }
    return nullptr;
}
```

## 主游戏 UI（UUI_GameMain）

主界面容器，管理各子面板的添加和生命周期：

```cpp
class UUI_GameMain : public UUI_Base
{
    // 生命周期
    void NativeConstruct() override;
    void NativeDestruct() override;
    void NativeTick(const FGeometry& MyGeometry, float InDeltaTime) override;
};
```

### 创建延迟

**HUD 的创建需要延迟**（约 0.2 秒），因为关卡加载时 ASC 和 AttributeSet 等组件还未完全初始化。延迟确保所有组件就绪后再创建 UI 面板。

### 全局 Drop 事件

`NativeOnDrop` 处理从控件外拖拽到主界面的事件，支持将物品从背包拖到外部丢弃等交互。

## 基地 HUD（XGRPGHUD）

```cpp
class AXGRPGHUD : public AHUD
{
    // 当前为空实现
};
```

AHUD 是 UE 内置的 HUD 类，这里留空，实际 UI 创建和管理由 `UUI_GameMain` 处理。AHUD 作为框架预留。

## 蓝图层配置

所有 UI 的蓝图层只需完成以下操作：

1. 创建 Widget Blueprint，继承自对应的 C++ 类
2. 在 Designer 中添加与 C++ `BindWidget` 变量同名的控件
3. 配置控件的初始外观（颜色、字体、位置等）
4. 将 C++ 中 `PlayAnimation` 引用的动画 k 帧并命名

## 注意事项

- `BindWidget` 绑定失败时不会报错，只会显示空白，需检查蓝图中控件名称是否一致
- `FString::SanitizeFloat` 会导致较多小数位，必要时应使用格式化 `FString::Printf`
- `NativeTick` 中不要做耗时操作，属性读取比较轻量但仍有开销
- 蓝图侧不需要写任何 Event Graph 逻辑，所有业务逻辑在 C++ 中完成
- 创建 HUD 时需考虑时机问题，建议用 Delay 或 OnPossessed 等回调
