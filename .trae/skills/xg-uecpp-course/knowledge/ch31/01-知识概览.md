# 第三十一章：知识概览

## 章节定位

第三十一章是插件开发的核心章节——从零构建一个**自包含的 WebSocket 服务器插件**，使 UE 项目具备运行 WebSocket 服务端的能力。区别于第二十九章（STT 客户端）和第三十章（TTS 客户端）的纯客户端模式，本章开发的是**服务端 + 客户端双角色**插件，服务于 MMO 分布式服务器间通信场景。

## 关键技术选择

| 决策 | 选择 | 原因 |
|------|------|------|
| 服务端 WebSocket | `IWebSocketServer`（WebSocketNetworking 插件） | UE 内置，无需第三方库 |
| 客户端 WebSocket | `IWebSocket`（WebSockets 模块） | 跨平台支持 |
| 实例模式 | `UGameInstanceSubsystem` | 生命周期与 GameInstance 一致，蓝图可访问 |
| 消息格式 | JSON（`FJsonObjectConverter`） | 调试友好，结构灵活 |
| 传输编码 | Base64 | 二进制安全，跨平台兼容 |
| 连接标识 | FGuid（配对双 GUID） | CollectionID + ServerConnectionID 全局唯一 |
| 平台限制 | 服务端 Win/Mac/Linux | WebSocketNetworking 不支持移动端服务端 |
