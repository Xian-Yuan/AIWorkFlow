# MMOARPG 分布式服务器架构与通讯拓扑

## 服务器层次结构

MMOARPG 采用分层分布式服务器架构，每层承担不同职责：

```
外部网络（Internet）
     │
     ▼
┌──────────────────────┐
│  Login Server        │  登录验证 + 网关分发
│  / Gateway           │  对外暴露，玩家通过此入口接入
└──────────┬───────────┘
           │ 内部网络（Intranet / LAN）
           ▼
┌──────────────────────┐
│  Logic Server        │  游戏逻辑运算
│                      │  处理玩家行为、战斗判定、技能触发
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  DS                  │  Dedicated Server（专用服务器）
│  (Dedicated Server)  │  无渲染、无音频、无动画的纯逻辑服务器
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Center Server       │  中心调度服务器
│                      │  协调各 Instance Server 的创建/销毁/分配
└──────────┬───────────┘
           │
           ▼
┌──────────────────────────────────────────────┐
│  Instance Server 1 │ Instance Server 2 │ ... │  副本/房间服务器
│  (副本/房间服务器)   │                     │     │  每个副本实例独立进程
└──────────────────────────────────────────────┘
```

## 服务器类型说明

| 类型 | 说明 | 使用场景 |
|------|------|---------|
| **Dedicated Server（DS）** | 无渲染/无音频/无动画的纯逻辑服务器进程 | 生产环境多人游戏 |
| **Listen Server（LS）** | 主机玩家同时作为服务器运行 | 合作模式、小型联机 |
| **Standalone** | 单机模式，在本地运行全部逻辑 | 单人游戏、调试 |
| **Center Server** | 中心调度服务，管理副本实例的创建和分配 | MMO 副本系统 |
| **Instance Server** | 每个副本/房间的独立服务器进程 | 副本、战场、房间 |

## 通讯拓扑

### 三种容器类型

```
1. 客户端容器（Client Container）
   ┌─────────┐     ┌──────────┐
   │ 客户端   │────→│ 服务器 A  │
   │(单进程)  │────→│ 服务器 B  │  一个客户端同时连接多个服务器
   │         │────→│ 服务器 C  │
   └─────────┘     └──────────┘

2. 服务器容器（Server Container）
   ┌─────────────┐
   │ 客户端 A ────│
   │ 客户端 B ────│──→ 服务器  一个服务器同时接受多个客户端
   │ 客户端 C ────│
   └─────────────┘

3. 多对多容器（Many-to-Many）
   ┌──────────┐     ┌──────────────┐
   │Center    │────→│ Instance 1   │
   │Server    │────→│ Instance 2   │  中心服务器管理多个实例
   │          │────→│ Instance N   │
   └──────────┘     └──────────────┘
```

### 容器特性

| 容器类型 | 结构 | 连接标识 | 典型场景 |
|---------|------|---------|---------|
| 客户端容器 | 一个客户端持有 N 个 WebSocket | 通过 FGuid 区分不同连接 | 客户端同时连接 Login + Logic 服务器 |
| 服务器容器 | 服务器持有 N 个 Client 的连接 | TMap<FGuid, ServerConnection> | DS 同时接受多个玩家连接 |
| 多对多容器 | M 个服务器 ↔ N 个服务器 | 双 GUID 配对 | Center Server 管理 Instance Server 集群 |

## 端口分配策略

| 服务器 | 端口示例 | 说明 |
|--------|---------|------|
| Login Server + Gateway | 7788 | 外部暴露，玩家接入 |
| Logic Server | 7777 | 内部通信 |
| DS（Dedicated Server） | 7777 | UE 默认 DS 端口 |
| Center Server | 9083（本章测试） | 管理与调度 |
| Instance Server | 动态分配 | Center Server 分配 |

## X 形连接关系

```
服务器端 X 形关闭关系：
  客户端                 服务器
    │                     │
    │──── ClientReqQuit ─→│  ① 客户端请求退出
    │                     │  状态 → PrepareToQuit
    │←── ServerAllowQuit ─│  ② 服务器允许退出
    │                     │  状态 → Quit → 释放资源
    
    │←── ServerReqQuit ───│  ① 服务器请求断开
    │                     │  状态 → Quit
    │──── ClientAllowQuit→│  ② 客户端确认断开
    │                     │  释放资源
```

任一方都可以主动发起关闭请求，另一方确认后完成断开。这种对称的握手关闭机制保证双方都能安全释放资源。
