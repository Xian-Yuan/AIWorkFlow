# 增强输入系统

## 概述

Enhanced Input 是 UE5 的新输入框架，替代旧的 Action/Axis 输入方式。核心资产包括 InputAction（输入动作）和 InputMappingContext（映射上下文），通过 UEnhancedInputLocalPlayerSubsystem 注册和管理。

## 资产体系

| 资产 | 用途 | 示例 |
|------|------|------|
| **InputAction** | 定义输入动作的名称、值类型（bool/float/Vector2D）和触发器 | `IA_Move`（Vector2D）、`IA_Jump`（bool） |
| **InputMappingContext (IMC)** | 将 InputAction 绑定到具体按键，管理按键冲突和优先级 | WASD → IA_Move |
| **Modifier** | 按键值后处理（取反、死区、映射范围） | 负方向取反、摇杆死区 |

## C++ 绑定流程

### 1. 模块依赖

```csharp
PublicDependencyModuleNames.AddRange(new[] { "EnhancedInput" });
```

### 2. 资产声明

```cpp
class AMyCharacter : public ACharacter
{
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = Input)
    UInputMappingContext* IMC_Default;

    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = Input)
    UInputAction* IA_Move;

    virtual void SetupPlayerInputComponent(UInputComponent* PlayerInputComponent) override;
};
```

### 3. 绑定实现

```cpp
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* PlayerInputComponent)
{
    UEnhancedInputComponent* Input = Cast<UEnhancedInputComponent>(PlayerInputComponent);

    Input->BindAction(IA_Move, ETriggerEvent::Triggered, this, &AMyCharacter::OnMove);

    // 注册 IMC
    if (APlayerController* PC = Cast<APlayerController>(GetController()))
    {
        UEnhancedInputLocalPlayerSubsystem* Subsystem = 
            ULocalPlayer::GetSubsystem<UEnhancedInputLocalPlayerSubsystem>(PC->GetLocalPlayer());
        Subsystem->AddMappingContext(IMC_Default, 0);
    }
}

void AMyCharacter::OnMove(const FInputActionValue& Value)
{
    FVector2D MoveVector = Value.Get<FVector2D>();
    AddMovementInput(GetActorForwardVector(), MoveVector.Y);
    AddMovementInput(GetActorRightVector(), MoveVector.X);
}
```

## IMC 注册与优先级

```cpp
Subsystem->AddMappingContext(IMC_Default, 0);       // 基础层
Subsystem->AddMappingContext(IMC_Weapon, 10);        // 武器层，覆盖基础层
Subsystem->RemoveMappingContext(IMC_Weapon);          // 移除武器层，恢复基础层
```

优先级数值越大越优先。同一按键在不同 IMC 中绑定的 InputAction 不同时，高优先级覆盖低优先级。

### Transient vs 非 Transient IMC

通过在 IMC 资产中标记 bTickBeforeAnimated 控制 Transient 行为，区分"需要动态弹出/隐藏"的 IMC（如 UI 菜单 IMC）和"始终生效"的 IMC（如基础移动 IMC）。

## 实战模式：武器 IMC 切换

```cpp
void AMyCharacter::EquipWeapon(UInputMappingContext* WeaponIMC)
{
    UEnhancedInputLocalPlayerSubsystem* Subsystem = GetInputSubsystem();
    Subsystem->RemoveMappingContext(IMC_Default);  // 移除基础 IMC
    Subsystem->AddMappingContext(WeaponIMC, 10);    // 启用武器 IMC
}

void AMyCharacter::UnequipWeapon()
{
    UEnhancedInputLocalPlayerSubsystem* Subsystem = GetInputSubsystem();
    Subsystem->RemoveMappingContext(IMC_Weapon);
    Subsystem->AddMappingContext(IMC_Default, 0);
}
```

## 三种项目模板差异

| 模板 | 输入方式 | 角色旋转 | CameraBoom |
|------|---------|---------|-----------|
| **FP** | EnhancedInput + IMC | `bUseControllerRotationYaw = true` | 无（Camera 挂载到枪口） |
| **TP** | EnhancedInput + IMC | `bUseControllerRotationYaw = true` | CameraBoom 跟随角色 |
| **TD** | EnhancedInput（IA_Click） | `bOrientRotationToMovement = true` | CameraBoom 固定俯视角 |
