# 配置与依赖注入

## 三种机制全景

UE 通过三种内建机制实现依赖注入与配置管理，无需第三方 DI 框架：

| 机制 | 角色 | 生命周期 | 章节 |
|------|------|---------|------|
| Subsystem | 运行时单例服务注入 | 引擎/GameInstance/World 生命周期 | Ch13 |
| DeveloperSettings | 类型化配置注入 | 持久化到 INI 文件 | Ch15 |
| CDO | 两种机制的底层对象模型 | 类加载时自动创建 | Ch3+15 |

共同特征：引擎自动管理生命周期、通过反射系统注册、全局可访问。

## Subsystem：运行时单例

### 五种类型

| 类型 | 获取方式 | 生命周期 | 常用度 |
|------|---------|---------|-------|
| UGameInstanceSubsystem | `GetGameInstance()->GetSubsystem<>()` | GameInstance 创建→销毁 | ★★★★★ |
| UWorldSubsystem | `GetWorld()->GetSubsystem<>()` | 关卡加载→卸载 | ★★★ |
| UEngineSubsystem | `GEngine->GetSubsystem<>()` | 引擎启动→关闭 | ★★ |
| ULocalPlayerSubsystem | `GetLocalPlayer()->GetSubsystem<>()` | LocalPlayer 创建→销毁 | ★★ |
| UEditorSubsystem | `GEditor->GetSubsystem<>()` | 编辑器启动→关闭 | ★ |

### 声明模板

```cpp
UCLASS()
class UXGMySubsystem : public UGameInstanceSubsystem, public FTickableGameObject
{
    GENERATED_BODY()
public:
    virtual bool ShouldCreateSubsystem(UObject* Outer) const override;
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

    // FTickableGameObject（可选）
    virtual void Tick(float DeltaTime) override;
    virtual bool IsTickable() const override { return !IsTemplate(); }
    virtual TStatId GetStatId() const override;
};
```

### 三个生命周期重写点

| 重写点 | 作用 | 典型逻辑 |
|--------|------|---------|
| ShouldCreateSubsystem | 条件创建控制 | 返回 bool |
| Initialize | 初始化资源 | 绑定回调、分配资源 |
| Deinitialize | 清理资源 | 置空指针、释放资源 |

### bFirstTick 模式

Subsystem Initialize 中某些依赖未就绪时，用第一个 Tick 做安全操作：

```cpp
bool bFirstTick = true;
void UXGMySubsystem::Tick(float DeltaTime)
{
    if (bFirstTick) { bFirstTick = false; /* 安全操作 */ }
}
```

### 获取方式

```cpp
// AActor 直接获取（推荐）
GetGameInstance()->GetSubsystem<UXGMySubsystem>();

// BlueprintFunctionLibrary
InWorldContextObject->GetWorld()->GetGameInstance()->GetSubsystem<UXGMySubsystem>();

// 静态单例（有风险）
static UXGMySubsystem* Singleton = nullptr;
```

## DeveloperSettings：类型化配置

### 声明模板

```cpp
UCLASS(Config = XGSampleSettings, defaultconfig)
class UXGSampleSettings : public UDeveloperSettings
{
    GENERATED_BODY()
public:
    virtual FName GetContainerName() const override;   // Project
    virtual FName GetCategoryName() const override;    // 侧边栏分组
    virtual FName GetSectionName() const override;     // 节名

    UPROPERTY(Config, EditAnywhere, BlueprintReadWrite, Category = "XG")
    FString XGAppKey = TEXT("None");
};
```

### 写入四步

```cpp
UXGSampleSettings* Settings = GetMutableDefault<UXGSampleSettings>();
Settings->XGAppKey = InAppKey;
Settings->Modify();
Settings->SaveConfig(CPF_Config, *ConfigPath);
```

### INI 映射路径

```
UCLASS 中的 Config=XGSampleSettings + defaultconfig
→ INI 段名: [/Script/{Module}.UXGSampleSettings]
→ 文件: Saved/Config/Windows/DefaultXGSampleSettings.ini
```

### INI 6 层层级

| 层级 | 文件 | 覆盖优先级 |
|------|------|-----------|
| Engine Base | Engine/Config/Base.ini | 最低 |
| Engine Platform | Engine/Config/Windows/Windows*.ini | ↑ |
| Project Default | Project/Config/Default*.ini | ↑ |
| Project Platform | Project/Config/Windows/Windows*.ini | ↑ |
| Saved/User | Saved/Config/Windows/Game*.ini | ↑ |
| Packaged | WindowsNoEditor/Project/Saved/Config/Windows/Game*.ini | 最高 |

## CDO 对比

| 维度 | Subsystem | DeveloperSettings |
|------|----------|------------------|
| 基类 | USubsystem 派生 | UDeveloperSettings 派生 |
| 获取方式 | GetSubsystem<T>() | GetMutableDefault<T>() |
| 生命周期 | 由 Outer 控制 | 类加载时创建，进程级 |
| 持久化 | 不自动持久化 | 通过 UPROPERTY(Config) 自动序列化 |
| 访问点 | 需先获取 Outer | 全局可访问 |
