# 异步执行模式

## 六种模式全景

| 模式 | 抽象层级 | 线程模型 | 返回值 | 适用场景 |
|------|---------|---------|--------|---------|
| FRunnable | 最低 | 独立 OS 线程 | 无（共享变量回传） | 长驻后台 Worker、音频采集、TCP 连接 |
| AsyncTask | 低 | TaskGraph 线程池 | 无 | 火抛任务、线程跳跃 |
| Async + TFuture | 中 | TaskGraph 线程池 | 有（TFuture.Get） | 简单异步计算 |
| FGraphEvent | 中 | TaskGraph 线程池 DAG | 无（事件通知） | 多任务依赖编排 |
| ParallelFor | 中 | TaskGraph 线程池并行 | 无（原地返回） | 数据并行计算 |
| FControlFlow | 高 | 内部 TaskGraph + 步骤队列 | 无（Delegate 回传） | 异步步骤编排、进度管理 |

## FRunnable 独立线程

### 核心接口

```cpp
class FXGMyRunnable : public FRunnable
{
    virtual bool Init() override;     // 新线程调用，初始化
    virtual uint32 Run() override;    // 新线程调用，主循环
    virtual void Exit() override;     // 新线程调用，清理
    virtual void Stop() override;     // 外部线程调用，请求停止
};
```

### 生命周期

```
FRunnableThread::Create() → Init() [新线程]
  → Run() [新线程，主循环]
  → Stop() [外部线程调用，设标志]
  → Run() 检测到退出标志 → return 0
  → Exit() [新线程] → 线程结束
```

### 线程安全标志

| 模式 | 适用场景 |
|------|---------|
| `FCriticalSection + bool` | 多个变量需要一致地读写 |
| `FThreadSafeBool` | 仅单个 bool 标志位（推荐） |

```cpp
// FRunnable + FThreadSafeBool 模式
class FXGSimpleRunnable : public FRunnable
{
    FThreadSafeBool bRunning = true;
    virtual uint32 Run() override
    {
        while (bRunning)
        {
            FPlatformProcess::Sleep(0.04f);
            // 工作...
        }
        return 0;
    }
    virtual void Stop() override { bRunning = false; }
};
```

## AsyncTask 与 Async

### 线程跳跃模式（最常用）

```cpp
AsyncTask(ENamedThreads::AnyThread, [this]() {
    // 后台线程池工作（耗时计算）
    
    AsyncTask(ENamedThreads::GameThread, [this]() {
        // 跳回 GameThread 操作 UObject
        SetMyNum(42);
    });
});
```

### 安全 GameThread 访问模式

```cpp
void SetMyNum(int32 InInt)
{
    if (!IsInGameThread())
    {
        AsyncTask(ENamedThreads::GameThread, [this, InInt]() {
            SetMyNum(InInt);
        });
        return;
    }
    // 当前在 GameThread，直接操作
    MyIntNum = InInt;
}
```

### Async + TFuture（带返回值）

```cpp
TArray<TFuture<int32>> Results;
for (int32 i = 0; i < 5; i++)
{
    Results.Add(Async(EAsyncExecution::Thread, [i]() {
        int32 Result = ExpensiveComputation(i);
        return Result;
    }));
}
// 阻塞等待所有结果
for (auto& Future : Results)
{
    int32 Val = Future.Get();
}
```

## FGraphEvent 任务 DAG

```cpp
// 单任务
FGraphEventRef Task = FFunctionGraphTask::CreateAndDispatchWhenReady(
    []() { FPlatformProcess::Sleep(1.0); }
);
Task->Wait();  // 阻塞

// 批量任务
FGraphEventArray Events;
for (int32 i = 0; i < 5; i++)
{
    Events.Add(FFunctionGraphTask::CreateAndDispatchWhenReady(
        [i]() { DoWork(i); }
    ));
}
FTaskGraphInterface::Get().WaitUntilTasksComplete(Events);

// 依赖链：A → B → C
FGraphEventRef A = FFunctionGraphTask::CreateAndDispatchWhenReady(DoA);
FGraphEventRef B = FFunctionGraphTask::CreateAndDispatchWhenReady(DoB, TStatId(), nullptr, ENamedThreads::AnyThread);
FGraphEventRef C = FFunctionGraphTask::CreateAndDispatchWhenReady(
    DoC, TStatId(), &{A, B}, ENamedThreads::GameThread
);
```

### DAG 模式

```
Gather → ProcessItem(1) [→] Combine [→] ApplyToGame
         ProcessItem(2) [→]
         ProcessItem(3) [→]
```

```cpp
// 参考：XGThreadSubsystem.cpp
FGraphEventArray CalculationEvents;
for (int32 i = 0; i < 3; i++)
    CalculationEvents.Add(CreateAnalysisTask(i));

FGraphEventRef CombineTask = FFunctionGraphTask::CreateAndDispatchWhenReady(
    [this]() { CombineAndApplyResults(); },
    TStatId(), &CalculationEvents, ENamedThreads::AnyThread
);

FFunctionGraphTask::CreateAndDispatchWhenReady(
    [this]() { FinalizeOnGameThread(); },
    TStatId(), &{CombineTask}, ENamedThreads::GameThread
);
```

## ParallelFor 数据并行

```cpp
// 基本用法（自动多线程）
ParallelFor(Array.Num(), [&](int32 i) {
    ProcessItem(Array[i]);
});

// 手动控制
ParallelFor(Items.Num(), [&](int32 i) {
    ProcessItem(Items[i]);
}, EParallelForFlags::ForceSingleThread  // 单线程调试
   | EParallelForFlags::Unbalanced       // 不均等分配
   | EParallelForFlags::BackgroundPriority); // 后台优先级

// 共享变量保护
int32 SafeCounter = 0;
FCriticalSection CS;
ParallelFor(Array.Num(), [&](int32 i) {
    ProcessItem(Array[i]);
    FScopeLock Lock(&CS);  // 并行安全更新
    SafeCounter++;
});
```

## TaskGraph 线程池陷阱

同一模式可解释多种阻塞场景：

| 场景 | 表现 | 根因 |
|------|------|------|
| FAsyncTask + 循环中 Sleep | 主线程 Tick 卡住 | TaskGraph worker 在 `Stop()` 中 Sleep 等待标志 |
| ParallelFor 内含 Sleep | 所有 TaskGraph worker 被占满 | ParallelFor 用掉所有线程池 worker |
| 外部模块调 AsyncTask 后 Wait | 出现死锁 | TaskGraph worker 等待另一个 TaskGraph task |

**本质规律**：TaskGraph 线程池 worker 线程如果在执行途中等待另一个 TaskGraph 的 task，就会占用一个 worker 槽位。如果 task 链依赖关系导致 worker 之间互相等待，线程池会被耗尽。

## FControlFlow 异步编排

```cpp
// 基本流程
TSharedPtr<FControlFlow> Flow = FControlFlowStatics::Create()
    ->QueueStep(TEXT("Step1"), [](FControlFlowNodeRef) { DoStep1(); })
    ->QueueStep(TEXT("Step2"), [](FControlFlowNodeRef) { DoStep2(); })
    ->QueueStep(TEXT("Step3"), [](FControlFlowNodeRef) { DoStep3(); })
    ->ExecuteFlow();

// 取消
Flow->CancelFlow();
Flow->ContinueFlow();  // 继续（被 Cancelled 挂起时）

// 进度跟踪
Flow->QueueStep(TEXT("Init"), [Node](FControlFlowNodeRef)
{
    InitWithProgress(Node, 20, TEXT("加载资源..."));
});
```

// 三阶段：Detect（检测）→ Broadcast（回调）
void UXGControlFlowsSubsystem::Tick(float DeltaTime)
{
    // 检测阶段
    for (auto& [Id, Task] : ManageList)
    {
        Task->UpdateTaskStatus();
    }
    // 广播阶段
    for (auto& [Id, Task] : ManageListCopy)

        if (Task->bShouldBroadcast)
            Task->OnCompleted.Broadcast(true);
    }
