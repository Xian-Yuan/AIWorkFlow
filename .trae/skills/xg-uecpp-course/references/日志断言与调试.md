# 日志断言与调试

## 概述

UE 提供三阶调试防御体系：日志（运行时信息记录）、断言（开发期正确性检查）、异常处理（非致命错误恢复）。核心工具包括 UE_LOG/UE_LOGFMT（日志）、check/verify/ensure（断言）和 Dump 文件（Crash 分析）。

## 日志系统

### 日志类别

```cpp
// 声明（在 .h 中，供其他文件引用）
DECLARE_LOG_CATEGORY_EXTERN(LogMyGame, Log, All);

// 定义（在 .cpp 中）
DEFINE_LOG_CATEGORY(LogMyGame);
```

### UE_LOG

```cpp
UE_LOG(LogMyGame, Log, TEXT("Value: %d"), Value);          // 普通日志
UE_LOG(LogMyGame, Warning, TEXT("Warning: %s"), *Str);     // 警告
UE_LOG(LogMyGame, Error, TEXT("Error: %d"), ErrorCode);   // 错误（打印调用栈）
```

| 日志级别 | 说明 | 默认控制台显示 |
|---------|------|-------------|
| `Fatal` | 致命错误，立即 Crash + Dump | 是 |
| `Error` | 错误级别，橙色 | 是（Sale 显示简化版） |
| `Warning` | 警告级别，黄色 | 是 |
| `Display` | 默认输出级别 | 是（Log = Display） |
| `Log` | 详细输出 | 否 |
| `Verbose` | 调试输出 | 否 |
| `VeryVerbose` | 最详细级别 | 否 |

### UE_LOGFMT（结构化日志）

```cpp
UE_LOGFMT(LogMyGame, Warning, "Player {Name} lost {Amount} HP", FString("Alice"), 42);
```

参数自动类型推断，不需要 `%d` / `%s` 格式符，更安全。

### 运行时控制

```cpp
// 在 GameInstance 中设置日志输出级别
FLogCategoryBase* Category = UE::Logging::FindCategory("LogMyGame");
if (Category) Category->SetVerbosity(ELogVerbosity::VeryVerbose);
```

控制台命令：`Log LogMyGame Verbose` 在运行时调整日志级别。

## 断言体系

### check（开发期严格检查）

```cpp
check(Ptr != nullptr);           // 条件失败 → 立即触发 Crash
checkf(Ptr != nullptr, TEXT("Ptr 为空，调用方: %s"), *CallerName);  // 带格式化消息
checkNoEntry();                  // 断言行不应被执行
checkNoReentry();                // 防止递归重入
unimplemented();                 // 标记未实现路径
```

- **Shipping 构建中**：所有 `check` 系列宏**编译为空**，不产生检查代码
- 适合开发期快速捕捉逻辑错误，不保护 Shipping 运行

### ensure（非致命检查）

```cpp
ensure(Ptr != nullptr);                        // 失败 → 打印调用栈 + Dump，但继续执行
ensureMsgf(Ptr != nullptr, TEXT("消息"));       // 带格式化消息
ensureAlways(Ptr != nullptr);                  // Shipping 也执行（ensure 默认 Shipping 不执行）
ensureMsgfAlways(Ptr != nullptr, TEXT("消息")); // Shipping 始终执行的 ensureMsgf
```

- 失败时生成一条 Error 日志 + 调用栈到 Dump 文件
- 返回 `bool`，可根据返回值做降级处理
- **适合"不应发生但发生后可以继续运行"的路径**

### verify（Shipping 保留计算）

```cpp
verify(Ptr != nullptr);              // 条件检查同 check，但 Shipping 保留表达式求值
verifySlow(Ptr != nullptr, TEXT("消息"));
```

- `check` 与 `verify` 的区别：`verify` 在 Shipping 中**保留表达式求值**，但**不检查结果**
- 适用于：表达式有副作用，Shipping 也需要执行的场景

### 三种断言对比

| 宏 | Debug/Development | Shipping | 典型场景 |
|----|-----------------|---------|---------|
| `check` | 检查并 Crash | 编译为空 | 前置条件验证 |
| `verify` | 检查并 Crash | 保留求值，不检查 | 带副作用的检查 |
| `ensure` | 检查并打印 Dump 后继续 | 默认不执行 | 非致命异常 |
| `ensureAlways` | 同 ensure | 检查并打印 Dump 后继续 | 关键路径的崩溃保护 |

## Dump 文件定位

```cpp
// 异常处理
try { ... }
catch (const std::exception& e)
{
    UE_LOG(LogTemp, Error, TEXT("STL 异常: %s"), UTF8_TO_TCHAR(e.what()));
}

// 多线程 Crash 日志
// 主线程：Dump 文件 + 日志 → 足够排除多数问题
// 渲染线程：单独线程日志（r.DumpCrashLog）
// Worker 线程：建议加 UE_LOG 桩
```
