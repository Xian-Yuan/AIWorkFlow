## 1. 核心设计理念

### 1.1 问题

当前系统：
- 约束是硬编码在 SKILL.md 中的静态文本
- 全随机只是抽选项，没有推敲
- 学到新特征后无法自动生成对应约束
- 渲染风格、设计流派等预设太少

### 1.2 目标

```
用户对话 → AI 联网搜索特征 → 提炼「结构规则」
    │                              │
    │         ┌────────────────────┘
    ▼         ▼
  约束库（Constraint Library）← 自我成长
    │
    ├─ 公共约束（全局生效，少量核心原则）
    └─ 条件约束（某个特征触发时才加载）
         ├─ 来源：联网搜索后的AI提炼
         ├─ 存储：SKILL.md 内的约束段 / 独立约束文件
         └─ 触发：用户选择了某个特征 → 自动加载对应约束
```

### 1.3 约束生命周期

```
                      用户输入新特征（如"蒸汽朋克"）
                              │
                              ▼
              ┌──────────────────────────┐
              │ 1. 搜索该特征的客观结构特征  │ ← 联网搜索
              └──────────┬───────────────┘
                         │
                         ▼
              ┌──────────────────────────┐
              │ 2. AI 提炼为设计约束       │ ← 从"铜管/齿轮/铆钉"提炼
              │   "蒸汽朋克→出现齿轮/铜管  │    "服装需有齿轮或铜管元素"
              │    铆钉/皮革束带至少1种"   │    "主色需含铜色系≥30%"
              └──────────┬───────────────┘
                         │
                         ▼
              ┌──────────────────────────┐
              │ 3. 入库（SKILL.md 约束段） │ ← 追加到条件约束表
              └──────────┬───────────────┘
                         │
                         ▼
              ┌──────────────────────────┐
              │ 4. 下次随机/对话触发该特征  │ ← 自动加载该约束
              │   时自动应用              │
              └──────────────────────────┘
```

## 2. 特征→约束自动提取引擎

### 2.1 提取 Prompt 模板

当用户输入/随机到一个设计特征值，且该值在约束库中不存在时，执行以下对话内部推理：

```
【内部推理：特征→约束提取】

该特征是：{特征值}（所属类别：{职业/种族/风格/力量来源/设计流派等}）

联网搜索该特征的客观设计特征。
从搜索结果中提取：
1. 该特征在角色设计中的标志性视觉符号（列举3-5个）
2. 该特征必须出现的元素（must-have）
3. 该特征禁止的元素（must-not-have，或与之冲突的元素）
4. 该特征对配色/比例/材质的影响

然后生成以下格式的约束条目：

## {特征值}
- 视觉符号：{1-3个最标志性的视觉符号}
- MUST：{必须包含的元素，如"尖耳""暗色能量纹理"}
- FORBID：{禁止的元素，如"不可使用暖色""不可穿厚重板甲"}
- 配色约束：{如有，如"主色必须含暗色系≥50%"}
- 比例约束：{如有，如"下半身比例需≥60%"}
- 低像素适配：{在低像素下如何简化}
```

### 2.2 约束存储格式

在 SKILL.md 末尾的动态约束段，格式如下：

```markdown
## 动态约束库（自动成长，请勿手动编辑）

<!-- CONSTRAINT: 类别=渲染风格 特征=像素风 来源=联网搜索 时间=2026-05-10 -->
### 像素风
- 视觉符号：大色块、清晰轮廓、像素颗粒
- MUST：3-5色配色，禁止渐变
- FORBID：半透明、模糊边缘、细线装饰
- 低像素适配：轮廓优先于细节，16x16格内需可识别
<!-- /CONSTRAINT -->

<!-- CONSTRAINT: 类别=设计流派 特征=新艺术运动 来源=联网搜索 时间=2026-05-10 -->
### 新艺术运动
- 视觉符号：有机曲线、植物藤蔓、不对称构图
- MUST：服装/武器至少有一处流畅的曲线装饰
- FORBID：尖锐直角、机械结构、几何对称
- 配色约束：主色可用金/铜/深绿/暗紫
- 低像素适配：曲线简化为2-3段的弧形
<!-- /CONSTRAINT -->
```

### 2.3 约束智能加载规则

```
当用户选择特征X时：
  1. 扫描约束库 → 匹配类别=当前步骤 AND 特征=X
  2. 匹配到 → 加载该约束到当前对话上下文
  3. 未匹配 → 触发联网搜索 → 提取约束 → 入库 → 加载

当用户随机生成时：
  1. 随机选取特征值
  2. 加载该特征对应的约束
  3. 检测选取的多个特征之间是否有约束冲突（如 FORBID 交叉）
  4. 如有冲突 → AI内部推敲解决 → 调整冲突项 → 输出最终搭配
```

### 3.2 关键系统提示词增强

```
原提示词8条 → 新增：

9. 当用户选择一个设计值时，自动检索约束库。如果该值存在对应约束，将其加载到当前对话记忆。如果不存在，在后台联网搜索该值的设计特征，提炼为约束并存入约束库。

10. 在全随机模式下，每选取3-5个值后做一次内部冲突检测。如果检测到冲突，用1-2句话解释调整原因，输出调整后的搭配。

```

## 4. 对话成长流水线

### 4.1 全随机自对话推敲流程

```
🎲 全随机模式启动
    │
    ├─ 阶段1：独立随机（1-2秒）
    │   每个步骤独立随机选取值
    │
    ├─ 阶段2：约束完整性检查（1秒）
    │   检查每个随机值在约束库中是否存在
    │   不存在 → 后台联网搜索 → 提取约束 → 入库
    │
    ├─ 阶段3：跨步骤冲突检测（2-3秒）
    │   扫描所有已选特征的 FORBID 交叉
    │   示例：职业=潜行(FORBID:暖色) AND 配色=红金 → ⚠️冲突
    │
    ├─ 阶段4：内部推敲（输出给用户看到）
    │   对每个冲突，AI 用内部对话解释并调整：
    │   "你选择了暗影刺客，但配色随机到了红金——
    │    这两个有冲突。我把配色调整为【深蓝+金】，
    │    保留金色作为点缀，深蓝呼应暗影主题。"
    │
    └─ 阶段5：输出最终角色 + 推敲记录
        完整角色设定 + "本次随机做了哪些调整"的简短说明
```

### 4.2 每次对话的学习产出

对话结束后的成长回顾（新增内容）：

```
📚 本次对话新增知识：
├── 新特征入库：3个（蒸汽朋克、巴洛克、半精灵）
├── 新约束入库：5条
├── 联网搜索次数：3次
├── 冲突检测次数：2次
└── 约束库当前总量：42条
```
