# 全随机自对话推敲流程

> 当用户选择"🎲 全随机"模式时，AI 执行本流程。  
> 核心原则：**不是抽奖，是自我推敲**——像和用户对话一样，AI 自己追问、自己回答、自己解决冲突。

---

## 整体架构

```
🎲 全随机模式启动
    │
    ├─ 轮次1：核心身份（步骤1-5）
    │   渲染 → 姓名 → 性别 → 体型 → 种族
    │   推敲点：体型与种族是否协调？
    │
    ├─ 轮次2：战斗定位（步骤6-8）
    │   职业 → 战斗风格 → 力量来源
    │   推敲点：职业是否匹配力量来源？战斗风格是否与体型冲突？
    │
    ├─ 轮次3：外观气质（步骤9-10）
    │   气质性格 → 发型
    │   推敲点：发型是否呼应气质？
    │
    ├─ 轮次4：视觉元素（步骤11-14）
    │   主元素 → 副元素 → 武器 → 服装板式
    │   推敲点：主副元素关系？武器与职业匹配？
    │
    ├─ 轮次5：最终统一（步骤15-17）
    │   配色 → 设计流派 → 补充
    │   推敲点：配色与所有已选特征是否冲突？流派是否统一？
    │
    └─ 输出：完整角色 + 推敲记录
```

---

## 每轮执行逻辑

### 步骤A：独立随机

```python
# 伪代码
for step in current_round:
    value = random_choice(presets[step])
    record(step, value)
```

### 步骤B：约束库检查

```python
for each selected value:
    constraint_file = f"constraints/{step_category}/{value}.md"
    if not exists(constraint_file):
        # 后台联网搜索该特征 → 提取约束 → 入库
        web_search_and_extract(value, step_category)
    load_constraint(constraint_file)
```

### 步骤C：跨步骤推敲（核心）

```
对当前轮次的所有已选特征：

1. 扫描 FORBID 交叉
   检查每一个已选特征的 FORBID 列表中是否有其他已选特征
   例如：
     职业=潜行 → FORBID:暖色
     配色=红金 → 暖色
     → ⚠️ 冲突

2. 检查 MUST 互补
   如果多个 MUST 可以互相加强 → 记录为"协同"
   例如：
     渲染=像素风 → MUST: 3-5色
     配色=红金黑 → 3色
     → ✅ 协同

3. AI 自我对话推敲（输出给用户看）
   对于每个找到的问题，AI 用对话语气解释：
   
   "我随机到了【暗影刺客】职业和【红金】配色。
    但暗影刺客的约束禁止暖色——红和金都是暖色系。
    这就像白天穿夜行衣，不太协调。
   
    我调整为【深蓝+暗金】：深蓝呼应暗影主题（占比70%），
    暗金只作为武器和腰扣的点缀（10%），不破坏潜行感。
    这样配色与职业不冲突了。"
```

### 步骤D：推进下一轮

```
当前轮次推敲完成后：
  - 所有冲突已解决
  - AI 用1句话总结本轮的推敲结果
  - 进入下一轮
```

## 跨轮次最终推敲

所有5轮完成后，做一次全局扫描：

```
1. FORBID 全局检查
   所有17个步骤的 FORBID 互相交叉

2. MUST 一致性检查
   是否有 MUST 互相矛盾？
   例如：像素风 MUST:3-5色 + 巴洛克 MUST:华丽装饰 → 低像素下"华丽"需要精简

3. 视觉重心检查
   主元素占比 > 副元素占比
   配色主色 > 辅色 > 点缀
   面部区域无过多强对比元素

4. AI 最终推敲（输出）
   "纵观全局，我做了3个调整：
    ① 巴洛克的金边在像素风下简化为单线，避免了华丽冲突
    ② 暗影力量来源在低像素下用一个紫色发光点表达
    ③ 发型从长卷发改为长直发——16x16格内卷发会糊成一团
    整体角色以「暗影潜行+新艺术曲线」为主轴。"
```

## 注意事项

- 每轮推敲的自我对话**不要超过 4 句话**——简洁为主
- 冲突调整后**不要回头改之前已确认的值**——只改当前轮次内的
- 如果用户中途打断要求修改某个值，退出全随机模式，切回手动模式
- 推敲记录中的调整必须说明**原因**（引用具体约束），不能只说"我改了"
