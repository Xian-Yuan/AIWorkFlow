# AIRPGWeb Asset System Overall Design

## Goal

为 `AIRPGWeb` 开发模式下的素材体系建立清晰的信息架构，把当前“列表 + 编辑器混合”的素材库重构为：

- `绘制素材`：负责像素级素材绘制与编辑
- `素材库`：负责素材浏览、检索、导入、预览与打开编辑

同时统一地图放置、素材浏览、导入与二次编辑所依赖的素材元数据模型。

## Background

当前 `AssetLibraryScreen` 同时承担了以下职责：

- 新建素材
- 编辑像素画板
- 保存素材
- 浏览已保存素材

这导致“素材管理”和“素材编辑”耦合在同一个界面里，不利于后续加入：

- 地图单格内平铺语义
- 真正的素材浏览检索
- 外部素材导入
- 导入素材后进入像素画板二次编辑

因此本次总设计的核心目标不是增加一个局部按钮，而是把素材系统升级为真正的资产工作流。

## Scope

### In Scope

- 明确 `绘制素材` 与 `素材库` 的模块边界
- 明确统一素材数据模型
- 明确地图放置时的平铺语义入口
- 明确导入与二次编辑的整体流转
- 明确后续子项目的职责拆分

### Out Of Scope

- 不在本次总设计中直接给出每个 UI 的逐像素实现细节
- 不在本次总设计中直接进入代码实现
- 不在本次总设计中重新设计整个地图编辑器架构

## High-Level Architecture

### Module Split

开发模式下拆分为两个独立模块：

1. `绘制素材`
2. `素材库`

其中：

- `绘制素材` 是素材编辑器
- `素材库` 是素材管理器

两者共用同一份素材仓库和同一份素材元数据模型，不各自维护一套互不兼容的数据结构。

### Shared Data Source

两模块都通过统一仓库访问素材数据：

- 保存素材
- 查询素材
- 更新素材
- 检索标签与尺寸

现有 `asset-library-repository.ts` 需要从“简单 list/get/save”扩展为真正的资产仓库接口，但仓库仍然是单一真相源。

### Shared Editor Entry

素材库中的素材可以进入 `绘制素材` 做二次编辑。

因此 `绘制素材` 不只支持：

- 新建空白素材

也必须支持：

- 打开已有素材
- 打开导入素材

## Unified Asset Model

统一素材模型必须覆盖以下维度：

### Identity

- `id`
- `name`
- `updatedAt`

### Visual Preview

- `thumbnail`
- `pixelWidth`
- `pixelHeight`
- `previewTileSize`

### Search And Organization

- `tags: string[]`
- `sourceType`
- `sourceMeta`

其中：

- `sourceType` 用来区分素材来源，例如：
  - `drawn`
  - `imported`

### Placement Behavior

- `defaultTiling`

该字段表示素材默认是否平铺，但不是地图侧最终唯一决定因素。

### Editable Pixel Data

- 原始像素数据

这是导入后继续编辑、手绘后再次编辑、另存与覆盖保存的基础。

## Tiling Rule Ownership

地图放置时的“是否平铺”采用双层控制：

1. 素材保存默认值 `defaultTiling`
2. 地图放置前临时覆盖 `overrideTiling`

最终放置优先级：

- 若地图当前选择显式设置了临时值，则使用临时值
- 否则回退到素材默认值

这样可以满足：

- 素材本身具有推荐放置方式
- 当前绘制时仍能按场景需要临时切换

## Product Modules

### Module 1: 绘制素材

职责：

- 新建素材
- 打开素材
- 像素级编辑
- 保存
- 覆盖保存
- 另存为

不负责：

- 全量素材浏览检索
- 素材卡片式管理

### Module 2: 素材库

职责：

- 浏览所有已保存素材
- 显示缩略图与名字
- 显示 tags 与尺寸
- 搜索与过滤
- 放大预览
- 导入素材
- 打开到绘制素材进行编辑

不负责：

- 承担主像素编辑器 UI

## Navigation Design

开发模式下的入口应拆分为两个独立菜单项：

- `绘制素材`
- `素材库`

推荐交互：

- 进入 `绘制素材`：默认进入空白或最近一次编辑状态
- 进入 `素材库`：默认进入资产浏览列表
- 在 `素材库` 中点击素材：进入预览详情
- 在预览详情中点击“编辑”：跳转到 `绘制素材`

## Save Behavior

素材编辑保存策略采用双选项：

- 覆盖原素材
- 另存为新素材

默认不强制单一路径，由用户在保存时明确选择。

这样可以避免：

- 修改现有素材时误覆盖
- 导入素材编辑后只能产生副本、不能回写

## Import Workflow

导入素材的总体规则：

- 支持从外部图片导入
- 默认保留原始尺寸
- 同时提供一键适配模板尺寸
- 导入后素材进入统一素材仓库
- 也可继续打开到绘制素材模块做二次编辑

## Search Model

素材库支持三类检索：

- 按名字
- 按 tag
- 按尺寸

尺寸检索以真实像素尺寸为准，而不是仅以模板分组尺寸为准。

## Subproject Breakdown

该总设计下拆成三个独立子设计：

1. 地图单格放置平铺规则
2. 真正素材库浏览与检索
3. 导入素材与二次编辑流程

它们共享总设计中的素材模型与模块边界。

## Risks

- 如果先做 UI 拆分而不先统一素材模型，后续会出现两套保存逻辑
- 如果平铺规则只放在地图层而不落到素材元数据，会导致调色板无法表达默认语义
- 如果导入素材不保留原始像素数据，二次编辑会退化成只能看缩略图
- 如果“绘制素材”和“素材库”仍然共用一个混合页面，后续搜索与编辑状态会相互干扰

## Acceptance Criteria

- 开发模式在设计上明确拆成 `绘制素材` 和 `素材库`
- 地图放置平铺规则支持“素材默认值 + 放置前临时覆盖”
- 素材模型支持标签、缩略图、尺寸、来源、默认平铺等信息
- 素材库支持浏览、搜索、导入、预览、进入编辑
- 绘制素材模块支持打开已有素材并进行覆盖/另存
