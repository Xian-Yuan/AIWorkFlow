# AIRPGWeb 墙体连接规律与材质表达设计

## 目标

本设计文档用于先定义 **tile 房屋墙体系统的结构规律**，再允许后续去做材质设计、完整房屋案例设计与地图编辑器实现。

本次设计的核心结论是：

1. 房屋首先是由 `墙壁 + 门` 围成的封闭空间，不是若干好看的墙块拼贴。
2. 墙体 tile 的形态先由 `拓扑连接关系` 决定，再由 `材质语言` 决定如何表现。
3. 任意一套墙体材质都必须满足“深色结构线可连续、门位可接入、外轮廓闭合、内部房间可闭合”的硬规则。
4. 在未满足连接规律前，禁止继续扩展新的墙体风格草图。

---

## 问题复盘

此前视觉草图出现的主要错误，不是“风格不够多”，而是“连接规律没有先被抽象清楚”。

具体表现为：

- 不同 tile 在单独看时像墙，但拼接成房屋后并不形成同一套结构语言。
- 深色像素、浅色填芯、拐角收口、T 型连接、十字连接与门位边界没有统一规则。
- 房屋案例过小、过方，且不同材质反复复用同一个简单模板，无法判断真正的结构成立度。
- 出现了非封闭房间、孤立墙段、伪隔断等违背房屋基本语义的情况。

因此，后续流程必须调整为：

1. 先定义墙体连接规律。
2. 再定义材质表达层。
3. 再生成完整封闭房屋案例。
4. 最后才进入编辑器中的 palette、tile 预览与实现。

---

## 核心原则

### 1. 封闭优先

建筑物是用墙壁和门围起来的封闭空间。

这条规则同时适用于：

- 外轮廓
- 内部房间
- 房间与房间之间的过渡
- 房间与外部之间的出入口

禁止出现以下情况：

- 只有单独墙段，没有形成闭合空间
- 墙与墙视觉上接触，但逻辑上不闭合
- 门位出现于没有墙体支撑的位置
- 内部隔断只画了一截，没有明确终点或门洞

### 2. 拓扑先于美术

单个墙体 tile 的类别，不由名字先决定，而由四方向连接关系决定。

每个墙格只根据 `上 / 右 / 下 / 左` 是否与其他墙格相连，归入固定类型。

### 3. 结构线必须连续

每套材质内部都必须指定：

- 主结构线
- 次结构厚度边
- 填芯材质
- 门位过渡区

若某种颜色承担“结构主轮廓”职责，则在相邻 tile 之间必须能连续拼接，尤其是：

- 横墙连接横墙
- 竖墙连接竖墙
- 拐角接横墙或竖墙
- T 型接三向墙
- 十字接四向墙
- 门位与两侧墙体的轮廓连续

### 4. 材质只改表达，不改语法

土、木、石、红砖等材质可以拥有不同的：

- 主色
- 轮廓线粗细
- 明暗关系
- 填芯纹理
- 门框表现

但它们不得拥有不同的连接语法。

换句话说：

- 材质可以不同
- 连接规则必须一致

## 墙体拓扑语法

以下规则是墙体系统的第一真相源。

### 基础连接类型

设一个墙格与四方向的连接布尔值分别为：

- `U`：上
- `R`：右
- `D`：下
- `L`：左

则墙格分类如下：

### 1. 横墙

- 条件：`L = true`，`R = true`，`U = false`，`D = false`
- 语义：房屋轮廓的水平段，或室内隔断的水平段

### 2. 竖墙

- 条件：`U = true`，`D = true`，`L = false`，`R = false`
- 语义：房屋轮廓的垂直段，或室内隔断的垂直段

### 3. 拐角

- 条件：恰有两个连接，且这两个连接互相垂直
- 四种方向：
  - `U + R`
  - `R + D`
  - `D + L`
  - `L + U`
- 语义：房屋外轮廓折角，或室内房间折角

### 4. T 型连接

- 条件：恰有三个方向连接
- 四种方向：
  - `U + R + D`
  - `R + D + L`
  - `D + L + U`
  - `L + U + R`
- 语义：一个隔断或分支从主墙体分出

### 5. 十字连接

- 条件：`U = R = D = L = true`
- 语义：四向同时相连的房间交汇点

### 6. 端头

- 条件：恰有一个方向连接
- 四种方向：
  - 仅上
  - 仅右
  - 仅下
  - 仅左
- 语义：合法场景只允许出现在“明确结束的内部隔断末端”
- 默认情况下，**外轮廓禁止出现端头**

### 7. 门位

- 门位不是独立漂浮结构。
- 门位本质上是墙体中的“开口状态”。
- 合法门位必须满足：
  - 左右有墙时可形成横门位
  - 上下有墙时可形成竖门位
- 门位两侧的结构线必须和相邻墙体连通

## 房屋闭合规则

### 外轮廓规则

- 房屋外轮廓必须是闭合环
- 外轮廓中不得出现端头
- 外轮廓墙格必须能追踪回到起点

### 内部房间规则

- 内部隔断要么形成闭合次级房间
- 要么通过合法门位与其他空间连接
- 不允许出现无意义悬空半截隔断

### 展示规则

任何后续预览图都必须满足：

- 至少展示一个完整封闭房屋
- 能看出门位
- 能看出内部房间或至少明确的内部分区
- 不得只展示零散墙段作为最终判断依据

---

## 像素语义规则

### 1. 主结构线

主结构线通常使用最深色像素，承担：

- 外轮廓边界
- 拐角收口
- T / 十字连接的连续关系
- 门框主轮廓

要求：

- 相邻 tile 的主结构线必须在边界处准确对齐
- 不能出现一个 tile 的深色结构线撞上另一个 tile 的浅色填芯
- 拐角点的深色像素必须与两侧相邻墙体的深色结构线相接

### 2. 厚度边

若墙体设计存在“厚度”表达，则厚度边可以是次深色或中间色。

要求：

- 厚度边只能依附于主结构线
- 不允许厚度边越界进入门洞
- 不允许厚度边在拐角处突然消失或增宽

### 3. 填芯

填芯是材质的主体表现：

- 泥墙
- 木板
- 石块
- 砖面

要求：

- 填芯永远服从主结构线围出的范围
- 不允许填芯在连接处盖住相邻 tile 的主结构线

### 4. 门位过渡

门位不是“把墙挖掉一块”这么简单，而是：

- 两侧墙体轮廓必须继续成立
- 门洞边缘应有明确过渡
- 门框应与墙体主结构线在颜色和位置上兼容

## 材质表达约束

### 土系

- 适合木骨泥墙、夯土、土坯
- 主结构线可偏木骨深色
- 填芯可偏暖土浅色
- 适合生活居所、村民屋、农舍

### 木系

- 适合梁柱 + 灰泥、木板、职业居所
- 主结构线可偏粗梁木色
- 填芯可偏浅灰泥或浅木板
- 适合作坊、旅店、职业住宅

### 石系

- 适合石砌房屋、主厅、仓屋
- 主结构线可偏冷灰深边
- 填芯可偏浅灰石面
- 适合大屋、正式建筑、石材结构

### 红砖创意分支

- 允许保留为风格分支
- 但仍必须服从同一拓扑语法
- 仅在完整房屋案例中成立时才可继续保留

---

## 自然区域规则

### 草地

当前阶段草地先回到纯净底块。

原因：

- 草地基础地块首先要稳定、可铺满、不过度抢戏
- 若基础形态未确认，不应过早叠加装饰性草纹

后续若要再加草叶纹理，应满足：

- 纹理不破坏地块平铺连续性
- 纹理不制造强烈角落高光
- 纹理不误导为功能性边界

### 树

树是多格对象，不应再被当成单格图标。

树的设计规则如下：

- 树干与树冠分件设计
- 树冠 tile 的底部必须自带一小截树干或可接树干的下缘
- 树干 tile 的顶部必须与树冠下缘直接相接
- 禁止出现树冠悬空、树干与树冠中间留空的情况
- 树冠允许占据多格，例如：
  - `2x2` 树冠 + `1x2` 树干
  - `3x2` 树冠 + `1x3` 树干

## 后续设计输出要求

在本设计之后，任何新的视觉方案都必须遵守以下要求：

### 房屋案例

- 至少展示完整封闭房屋
- 不得只展示单独结构块让用户判断整体可行性
- 房屋结构不能全部复用同一个简单模板
- 至少存在面积、轮廓或内部房间组织上的差异

### 墙体检查项

每套房屋案例必须检查：

- 外轮廓是否闭合
- 内部房间是否闭合或通过门位合理连接
- 深色结构线是否连续
- 门位是否真正嵌入墙体
- 是否存在孤墙、伪隔断、悬空端头

### 自然对象检查项

- 草地是否干净、可平铺
- 树冠与树干是否连续
- 多格对象是否在连续瓦片地图中成立

## Skill 边界目录

本轮先不真正创建 Skill，但先定义 Skill 目录边界。

建议 Skill 名称方向：

- `tile-wall-topology-designer`

建议职责：

- 接收房屋平面、门位、材质类型
- 先做拓扑归类
- 再套用材质语义
- 再生成可检查的完整房屋方案

建议输入：

- 房屋外轮廓
- 房间分区
- 门位位置
- 材质类型
- 目标风格约束

建议输出：

- 拓扑分类结果
- 每类墙体 tile 的像素语义说明
- 完整封闭房屋案例
- 检查清单

---

## 本轮结论

本轮真正的真相源不是“又一版风格图”，而是下面这条规则：

> **先让墙体拓扑和像素语义成立，再允许做材质风格与房屋案例设计。**

如果后续视觉稿不满足这条规则，即使单格看起来好看，也不能进入正式实现。
