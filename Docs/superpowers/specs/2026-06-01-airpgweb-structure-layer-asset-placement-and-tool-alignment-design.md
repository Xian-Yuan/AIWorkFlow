# AIRPGWeb 结构层素材落图与素材库工具对齐设计

日期：2026-06-01
项目：`g:\UEGameDevelopment\Project\AIRPGWeb`
状态：设计已确认，待审阅

## 1. 目标

本设计用于解决 `AIRPGWeb` 当前地图绘制与素材库之间的两个关键断层：

- 素材库中保存的 `结构层` 素材，在地图绘制中虽然可选中，但实际绘制后没有按真实尺寸正确显示
- 素材库编辑器的工具体系、子工具组织和交互节奏没有与地图绘制保持一致

本轮目标不是重做整个地图编辑器，也不是引入高级素材实例系统，而是在现有架构上完成以下收敛：

- 结构层素材可以按真实尺寸落到地图原子网格中
- 素材尺寸统一受 `8x8` 原子网格约束
- 地图缩放只影响显示倍率，不改变原子格归属
- 素材库工具体系完全向地图绘制靠齐，以地图绘制为唯一标准

## 2. 已确认约束

### 2.1 原子网格固定为 8x8

- `8x8` 是地图与素材共用的最小原子单位
- 所有地图绘制、素材绘制、结构层素材落图都必须对齐到这套 `8x8` 原子网格
- 标准格只是更粗的分组线，不改变原子网格本身

### 2.2 结构层素材必须按实际尺寸绘制

- 结构层素材不能再被压成“一个标准格一张图”的缩略图模型
- 结构层素材进入地图画布后，应按素材自己的真实宽高换算成若干个 `8x8` 原子格
- 最终显示倍率统一跟随地图编辑器的 `zoom`

### 2.3 非 8 的倍数尺寸自动向上补齐

用户已确认补齐策略采用：

- 若素材宽度不是 `8` 的倍数，自动向上补齐到最近的 `8` 倍数
- 若素材高度不是 `8` 的倍数，自动向上补齐到最近的 `8` 倍数
- 补齐后的额外区域视为透明，不写入颜色

示例：

- `18x10` -> 对齐后为 `24x16`
- `32x17` -> 对齐后为 `32x24`

### 2.4 首版放置锚点为左上角

- 在地图绘制中点击一个起始位置后，结构层素材以该位置作为左上角锚点
- 素材按自身对齐后的宽高向右下展开
- 本轮不实现居中锚点、旋转、镜像和自由偏移

### 2.5 工具体系以地图绘制为准

- 地图绘制是工具体系真相源
- 素材库向地图绘制靠齐
- 不反向修改地图绘制去适配素材库旧交互

## 3. 当前问题分析

### 3.1 结构层素材绘制失败的根因

当前地图编辑器把素材库资产主要当成“单格调色板项”消费：

- 地图调色板中保存的是 `id / name / layerId / thumbnail`
- 画布绘制逻辑仍然以“一个地图格子对应一个 tileId”为主
- 结构层像素素材虽然能被选中，但缺少“按真实尺寸映射到多个原子格”的放置与渲染链路

因此当前失败不是单纯的显示 bug，而是数据模型与渲染模型不完整：

1. 调色板项没有携带真实尺寸信息
2. 地图画布没有结构层素材实例的真实占用范围概念
3. 渲染层仍按单格 tile 逻辑处理像素素材

### 3.2 素材库工具未对齐的根因

素材库目前只是“功能近似”，但并未与地图绘制共享一致的交互模型：

- 地图绘制是左侧主工具入口 + 画笔浮层子工具
- 素材库是右侧面板双排按钮
- 两边虽然都有 `画笔 / 填充 / 边框矩形 / 吸管` 等概念，但入口布局、状态表达、交互节奏并不一致

这会导致：

- 用户需要分别记两套操作方式
- 同名工具的行为预期容易漂移
- 后续扩展子工具时容易继续分叉

## 4. 设计范围

### 4.1 本轮包含

- 结构层像素素材的真实尺寸元数据补充
- 结构层素材在地图中的左上角锚点放置规则
- 非 `8` 倍数尺寸的自动向上补齐规则
- 地图画布中结构层素材的原子格级显示
- 缩放与真实尺寸显示的统一规则
- 素材库工具组织、入口和交互流程向地图绘制对齐
- 针对以上行为的回归验证

### 4.2 本轮不包含

- 多锚点模式
- 旋转、镜像、自由偏移
- 结构层复杂实例编辑器
- 跨图层组合素材
- 地图绘制反向迁就素材库旧工具布局

## 5. 结构层素材尺寸模型

### 5.1 素材保留原始尺寸

素材库资产继续保留原始字段：

- `pixelWidth`
- `pixelHeight`

这两个字段表示素材真实像素宽高，不因地图显示规格改变而改写。

### 5.2 新增对齐后尺寸语义

地图消费结构层素材时，需要额外得到对齐后的宽高语义：

- `alignedPixelWidth`
- `alignedPixelHeight`
- `atomCols = alignedPixelWidth / 8`
- `atomRows = alignedPixelHeight / 8`

其中：

- `alignedPixelWidth = ceil(pixelWidth / 8) * 8`
- `alignedPixelHeight = ceil(pixelHeight / 8) * 8`

### 5.3 地图调色板消费的数据不再只看缩略图

地图调色板中的像素素材项不能再只包含：

- `id`
- `name`
- `layerId`
- `thumbnail`

还必须带上：

- `pixelWidth`
- `pixelHeight`
- `alignedPixelWidth`
- `alignedPixelHeight`
- `atomCols`
- `atomRows`

否则地图画布无法知道结构层素材应该占用多少个原子格。

## 6. 放置与渲染规则

### 6.1 放置规则

当用户在地图绘制中选中一个 `结构层` 像素素材并点击画布时：

1. 点击位置被视为素材左上角锚点
2. 锚点必须落在地图的 `8x8` 原子格上
3. 素材按 `atomCols x atomRows` 向右下展开
4. 素材实例不允许出现半格偏移

### 6.2 绘制规则

结构层像素素材绘制到地图后，应满足：

- 显示时保留素材真实宽高关系
- 补齐后的透明区域可见为空，不写入实体像素
- 缩放前的逻辑占用范围来自 `atomCols / atomRows`
- 缩放后的最终显示由地图编辑器 `zoom` 决定

### 6.3 缩放规则

- `zoom` 只改变显示倍率
- `zoom` 不改变素材实例归属到哪些原子格
- `zoom` 不改变结构层素材的补齐规则
- `zoom` 不改变左上角锚点

### 6.4 与标准格的关系

- 标准格是地图画布的更粗分组线
- 结构层素材最终仍然只和 `8x8` 原子格绑定
- 即使标准格是 `16 / 24 / 32`，素材也不应被强行压缩成单标准格贴图

## 7. 地图侧数据与渲染策略

### 7.1 首版不引入独立复杂实例系统

本轮不把地图完全重构成复杂的“素材实例编辑器”，而是在现有结构上补最小必要信息。

目标是先解决：

- 结构层素材能放进去
- 能按真实尺寸看到
- 缩放和原子格对齐正确

### 7.2 地图侧需要有“结构素材占用范围”概念

当前单 `tileId` 模型不足以表达结构层素材跨多个原子格的显示需求，因此地图侧至少需要支持：

- 识别当前选中的像素素材属于结构层
- 根据素材元数据计算占用范围
- 在画布渲染阶段按占用范围切片显示素材

如果仍保持“结构层单格仅渲染单个 tileId”的逻辑，则结构层素材会继续表现为：

- 无法显示
- 被压缩
- 只显示左上角一小块

### 7.3 渲染优先级

当结构层存在像素素材渲染时：

- 结构层像素素材显示优先于结构层旧的单格缩略预览方式
- 但不应破坏地图现有的缩放、预览框和 ghost map 空间换算

本轮要求所有与结构层像素素材相关的显示都共享同一套原子格换算，不允许出现：

- 主图按真实尺寸显示
- 预览框按单格显示
- ghost 或 hover 仍按旧单格逻辑显示

## 8. 素材库工具对齐设计

### 8.1 对齐目标

素材库工具交互应向地图绘制靠齐到“用户不需要切换心智模型”的程度。

这意味着：

- 主工具命名一致
- 子工具命名一致
- 主工具入口位置表达一致
- 子工具展开方式一致
- 拖拽预览与提交时机一致

### 8.2 主工具统一

素材库主工具统一为：

- `画笔`
- `橡皮`
- `吸管`

不再以素材库自己的旧按钮布局作为主组织方式。

### 8.3 画笔子工具统一

素材库画笔子工具统一为：

- `笔刷`
- `填充`
- `边框矩形`

与地图绘制保持同名、同顺序、同高亮逻辑。

### 8.4 交互节奏统一

素材库的交互节奏统一为：

1. 点击主工具
2. 打开对应子工具面板
3. 选择子工具
4. 按下开始操作
5. 拖拽实时预览
6. 抬起提交结果

不再保留“只是放两排小按钮”的旧主交互表达。

### 8.5 橡皮行为

素材库橡皮在本轮继续保留已有能力，但表达方式向地图绘制靠齐：

- 顶层主工具仍叫 `橡皮`
- 进入橡皮后，优先体现与地图绘制一致的操作习惯
- 本轮允许保留素材库已有的 `矩形擦除` 能力

如果地图绘制未来补普通橡皮子模式，再同步到素材库；本轮不反过来修改地图绘制。

## 9. 验证清单

### 9.1 结构层素材显示

- 保存一个 `structure` 图层素材
- 在地图绘制中选中该素材
- 点击地图后，素材按真实补齐尺寸显示
- 显示范围与 `atomCols x atomRows` 一致

### 9.2 原子格对齐

- 结构层素材不得出现半格偏移
- 所有边界都必须贴齐 `8x8` 原子格

### 9.3 缩放一致性

- `100% / 放大 / 缩小` 时，素材实例只改变显示倍率
- 原子格归属不变化
- 预览与提交结果保持一致

### 9.4 补齐规则

- 对非 `8` 倍数尺寸素材，地图显示按向上补齐结果渲染
- 补齐区域应保持透明

### 9.5 工具对齐

- 素材库主工具名称、子工具名称、展开路径与地图绘制一致
- 拖拽预览和抬起提交时机与地图绘制一致

### 9.6 页面验证

每次前端改动后都必须：

1. 确认当前有效预览 URL
2. 重新打开最新页面
3. 确认页面实际可访问
4. 确认不是旧端口旧标签页

## 10. 风险点

- 地图当前偏单格 `tileId` 的数据模型，可能不足以完整表达结构层大素材范围
- 结构层旧规则渲染与新像素素材渲染可能发生覆盖冲突
- 如果素材库只改 UI 按钮，不改状态流与提交流程，最终仍会“看起来一样，行为不一样”
- 如果只修主画布，不同步修预览与缩放换算，结构层素材仍会出现局部错位

## 11. 推荐实施顺序

1. 先补地图消费的像素素材尺寸元数据
2. 再补结构层素材的原子格范围换算与显示
3. 再补结构层素材的缩放与预览一致性
4. 最后把素材库工具组织和交互完全对齐到地图绘制
5. 全程配套回归测试与页面重新打开验证
