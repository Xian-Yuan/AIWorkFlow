# AIRPGWeb Tile Topology And Drawing Tools V2 Design

日期：2026-05-31
项目：`g:\UEGameDevelopment\Project\AIRPGWeb`
状态：设计已确认，待写实施计划

## 1. 目标

在现有游戏内地图编辑器基础上，继续完成两条首要主线：

- 把房屋墙体、门位和门板的显示进一步推进到更接近“施工图/户型图”的效果
- 增强编辑器的基础绘制效率，让房屋和院落可以更快画出来

本轮不追求“工具最全”，而是追求“先可用”：

- 能稳定画闭合房屋轮廓
- 能高效画矩形结构和地面区域
- 能快速复用已有 tile
- 能清晰地区分门位、开门、关门

## 2. 范围

### 2.1 本轮包含

- 完整拓扑 skin：
  - 角
  - T 字连接
  - 十字连接
  - 端头
  - 开门
  - 关门
- 新工具：
  - 边框矩形
  - 填充矩形
  - 填充
  - 吸管

### 2.2 本轮不包含

- 复杂批量选区
- 自由变换
- 旋转/镜像工具
- 高级多段笔刷
- 自动推断墙角与连接关系

本轮依然坚持手工精确模式，不改为自动拓扑生成。

## 3. 设计原则

### 3.1 继续坚持手工精确模式

你已经确认：结构拓扑由用户直接选 tile family 再绘制，不采用“先画一条线、编辑器自动猜角/T/十字”的主流程。

所以本轮只增强：

- 显示质量
- 绘制效率
- 规则约束

不改变拓扑编辑方法论。

### 3.2 整格 skin 优先于字符表现

当前结构层已经从字符预览切到整格 DOM/CSS skin。

本轮继续沿这个方向推进：

- tile 内容必须铺满整格
- 线宽必须统一
- 墙门连接必须连续
- 网格线只作为编辑参考叠加层

### 3.3 工具按当前层工作

所有新工具都只作用于当前层：

- 边框矩形
- 填充矩形
- 填充
- 吸管

这样可以保持：

- 分层清晰
- 不跨层污染
- 交互预期稳定

## 4. Topology Skin V2

### 4.1 结构层要补全的拓扑显示

当前需要完整补齐以下结构 tile family 的 skin：

- `outer-wall-h`
- `outer-wall-v`
- `outer-corner-tl`
- `outer-corner-tr`
- `outer-corner-bl`
- `outer-corner-br`
- `outer-t-up`
- `outer-t-right`
- `outer-t-down`
- `outer-t-left`
- `outer-cross`
- `outer-end-up`
- `outer-end-right`
- `outer-end-down`
- `outer-end-left`

内墙与院墙采用同样的拓扑集合，只在视觉色值和语义上区分：

- `inner-*`
- `yard-*`

### 4.2 开口层要补全的显示

开口层需要清晰区分：

- `door-leaf-h-open`
- `door-leaf-h-closed`
- `door-leaf-v-open`
- `door-leaf-v-closed`

门板仍然遵守已经确认的规则：

- 门板占满整格
- 不采用压在线上的半格门方案
- 开门态和关门态都是完整 tile

### 4.3 视觉目标

本轮的视觉目标不是“更花哨”，而是“更能准确表达房屋结构”：

- 角要真正连通两条墙
- T 字要清晰表达主墙和支墙连接
- 十字要表达四向连接
- 端头要清楚地表达“墙在这里结束”
- 开门和关门要一眼能区分

### 4.4 视觉优先级

本轮 skin 优先级按以下顺序实现：

1. 连通性正确
2. 线宽统一
3. 整格铺满
4. 开关门区分清楚
5. 再考虑细节装饰感

## 5. 工具系统 V2

### 5.1 工具列表

当前工具栏扩展为：

- 画笔
- 橡皮
- 吸管
- 填充
- 边框矩形
- 填充矩形
- 撤销
- 重做
- 保存
- 另存为
- 设为运行时地图

### 5.2 边框矩形

适合：

- 房屋外轮廓
- 院墙
- 房间边界

行为规则：

- 鼠标按下记录起点
- 拖到终点形成矩形包围盒
- 松开后只绘制外边框
- 只改当前层

如果当前选中 tile 是横墙或竖墙，系统不自动替你推断角/T/十字。

本轮首版的基础策略是：

- 边框矩形优先用于地表和区域层时，可直接生成整圈相同 tile
- 在结构层，边框矩形建议与精确拓扑 palette 配合使用
- 如果选中的 tile 不适合矩形边框场景，UI 需要给出提示而不是静默乱画

### 5.3 填充矩形

适合：

- 铺室内地面
- 铺院地
- 铺区域层
- 铺美术覆盖层

行为规则：

- 鼠标按下记录起点
- 拖到终点形成矩形区域
- 松开后将整个矩形内全部写为当前 tile
- 只作用于当前层

### 5.4 填充

填充是单击型工具，不是拖拽型工具。

首版规则：

- 从当前格开始
- 只在当前层内工作
- 以“同一层中与起点相同 tileId 的连通区域”为边界
- 将该区域替换为当前选中 tile

这样做的原因：

- 规则简单
- 用户预期明确
- 不需要额外引入跨层判断
- 不会把不同逻辑层混在一起

### 5.5 吸管

吸管只读取当前层。

行为规则：

- 点击某格
- 读取该格当前层的 tileId
- 立即把它设为当前选中 tile
- 不修改地图内容

这样能提升：

- 墙角复用效率
- 地面材质复用效率
- 门位与门板切换效率

## 6. 工具与层的适配

### 6.1 结构层

结构层推荐使用：

- 画笔
- 橡皮
- 吸管
- 边框矩形（有限制）

结构层不推荐把“填充”作为主工具使用，因为容易把复杂拓扑一口气刷坏。

### 6.2 地表层

地表层推荐使用：

- 画笔
- 填充
- 填充矩形
- 吸管

这是铺草地、院地、室内地面的主工作层。

### 6.3 区域层

区域层推荐使用：

- 填充矩形
- 填充
- 画笔

这样能更快标注：

- `yard`
- `frontRoom`
- `sleepRoom`
- `storage`
- `kitchen`

### 6.4 开口层

开口层推荐使用：

- 画笔
- 橡皮
- 吸管

门板和门位的关系需要精确控制，不建议在本轮给开口层启用矩形工具。

### 6.5 美术覆盖层

美术覆盖层推荐使用：

- 画笔
- 填充
- 填充矩形
- 吸管

它最适合配合未来的小像素块微纹理美化。

## 7. 校验与限制

### 7.1 门位合法性继续保留

当前已实现：

- 门板必须落在匹配的 `door-socket` 上

本轮保持不变，并继续应用于：

- 单格画笔
- 吸管后再绘制
- 矩形工具最终落图结果
- 保存
- 发布

### 7.2 工具限制提示

如果用户当前选中的 tile 与当前工具不适配，应明确提示，而不是静默执行错误结果。

例如：

- 用 `outer-corner-tl` 执行填充矩形
- 用 `door-leaf-h-closed` 执行边框矩形
- 用 `door-leaf-v-open` 执行填充

本轮建议策略：

- 直接阻止执行
- 状态栏给出中文提示

### 7.3 锁层规则继续保留

锁定层不可被任何工具修改，包括：

- 画笔
- 橡皮
- 矩形工具
- 填充

## 8. 交互细节

### 8.1 工具切换

当前工具栏中，矩形工具拆成两个独立模式：

- `边框矩形`
- `填充矩形`

不在同一个按钮里再嵌子菜单，避免增加额外操作层级。

### 8.2 单击与拖拽优先级

画笔模式下：

- 单击：一格
- 长按拖动：连续路径

矩形模式下：

- 按下：记录起点
- 拖动：预览矩形范围
- 松开：一次性落图

填充模式下：

- 单击：执行填充

吸管模式下：

- 单击：读取当前层 tile

### 8.3 预览反馈

本轮建议新增最小预览反馈：

- 矩形工具拖动时显示半透明预览框
- 当前工具状态在工具栏中明确高亮
- 非法操作时在状态栏显示提示

本轮不做复杂 hover ghost 或多色批量预览系统。

## 9. 测试策略

### 9.1 单元测试

需要补的单元测试包括：

- 拓扑 skin 选择与 class 生成
- 边框矩形生成的目标坐标集合
- 填充矩形生成的目标坐标集合
- 填充算法的连通区域结果
- 吸管读取当前层 tile 的行为
- 工具在锁层下被拦截

### 9.2 E2E 测试

需要补的 Playwright 流程包括：

- 切换到边框矩形工具并画出房屋边界
- 切换到填充矩形工具并铺地面
- 使用填充工具对当前层某区域灌入指定 tile
- 使用吸管选中已有 tile 后继续绘制
- 非法工具与 tile 组合时出现提示

## 10. 实施顺序建议

建议按以下顺序实现：

1. 补齐拓扑 skin class 与整格视觉
2. 为工具系统增加：
   - `picker`
   - `fill`
   - `rectangle-outline`
   - `rectangle-fill`
3. 先做纯坐标生成与填充算法单测
4. 再接画布交互
5. 最后补 Playwright 回归

这个顺序可以保证：

- 先把表现层真相源补完整
- 再增加交互
- 最后统一回归

## 11. 当前确认结论

截至当前，本轮新增设计视为已确认：

- 下一轮采用“先可用”策略，而不是一次追求过度完整
- 墙门拓扑 skin 要补齐角/T/十字/端头/门开关
- 新增四个基础工具：
  - 边框矩形
  - 填充矩形
  - 填充
  - 吸管
- 矩形工具拆成两个独立模式，而不是单一矩形工具
- 填充基于当前层中“与起点相同 tileId 的连通区域”
- 暂不实现复杂批量选区与高级变换
