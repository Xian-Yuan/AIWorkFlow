# AIRPGWeb Map Tile Tiling Design

## Goal

为地图编辑器中的像素素材放置增加“单格内是否平铺”的明确语义，使小尺寸素材放进更大标准网格时，可以在放置前决定：

- 非平铺：只按真实像素尺寸显示
- 平铺：在当前单个地图格子里重复铺满

## Background

当前地图放置更偏向“一个 tileId 对应一个固定可视结果”。

但用户希望以下场景可控：

- 素材本身是 `16x16`
- 地图当前标准网格是 `32x32`
- 放置前可决定：
  - 只显示 `16x16`
  - 或在一个 `32x32` 格子里平铺 `4` 个 `16x16`

这意味着“素材尺寸”和“单格显示策略”必须解耦。

## Scope

### In Scope

- 为像素素材增加默认平铺字段
- 为地图调色板增加临时平铺切换入口
- 为 TileCanvas 增加单格内平铺渲染语义
- 定义平铺与非平铺的优先级与限制

### Out Of Scope

- 不改变地图格子本身的数据结构
- 不重新设计多格占地素材的放置体系
- 不在本次设计中支持旋转、镜像、九宫格等高级铺法

## Core Rule

### Two-Layer Decision

平铺决策由两层组成：

1. 素材默认值 `defaultTiling`
2. 当前地图放置临时值 `overrideTiling`

最终决策优先级：

- 若存在 `overrideTiling`，使用它
- 否则回退到 `defaultTiling`

### Render-Only Behavior

平铺与否只影响单个地图格子内部如何渲染素材，不改变：

- 素材原始像素数据
- 素材真实尺寸
- 地图单元本身仍然是一个格子

## Non-Tiling Behavior

当 `tiling = false` 时：

- 素材按真实像素尺寸显示
- 使用左上角锚点落入当前单格
- 若素材小于当前标准格子，则单格中剩余区域保持空白

示例：

- 当前标准格 `32x32`
- 素材真实尺寸 `16x16`
- 结果：该格子左上显示一个 `16x16` 素材，其余区域不自动补满

## Tiling Behavior

当 `tiling = true` 时：

- 素材在当前单格的标准像素范围内重复绘制
- 重复单位以素材真实尺寸为基准
- 直到填满当前单格的标准像素区域

示例：

- 当前标准格 `32x32`
- 素材真实尺寸 `16x16`
- 结果：当前单格内出现 `2 x 2` 共 `4` 个重复块

## Suitability Rules

### Suitable Case

最适合平铺的是：

- 小于或等于当前标准格子的素材

### Unsupported Case

对于大于当前标准格子的素材：

- 默认不启用平铺
- UI 上可禁用平铺开关或给出不可平铺提示

原因：

- 大于单格的素材进入“平铺”语义不符合直觉
- 也容易与多格占地素材的渲染逻辑冲突

## Data Model

素材元数据新增：

- `defaultTiling: boolean`

地图编辑器会话态新增：

- `tilePlacementTilingOverride: boolean | null`

其中：

- `null` 表示不覆盖，使用素材默认值
- `true` 表示强制平铺
- `false` 表示强制非平铺

## UI Design

### Palette Entry Control

在地图调色板中，每个像素素材项后增加一个简洁的“平铺”切换入口。

交互语义：

- 显示当前有效状态
- 点击可切换本次放置的平铺方式
- 不修改素材本体默认值

### Material Editor Default

在 `绘制素材` 或素材详情中，可编辑素材默认值：

- 默认平铺：开 / 关

这用于设定该素材更常见的推荐放置方式。

## Rendering Integration

TileCanvas 在绘制像素素材时，需要按单格内部进行两类路径分发：

### Path 1: Non-Tiled

- 使用素材真实宽高
- 只绘制一次
- 左上锚点显示

### Path 2: Tiled

- 读取当前标准格的标准像素尺寸
- 按素材真实宽高计算横向和纵向重复次数
- 在单格内重复背景切片绘制

## Search And Preview Impact

素材库卡片与缩略图本身不执行单格平铺预览，避免资产浏览和地图放置语义混淆。

素材详情中可以显示：

- 默认平铺：是 / 否

但真正单格平铺效果以地图调色板和地图画布为准。

## Risks

- 若平铺状态直接写回 `tileId` 而没有单独字段，地图中同一素材会失去统一身份
- 若非平铺与平铺共用同一个 background-size 逻辑，可能再次出现尺寸语义错乱
- 若对大于单格素材也强制平铺，会产生不可预测显示

## Acceptance Criteria

- 像素素材拥有 `defaultTiling`
- 地图调色板拥有当前放置用的临时平铺覆盖
- 非平铺时，小素材按真实尺寸显示在单格左上
- 平铺时，小素材在单个格子里合理重复铺满
- 素材默认值与临时覆盖的优先级明确且一致
