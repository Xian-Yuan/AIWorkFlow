## 三、强制深度搜索机制（MUST READ）

### 机制一：搜索轮次强制（最少 N 轮）

你必须执行至少 **N 轮搜索**，每轮搜索一个不同的关键词组合或不同的网站。N 轮是下限，不是上限——发现新线索就追加。

| 轮次 | 搜索目标 | 搜索源类型 | 关键词方向 |
|------|---------|-----------|-----------|
| R1 | [目标] | 🟢 GitHub | [关键词] |
| R2 | [目标] | 🔵 arXiv | [关键词] |
| ... | ... | ... | ... |

### 机制二：发现驱动追加搜索

每轮搜索后，如果发现以下情况，**必须追加搜索**（不计入 N 轮下限）：
- 搜索结果中提到了你不知道的项目/论文 → 追加搜索该项目/论文
- 搜索结果中的引用/参考列表有高价值条目 → 追加搜索那些条目
- 某个项目的 GitHub 仓库有架构设计文档 → 读取并搜索其中提及的相关项目
- 搜索结果中出现了新的关键词/术语 → 用新关键词追加搜索

### 机制三：每个项目必须深挖

找到项目后，不要只看 README。必须执行以下深挖步骤：
1. 读取 README.md（架构概述）
2. 读取项目的架构设计文档（`docs/`、`ARCHITECTURE.md`、`DESIGN.md`）
3. 读取核心模块的源码目录结构
4. 读取配置文件格式（YAML/JSON/TOML）
5. 读取 Issues 中标有 `architecture`/`design` 的讨论

### 机制四：交叉验证

对每个重要发现，至少在 2 个不同来源验证。

### 机制五：搜索源类型强制覆盖

每个搜索维度**必须**在至少 3 种不同类型的搜索源上执行。

| 搜索源类型 | 标记 | 要求 |
|-----------|------|------|
| 🟢 开源项目 (GitHub) | 必选 | 每轮至少 1 次搜索 |
| 🔵 学术论文 (arXiv/Scholar) | 必选 | 论文相关轮次必搜 |
| 🟡 技术博客 (Medium/Dev.to/个人博客) | 必选 | 实践相关轮次必搜 |
| 🟠 中文社区 (知乎/掘金/CSDN/B站) | 必选 | 至少 1 轮专搜 |
| 🟣 技术论坛 (Reddit/HN/StackOverflow) | 必选 | 至少 1 轮专搜 |
| 🔴 官方文档 (框架/规范) | 推荐 | 框架相关轮次搜 |
