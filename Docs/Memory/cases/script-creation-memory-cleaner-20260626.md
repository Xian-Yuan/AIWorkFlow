# 创作案例：《记忆清洗师》剧本创作全流程

## 一、做了什么

从已有 AI 视频提示词文档（v1）出发，经过 6+ 轮剧本逻辑打磨和规范完善，
最终产出：
1. **通用提示词模板**：prompt-template-universal.md（可复用到任何剧本）
2. **提示词规范对照表**：prompt-spec-checklist.md（v1→v2 升级 checklist）
3. **创作流程案例**：本文档（教训+改进建议）

## 二、核心创作原则（可复用）

### 剧本层面
1. **不解释为什么，只展示是什么**——背景用画面暗示，不给文字交代
2. **记忆碎片是锤子不是拼图**——每个碎片独立有重量，合起来有递进
3. **具体动作 > 情绪标签**——"举外套"不是"失落"，"夹菜到空碗"不是"孤独"
4. **双视角制造情感落差**——他的美好回忆 vs 她不记得的日常片段
5. **结尾留白比抒情有力**——一行法律条文不反驳不解释
6. **反应弧需分阶段**——错愕→领悟→懊悔→痛苦→释然，每阶段用具体动作

### 提示词层面
1. **负面提示词分层**：通用层 + 镜头特定层（按手部/面部/外景风险选词）
2. **音频占 80%**：逐镜头指定环境音/SFX/BGM，静默也是一种声音
3. **灯光需要数值**：K/对比度/方向/特殊光源，不能只写"冷白顶光"
4. **服装逐镜追踪**：褶皱/位置变化写到每个镜头
5. **嘴唇规避**：对白镜头用侧脸/背影/切屏
6. **手部保护**：含手部特写的镜头加特定负面词

### 工作流层面
1. **创作决策用户拍板，技术实现 AI 自主**
2. **剧本逻辑必须先打磨到无漏洞，再跑 pipeline**
3. **研究文档前置**——创作前先读完所有 research_*.txt
4. **对话过程沉淀为案例**——被否的方案和原因同等重要

## 三、关键决策记录

| 决策 | 被否方案 | 最终选择 | 原因 |
|------|----------|----------|------|
| 反转设计 | A/B/C 三种 | 全部否决 | 新增设定太多 |
| 碎片情绪 | "心里是闷的" | 具体动作 | AI 无法生成抽象情绪 |
| 碎片结构 | 1个/3个 | 4个(暖→走→等→放) | 有对比有递进 |
| 碎片视角 | 混合 | 纯女主中性视角 | 与男主暖色回忆形成双视角 |
| 男主反应 | 分散在碎片流中 | 集中清除后一次给 | 积蓄力量 |
| 结尾 | 字幕抒情 | 行动意象 | 留白更有力 |
| 失忆原因 | 事故/离开/多次清除 | 不解释 | "第二次清除"暗示 |
| 术前告知 | 无 | 加入 | 制造戏剧张力 |

## 四、工作流改进建议

### 当前流程
```
灵感(S0) → 立项(S1) → 剧本(S2) → 角色(S3) → 分镜(S4) → 视频提示词(S5)
```

### 建议加入
```
灵感(S0) → 立项(S1) → 剧本(S2) → 【剧本审查 S2.5】 → 角色(S3) → 分镜(S4) → 连续性(S4.5) → 视频提示词(S5) → 【规范审查 S5.5】
```

**S2.5 剧本审查（新增）：** 在生成角色/分镜之前，用 checklist 检查：
- [ ] 失忆/分离/重逢等核心设定的逻辑自洽
- [ ] 每个角色的反应是否符合人性（不装、不跳）
- [ ] 碎片/闪回有因果递进
- [ ] 开场有钩子（前3秒决定留存）
- [ ] 结尾有留白（不给解释，只给余韵）

**S5.5 规范审查（新增）：** 生成提示词后自动检查 16 个维度是否全覆盖。

### 研究文档前置
创作阶段开始时必须自动加载所有 research_*.txt，确保从行业标准出发。

### 案例检索
下次创作新剧本时，先检索 Docs/Memory/cases/ 中的创作案例，
提取可复用的原则和被否方案的教训。
